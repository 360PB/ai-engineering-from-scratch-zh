# 思维树和 LATS：审慎搜索

> 单链思维轨迹没有回溯空间。ToT（Yao 等人，2023）将推理变成一棵树，每个节点自我评估。LATS（Zhou 等人，2024）用蒙特卡洛树搜索统一 ToT、ReAct 和 Reflexion。24 点游戏从 4%（CoT）提高到 74%（ToT）；LATS 在 HumanEval 上达到 92.7% pass@1。

**类型：** 构建
**语言：** Python（标准库）
**前置要求：** Phase 14 · 01（Agent 循环）、Phase 14 · 03（Reflexion）
**时间：** 约 75 分钟

## 学习目标

- 将推理框定为搜索：节点是"思考"，边是"扩展"，值是"多有前景"。
- 实现标准库 ToT 风格广度优先树搜索，带自我评估评分。
- 扩展到带选择 / 扩展 / 模拟 / 反向传播的玩具 LATS MCTS 循环。
- 决定何时搜索值得 token 倍增器（24 点游戏、代码生成），何时单一轨迹足够（简单问答）。

## 问题

链式思维是一条直线。如果第一步错了，每个后续步骤都在错误前提下工作。在 24 点游戏（用四个数字通过 + − × ÷ 得到 24）上，GPT-4 CoT 达到 4% 准确率。模型很早就选错了子表达式，无法恢复。

推理需要的是提出多个候选、评估它们、选择有前景的候选，并在死路出现时回溯的能力。这就是搜索。思维树和 LATS 是两个规范形式。

## 概念

### 思维树（Yao 等人，NeurIPS 2023）

每个节点是一个连贯的中间步骤（"一个思考"）。每个节点可以扩展到 K 个子思考。LLM 用评分提示自我评估每个节点。搜索探索树——BFS、DFS 或 beam。

```
                     (根: "从 4 6 4 1 找 24")
                    /               |            \
           ("6 - 4 = 2")    ("4 + 1 = 5")    ("4 * 6 = 24")  <- 评分: 高
              /   \              |                  |
          ...    ...          ...                完成
```

自我评估是承重部分。论文展示三种变体：`sure / likely / impossible` 分类、`1..10` 数值评分和候选间投票。三种都在 24 点游戏上大幅击败 CoT（GPT-4 的 4% → 74%）。

### LATS（Zhou 等人，ICML 2024）

LATS 在 MCTS 下统一 ToT、ReAct 和 Reflexion。LLM 扮演三个角色：

- **策略**：提出候选下一步动作（ReAct 风格）。
- **价值函数**：对部分轨迹评分（ToT 风格自我评估）。
- **自我反思者**：失败时写一段自然语言反思（Reflexion 风格）并用它来重新播种未来 rollout。

环境反馈（观察）混入价值函数，因此搜索由真实工具结果告知，而不仅仅是模型意见。论文当时结果：HumanEval pass@1 92.7%（GPT-4，SOTA），WebShop 平均 75.9（GPT-3.5，接近梯度微调）。

### MCTS，最小化

每次迭代四个阶段：

1. **选择** — 使用 UCT（树上置信上限）从根走到叶子。
2. **扩展** — 通过策略生成 K 个子节点。
3. **模拟** — 使用策略从子节点 rollout，使用价值函数评分叶子（或环境奖励）。
4. **反向传播** — 沿路径更新访问计数和价值估计。

UCT 公式：`Q(s, a) + c * sqrt(ln N(s) / N(s, a))`。第一项是开发；第二项是探索。为每个任务调整 `c`。

### 成本现实

搜索爆炸式消耗 token。24 点游戏上的 ToT 使用 CoT 的 100-1000 倍 token。LATS 类似。这不是免费的；将搜索保留用于：

- 单一轨迹明显不足的任务（24 点游戏、复杂代码）。
- 正确性比挂钟时间更重要的任务。
- 有廉价、可靠价值函数的任务（代码的单元测试、math 的显式目标）。

如果你的任务有单一正确答案和嘈杂评估器，搜索通常使事情更糟——它找到一个"评分好"但错误的答案。

### 2026 年定位

大多数生产 Agent 不运行 LATS。它们运行带工具接地验证的 ReAct（CRITIC，第 05 课）。搜索出现在专门领域：

- 以测试作为价值函数的编码 Agent（HumanEval 风格）。
- 探索多个查询路径的深度研究 Agent。
- LangGraph 子图中的规划密集型工作流。

AlphaEvolve（第 11 课）是 2025 年极端：代码上的进化搜索、机器可检查 fitness、前沿收益（56 年来第一个 4x4 矩阵乘改进）。

## 构建它

`code/main.py` 实现：

- 在风格化"选择算术运算"任务上的小型 ToT BFS。
- 在同一任务上的玩具 LATS MCTS 循环（选择 / 扩展 / 模拟 / 反向传播）带 UCT 选择。
- 一个组合符号评分加自我评估评分的价值函数。

运行它：

```
python3 code/main.py
```

追踪显示 ToT 每个节点扩展三个候选，BFS，与 LATS 通过 MCTS 收敛到最佳 rollout 对比。两者都打印 token 计数。

## 使用它

LangGraph 将 ToT 风格探索作为子图模式发货；LangChain 团队关于 LATS 的博客（2024 年 5 月）是参考教程。LlamaIndex 发货一个 `TreeOfThoughts` Agent。对于大多数 2026 年生产 Agent，此模式位于 `if task_complexity > threshold: use_search()` 门控之后——见第 05 课评估器-优化器模式。

## 发布它

`outputs/skill-search-policy.md` 给定任务形状、预算和评估器保真度，在线性 ReAct、ToT、LATS 和进化搜索之间选择。

## 练习

1. 用 UCT c=0.1 vs c=2.0 运行玩具 LATS。追踪中什么改变了？

2. 将价值函数交换为更嘈杂的评分器（添加随机抖动）。MCTS 仍然找到最佳叶子吗？它能容忍的最小信噪比是多少？

3. 实现 beam-search ToT（在每层保留 top-k）并与 BFS 比较。在紧张 token 预算上哪个更好？

4. 阅读 LATS 第 5.1 节。重现 HumanEval 轨迹计数：要达到报告的 pass@1 需要多少次 rollout？

5. 阅读 LATS 论文关于"LATS 帮助较少的场景"的讨论。写一段决策规则，将任务形状映射到搜索策略。

## 关键术语

| 术语 | 人们通常说 | 实际含义 |
|------|-----------|----------|
| Tree of Thoughts（思维树） | "分支 CoT" | Yao 等人——带自我评估的思维节点树 |
| LATS | "LLM 的 MCTS" | Zhou 等人——在 MCTS 下统一 ToT + ReAct + Reflexion |
| UCT | "置信上限" | 平衡开发（Q）和探索（ln N / n）的选择公式 |
| Value function（价值函数） | "这个状态有多好" | 提示的 LLM 评分或环境奖励；反馈反向传播 |
| Policy（策略） | "动作提议器" | ReAct 风格生成器；发出候选下一步思考/动作 |
| Rollout（Rollout） | "模拟轨迹" | 使用策略从节点走到叶子，用价值评分 |
| Backpropagate（反向传播） | "更新祖先" | 将叶子的奖励沿路径推送，更新访问计数和 Q |
| Search cost（搜索成本） | "Token 爆炸" | 24 点游戏上比 CoT 多 100-1000 倍；采用前预算 |
