# Agent 失败模式：为何 Agent 会出错

> MASFT（Berkeley，2025）将 14 种多 Agent 失败模式分为 3 大类。Microsoft 的分类学记录了现有 AI 失败如何在 Agent 场景下放大。行业现场数据收敛到 5 种反复出现的模式：幻觉操作、范围蔓延、级联错误、上下文丢失、工具误用。

**类型：** 概念学习 + 动手实现
**语言：** Python（标准库）
**前置知识：** 第 14 阶段 · 05（Self-Refine 和 CRITIC），第 14 阶段 · 24（可观测性）
**时间：** 约 60 分钟

## 学习目标

- 说出 MASFT 的三大失败类别和每类至少四种具体模式。
- 解释 Agent 化为何会放大现有 AI 失败模式（偏见、幻觉）。
- 描述五种行业反复出现的模式及其缓解措施。
- 实现一个标准库检测器，为 Agent 追踪打上失败模式标签。

## 问题背景

团队交付的 Agent 在 90% 的追踪上是正常的。这 10% 的失败不是随机噪声——它们落入有限数量的反复出现的类别。一旦能说出它们的名字，就能监控它们并修复它们。

## 核心概念

### MASFT（Berkeley，arXiv:2503.13657）

多 Agent 系统失败分类学。14 种失败模式聚为 3 类。标注者间 Cohen's Kappa = 0.88——类别间可可靠区分。

核心主张：失败是多 Agent 系统的根本性设计缺陷，而非可通过更好基础模型修复的 LLM 局限。

### Microsoft Agentic AI 系统失败模式分类学

- 现有 AI 失败（偏见、幻觉、数据泄露）在 Agent 场景下放大。
- 自主性带来新失败：规模上的非预期操作、工具误用、任务漂移。
- 这份白皮书是 Agent 产品的风险登记册。

### Agentic AI 中的故障特征（arXiv:2603.06847）

- 失败源于编排、内部状态演化和环境交互。
- 不仅仅是"坏代码"或"模型输出差"。

### LLM Agent 幻觉调查（arXiv:2509.18970）

两种主要表现：

1. **指令遵循偏差** — Agent 未遵循 system prompt。
2. **长程上下文误用** — Agent 遗忘或误用早期轮次中的上下文。

子意图错误：遗漏（漏做步骤）、冗余（重复步骤）、乱序（步骤顺序颠倒）。

### 五种行业反复出现的模式

Arize、Galileo、NimbleBrain 2024–2026 现场分析收敛到：

1. **幻觉操作。** Agent 调用了不存在的工具，或捏造参数。
2. **范围蔓延。** Agent 扩大了任务范围，超出用户请求（创建额外的 PR、多发邮件）。
3. **级联错误。** 一个错误调用触发下游连锁反应。一个虚假的 SKU 幻觉触发四个 API 调用——多系统事故。
4. **上下文丢失。** 长周期任务遗忘早期轮次的约束。
5. **工具误用。** 调用了正确的工具但参数错误，或干脆调用了错误的工具。

级联是致命的那种。Agent 无法区分"我失败了"和"任务不可能完成"，经常在收到 400 错误时幻觉出成功消息来闭合循环。

### 缓解：每步加门

在推理链的每步设置自动验证门，与环境状态对照检查事实基础。具体来说：

- 每步安全分类器（第 21 课）。
- 工具调用参数验证（第 6 课）。
- 交叉检查检索内容与已知事实（第 5 课，CRITIC）。
- 通过重新探测状态来检测成功幻觉（文件真的创建了吗？）。

### 失败监控的常见误区

- **只标记崩溃。** 大多数 Agent 失败产生看似有效的输出。需要内容级检查。
- **没有基准。** 漂移检测需要一个"上次正常"状态；没有它无法判断"这在变差"。
- **告警过多。** 每个失败都发通知。聚类并限速。

## 练习

1. 添加"成功幻觉"检测器：Agent 报告成功但目标状态未变。
2. 给 100 条你构建的产品真实追踪打标签。哪种模式最多？修复成本是多少？
3. 实现"级联半径"指标：给定第 N 步的失败，影响了多少下游步骤？

5. 将一个检测器接入 CI：若 ≥5% 的追踪命中某模式则构建失败。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| MASFT | "多 Agent 失败分类学" | Berkeley 的 14 模式分类 |
| Cascading error | "连锁失败" | 早期一个错误向后续 N 步蔓延 |
| Context loss | "遗忘约束" | 长周期轮次丢失早期轮次的上下文 |
| Tool misuse | "工具/参数错误" | 调用有效但执行不当 |
| Success hallucination | "伪造完成" | Agent 在收到 400 时声称成功；状态未变 |
| Scope creep | "越界" | Agent 做了超出要求的事 |
| Instruction-following deviation | "不服从" | 忽略 system prompt 或用户约束 |
| Sub-intention errors | "计划缺陷" | 执行计划时的遗漏、冗余、乱序 |
