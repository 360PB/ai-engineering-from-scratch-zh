# 评估驱动的 Agent 开发

> Anthropic 的指引："从简单提示词开始，用完整评估来优化，只有在必要时才引入多步 Agent 系统。" 评估不是最后一步。它是驱动 Phase 14 所有其他决策的外部循环。

**类型：** 学习 + 动手实现
**语言：** Python（标准库）
**前置要求：** Phase 14 全部内容
**时长：** 约 60 分钟

## 学习目标

- 说清楚三层评估体系——静态基准测试、自定义离线评估、在线生产评估——以及各自的用途。
- 解释评估器-优化器紧循环。
- 描述 2026 年最佳实践：评估与代码同址、在 CI 中运行、拦截 PR。
- 将 Phase 14 的每一讲连接到它所生成的评估用例。

## 问题

Agent 在演示中通过，在生产中失败，而演示无法预测这些失败。基准测试回答的是"这个模型整体能力如何"，而不是"这个 Agent 是否在为我的产品交付正确的补丁"。解决方案是三层评估体系，持续运行，将每条护栏和学到的规则都映射到一个评估用例。

## 概念

### 三层评估体系

1. **静态基准测试** — SWE-bench Verified 用于代码（Lesson 19），WebArena/OSWorld 用于浏览器/桌面（Lesson 20），GAIA 用于通用能力（Lesson 19），BFCL V4 用于工具调用（Lesson 06）。用于跨模型比较和回归拦截。污染是真实存在的：SWE-bench+ 发现 32.67% 的解决方案泄漏。始终报告 Verified 分数或经审计分数。

2. **自定义离线评估** — 贴合你的产品形态：
   - LLM 作为裁判（Langfuse、Phoenix、Opik — Lesson 24）。
   - 基于执行（运行补丁，检查测试）。
   - 基于轨迹（将动作序列与黄金轨迹对比；OSWorld-Human 显示顶级 Agent 的步数是黄金轨迹的 1.4–2.7 倍）。

3. **在线评估** — 生产环境：
   - 会话回放（Langfuse）。
   - 护栏触发告警（Lesson 16、21）。
   - 每步成本/延迟追踪（Lesson 23 OTel 链路）。

### 评估器-优化器循环（Anthropic）

紧循环：

1. 提议者生成输出。
2. 评估器评判。
3. 反复优化直到评估通过。

这是 Self-Refine（Lesson 05）的通用化形式。任何你在意的 Agent 流程都可以包装在评估器-优化器中以提升可靠性。

### 2026 年最佳实践

- 评估与代码同址。
- 每个 PR 在 CI 中运行评估。
- 拦截合并，评估分数下降（例如"相比 main 不允许下降超过 5%"）。
- 每条护栏映射到一个评估用例。
- 每条学到的规则（Reflexion、pro-workflow 学规则）映射到一个失败用例。

### 评估驱动开发的失败场景

- **没有基线。** 没有上一个已知正常的基线，评估结果无法解读。保存基线。
- **LLM 裁判缺乏依据。** 裁判也会产生幻觉。CRITIC 模式（Lesson 05）——让裁判基于外部工具获取依据。
- **过度拟合评估。** 优化评估得分，但偏离了生产实用价值。轮换用例。
- **评估不稳定。** 非确定性用例导致误报。固定随机种子、快照状态。

## 动手实现

`code/main.py` 是一个标准库评估框架：

- 用例注册表（含分类：基准测试、自定义、在线）。
- 被测脚本 Agent。
- 评估器-优化器循环：提议、评判、反复精炼直到通过或达到最大轮次。
- CI 拦截：聚合通过率 + 与基线的回归对比。

运行：

```
python3 code/main.py
```

输出：每个用例的通过/失败、回归标志、CI 拦截判定。

## 用现成库

- 在与 Agent 代码相同的仓库中编写评估用例。
- 通过 CI 在每个 PR 上运行。
- 发生回归时让构建失败。
- 追踪通过率随时间的变化。
- 将每个生产故障关联到一个新用例。

## 产出

`outputs/skill-eval-suite.md` 构建三层评估套件，含 CI 拦截和回归追踪。

## 练习

1. 选取一个生产故障。编写一个能复现它的评估用例。你的 Agent 现在能通过吗？
2. 为你的领域构建一个三维度 LLM 裁判评分表（事实性、语气、范围）。对 50 个会话打分。
3. 将评估套件接入 CI。回归 ≥5% 时让构建失败。
4. 添加轨迹效率指标：Agent 用了多少步，与黄金轨迹相比如何？
5. 将 Phase 14 每讲映射到评估套件中的一个用例。发现有遗漏？那就是需要补齐的差距。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| 静态基准测试 | 现成评估 | SWE-bench、GAIA、AgentBench、WebArena、OSWorld |
| 自定义离线评估 | 领域评估 | 基于你的产品形态的 LLM 裁判/执行/轨迹评估 |
| 在线评估 | 生产评估 | 会话回放、护栏告警、成本/延迟追踪 |
| 评估器-优化器 | 提议-评判-精炼 | 反复迭代直到裁判通过 |
| CI 拦截 | 合并阻断器 | 评估发生回归时让构建失败 |
| 基线 | 上一个已知正常 | 用于检测回归的参考分数 |
| 轨迹效率 | 步数与黄金轨迹比较 | Agent 步数除以人类专家最少步数 |
