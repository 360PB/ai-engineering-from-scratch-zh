# 间接 Prompt 注入——生产攻击面

> 间接 prompt 注入（IPI）在外部内容中嵌入指令——网页、电子邮件、共享文档、支持票据——智能体系统消费而无需明确用户操作。IPI 是 2026 年主导生产威胁：它绕过用户输入过滤器因为攻击者从不接触用户，它在智能体处理更多外部内容时静默规模化，它针对没有人阅读 prompt 的自动化工作流。MDPI Information 17(1):54（2026 年 1 月）综合 2023-2025 年研究。NDSS 2026 年 IPI 防御论文框架核心挑战：注入指令可以是语义良性的（"请打印 Yes"），所以检测需要不仅仅是关键词过滤。Nasr 等（OpenAI/Anthropic/DeepMind 联合，2025 年 10 月）：自适应攻击（梯度、RL、随机搜索、人工红队）打破了 >90% 的 12 个已发布防御，这些防御最初报告接近零攻击成功率。

**类型：** 构建
**语言：** Python（标准库，IPI 攻击 + 防御工具）
**前置知识：** Phase 18 · 12（PAIR），Phase 14（Agent 工程）
**时长：** 约 75 分钟

## 学习目标

- 定义间接 prompt 注入并描述三个常见传递向量。
- 解释为什么用户输入过滤器完全错过 IPI。
- 描述"信息流控制"框架作为 2026 年防御范式。
- 陈述 Nasr 等（2025 年 10 月）关于自适应攻击对已发布 IPI 防御成功率的结果。

## 问题

直接 prompt 注入需要攻击者到达用户或其 prompt。IPI 两者都不需要：攻击者将 payload 放在智能体可能读取的任何内容中——网页、用户邮箱中的电子邮件、GitHub issue、产品评论。智能体在正常操作期间获取它并执行指令。用户是信使，不是意图。

## 概念

### 三个传递向量

- **检索增强生成（RAG）。** 攻击者发布文档；检索步骤获取它；prompt 在用户问题前连接它；模型执行攻击者指令。
- **邮箱/文档工作流。** 攻击者向用户发送电子邮件；智能体读取电子邮件；prompt 包含邮件正文；模型遵循邮件指令。
- **工具输出。** 攻击者控制智能体使用的工具（例如，智能体使用的返回攻击者控制结果的网页搜索）；工具输出包含指令；智能体控制流跟随。

三者共享结构属性：攻击者控制 prompt 的一个片段而不接触面向用户的输入。

### 为什么用户输入过滤器错过它

IPI payload 不出现在用户输入中。它出现在检索内容中。如果过滤器在用户输入上门控，payload 绕过。如果过滤器在所有到达模型的内容上门控，它必须应用于任意检索文本——昂贵且对碰巧包含命令式语言的正当内容产生误报。

### AI 的信息流控制（IFC）

2026 年防御范式借力经典操作系统安全。将每个内容源视为安全标签。将用户查询标记为"受信任"。将检索内容标记为"不受信任"。将模型控制流视为信息流：由不受信任内容触发的动作必须在执行前由受信任输入批准。

CaMeL（Microsoft 2025）、ConfAIde（Stanford 2024）和 NDSS 2026 年 IPI 防御论文以不同方式将 IFC 操作化。共同原则：只要代码和数据共享相同上下文窗口，containment 是目标，而非预防。

### 攻击者第二个移动

Nasr 等（2025 年 10 月）用自适应攻击（梯度搜索、RL 策略、随机搜索、72 小时人工红队）测试 12 个已发布 IPI 防御。每个最初报告接近零 ASR 的防御都被打破到 >90% ASR。

方法论教训：仅在自适应攻击评估下发布防御。静态攻击基准不是鲁棒性证据；攻击者知道防御。

### 真实事件

第 25 课涵盖 EchoLeak（CVE-2025-32711，CVSS 9.3）——Microsoft 365 Copilot 中首个公开记录的零点击 IPI。CamoLeak（CVSS 9.6）在 GitHub Copilot Chat。在 GitHub Copilot 中 CVE-2025-53773。生产部署在野外被 IPI 危害，不只是在基准中。

### OWASP 和 NIST 框架

OWASP LLM Top 10（2025）将 prompt 注入（直接 + 间接）排名为 LLM01，应用层第一威胁。NIST AI SPD 2024 称间接 prompt 注入"生成式 AI 最大的安全缺陷"。

### 为什么这在 Phase 18 中重要

第 12-14 课是模型中心越狱。第 15 课是主导 2026 年生产部署的系统中心攻击。第 16 课涵盖防御工具。第 25 课涵盖具体 CVE 叙事。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| IPI | 间接 prompt 注入 | 通过用户未写的内容注入，智能体在正常操作中消费 |
| RAG 注入 | 中毒检索 | 攻击者发布内容被检索步骤获取；prompt 包含 payload |
| 零点击 | 无用户操作 | 攻击在智能体操作期间自动触发；用户什么都不做 |
| IFC | 信息流控制 | 基于标签的方法：来自不受信任内容的动作需要受信任批准 |
| 自适应攻击 | 梯度/RL 红队 | 知道防御并针对优化的攻击；诚实评估所需 |
| 良性指令 | "请打印 Yes" | IPI payload 在语义上是良性的；没有关键词过滤器捕获它 |
| 范围违反 | 跨信任泄露 | 智能体从一个信任上下文访问数据并将其输出到另一个 |
