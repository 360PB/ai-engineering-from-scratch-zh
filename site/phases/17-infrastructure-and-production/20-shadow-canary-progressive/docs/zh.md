# 影子流量、金丝雀发布和 LLM 渐进式部署

> LLM 上线结合了软件部署中最难的部分：无单元测试、弥散的失败模式、延迟信号。序列是：(1) 影子模式——将生产请求复制到候选模型，记录，比较，零用户影响；捕获明显分布问题但不是质量保证；(2) 金丝雀发布——渐进式流量转移 10% → 25% → 50% → 75% → 100%，每步有关卡；追踪延迟百分位数、每请求成本、错误/拒绝率、输出长度分布、用户反馈率；(3) A/B 测试在稳定性确认后比较不同方案。非决定性不可约——相同输入下运行间精度差异可达 15%，来自 GPU FP 非结合性加批大小方差。成本是变量，不是常量——好 20% 的模型可能每调用贵 3 倍。回滚速度决定一切：如果回滚需要重新部署，你太慢了。策略在 config/flags 里；模型在带固定摘要的仓库里；回滚 = 翻转策略 + 恢复阈值 + 秒级固定旧模型。

**类型：** 精读
**语言：** Python（标准库，玩具级金丝雀进度模拟器）
**前置要求：** Phase 17 · 13（可观测性）、Phase 17 · 21（A/B 测试）
**时长：** 约 60 分钟

## 学习目标

- 区分影子模式（零影响比较）、金丝雀（实时流量渐进）和 A/B（稳定性确认后比较）。
- 枚举五个 LLM 特有的金丝雀指标（延迟、每请求成本、错误/拒绝、输出长度分布、用户反馈）。
- 解释为什么 LLM 非决定性（高达 15%）改变了"稳定"在发布中的含义。
- 设计秒级回滚路径（策略翻转）而非小时级（重新部署）。

## 背景问题

你上线了一个新模型。离线评估显示精度提升 3%。你在生产翻了开关。24 小时内，成本上升 40%，用户点踩上升 8%，三个客户工单报告"答案很奇怪"。你回滚。重新部署花 3 小时。你的周末毁了。

每一步都是可以避免的。影子模式会在任何用户看到之前捕获 40% 成本尖峰。金丝雀会在 10% 时因为点踩移动而停止。策略标志回滚只需 30 秒。纪律弥合了"离线评估看起来不错"和"真实用户开心"之间的差距。

## 核心概念

### 影子模式

候选模型接收与生产相同的请求；输出被记录，但不返回给用户。零用户影响。记录：

- 输出内容（与生产 diff）。
- Token 数（成本差异）。
- 延迟。
- 拒绝和错误。

捕获：成本失控、长度回归、明显拒绝变化、硬错误。不捕获：用户会感知到的质量差异。影子是冒烟测试，不是质量测试。

### 金丝雀发布

渐进式流量转移带关卡。典型进度：1% → 10% → 25% → 50% → 75% → 100%。每步按 5 个指标关卡：

1. **延迟百分位数** — P50、P95、P99。突破：金丝雀 P99 > 基线 1.5 倍。
2. **每请求成本** — 混合美元。突破：比基线高 >20%。
3. **错误/拒绝率** — 5xx 加显式拒绝。突破：比基线高 2 倍。
4. **输出长度分布** — 均值 + P99。突破：分布偏移。
5. **用户反馈率** — 点踩/工单提交。突破：比基线高 1.5 倍。

### 非决定性是新的方差

相同输入产生不同输出。原因：

- GPU FP 非结合性（浮点归约顺序因批而异）。
- 批大小方差（同一提示词在 128 batch 和 16 batch 中不同）。
- 采样（temperature > 0）。

实测：相同评估集运行间精度差异高达 15%。发布中的"稳定"意味着指标在预期方差内，不是与基线完全一致。把关卡设在噪声之上。

### 成本是变量

好 20% 的模型可能每调用贵 3 倍。每请求成本是五个关卡之一。发布"更好"的模型但搞坏单位经济是回滚情况。

### 回滚是武器

- 策略标志（特性标志系统）：配置里翻百分比；秒级生效。
- 模型固定（仓库摘要）：固定模型不自动升级。
- 回滚 = 恢复标志 + 将固定摘要设为上一版。秒级，不是小时。

如果你的栈需要重新部署才能回滚，在上线前修复它。

### 工具链

**Argo Rollouts / Flagger** — Kubernetes 渐进式交付控制器。与 Istio/Linkerd 加权路由集成。

**Istio 加权路由** — 服务网格级流量分割。

**KServe / Seldon Core** — 带内置金丝雀的模型服务。

**特性标志** — LaunchDarkly、Flagsmith、Unleash。策略级翻转，无需重新部署。

### 指标节奏

金丝雀关卡每 5-15 分钟检查一次（取决于流量）。1% 流量 10 req/min 提供每窗口 50-150 数据点——延迟够但用户反馈嘈杂。10% 提供约 10 倍。各进度应暂停足够时间积累足够样本。

### A/B 步是可选的

如果新模型差异明显（不同行为、不同成本曲线、不同语气），金丝雀通过后在 50% 做 A/B 测试。如果只是改进版本，金丝雀关卡通过后跳到 100%。

### 必须记住的数字

- 金丝雀进度：1% → 10% → 25% → 50% → 75% → 100%。
- 非决定性上限：相同输入运行间差异高达 15%。
- 五个金丝雀指标：延迟、成本、错误/拒绝、输出长度、用户反馈。
- 成本关卡：比基线高 >20% 是突破。
- 回滚：秒级，不是小时。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|----------|----------|
| 影子模式 | "复制到新模型" | 零影响发送到候选模型用于记录 |
| 金丝雀 | "渐进流量" | 带关卡的渐进用户暴露发布 |
| 关卡 | "发布检查" | 阻止进度的指标阈值 |
| 非决定性 | "LLM 方差" | 运行间不可约差异 |
| 策略标志 | "标志翻转回滚" | 配置级回滚，秒级而非小时级 |
| 模型固定 | "仓库摘要" | 模型版本的不可变引用 |
| Argo Rollouts | "K8s 渐进式" | Kubernetes 原生金丝雀/回滚控制器 |
| KServe | "推理 K8s" | 带金丝雀原语的模型服务 |
| Istio 加权 | "网格分割" | 服务网格流量分割器 |
