# InternVL3：原生多模态预训练

> InternVL3 之前的每款开源 VLM 都遵循相同的四步方案：取一个在数万亿文本 token 上训练过的文本 LLM，接上视觉编码器，然后微调接缝处。这有效但有对齐债务——文本 LLM 已将其全部预训练预算花在纯文本上，不能原生理解视觉 token。当你事后添加视觉时，LLM 必须在不忘记文本的情况下重新学习如何将视觉输入与其文本推理关联。InternVL3（Zhu 等，2025 年 4 月）拒绝了事后方案：一次预训练运行，从第一步起文本和多模态就交错在一起。在 780 亿参数下匹配 Gemini 2.5 Pro 在 MMMU-Pro 上的表现。本节解读原生预训练的理由以及当你实现它时什么改变了。

**类型：** Learn
**语言：** Python（标准库，训练语料混合器）
**前置知识：** Phase 12 · 05，Phase 12 · 07（配方）
**时间：** 约 120 分钟

## 学习目标

- 解释为什么事后 VLM 训练累积对齐债务，引用三个可测量症状（灾难性遗忘、答案漂移、视觉-文本不一致）。
- 描述 InternVL3 的原生预训练语料混合及文本 : 交错 : 标题的比例为什么重要。
- 将 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE 比较。
- 说出视觉分辨率路由器（ViR）和解耦视觉-语言（DvD）部署优化。

## 问题背景

事后 VLM 训练是默认方案。LLaVA、BLIP-2、Qwen-VL、Idefics——都取一个已经预训练的 LLM（Llama、Vicuna、Qwen、Mistral）并添加视觉。训练阶段通常如下：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在标题对上训练以对齐 embedding。
2. 解冻 LLM，在指令数据上训练（LLaVA-Instruct、ShareGPT4V）。
3. 可选的任务特定微调。

对齐债务有三个症状：

- 灾难性遗忘。事后 VLM 忘记纯文本技能。GSM8K 分数下降 5-10 分。Hellaswag 分数下降。纯文本 agent 回归。
- 答案漂移。相同视觉问题的微小措辞变化得到不同答案。视觉编码器与 LLM 的连接比 LLM 自己 token 的连接弱。
- 视觉-文本不一致。VLM 能正确描述图像，然后回答与自身描述矛盾的问题。视觉 token 不以文本相同的方式参与 LLM 的内部一致性检查。

这些症状有充分记录。MM1.5 第 4 节量化了它们。LLaVA-OneVision 的消融暗示了它们。原生预训练是答案。

## 核心概念

### 原生多模态预训练

InternVL3 从一开始就是原生多模态的语料上从头训练。混合比例为：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图文数据（OBELICS、MMC4 风格）
- 20% 成对图文标题数据
- 5% 视频文本数据

从第一个梯度步起，视觉 token、文本 token 和跨模态交互都以相同的损失参与。无对齐预训练，无投影器冻结阶段，无需从灾难性遗忘中恢复。

基础模型训练是单阶段的。指令微调跟随，但基础模型已经从第一步就将视觉 token 理解为头等公民。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用固定轴分配的 M-RoPE。InternVL3 引入 V2PE：位置编码随模态类型（文本、图像、视频）变化，带可学习缩放。实际上：

- 文本 token 得到 1D 位置（文本索引）。
- 图像 patches 得到 2D 位置（行，列）。
- 视频帧得到 3D 位置（时间，行，列）。

三者共享相同的 RoPE 频率底数，但每波段隐藏维度的分配是学习参数而非固定分割。在预训练期间灵活权衡时间 vs 空间频率分辨率。

V2PE 的消融声称：在相同计算量下比 M-RoPE 在视频基准上高 1-2 分。不是革命，但更干净。

### 视觉分辨率路由器（ViR）

部署优化。并非所有图像都需要全分辨率编码。一张低细节的单物体照片在 1280px 原生编码时浪费 token。ViR 是一个小型分类器，在编码前预测回答问题所需的最小分辨率。

路由有三个层级：低分辨率（256 token）、中等（576）、高（2048+）。在生产流量中，60% 的查询低或中等就足够。净效果：同等质量下 2-3 倍吞吐量。

### 解耦视觉-语言部署（DvD）

当你要服务大型 VLM 时，视觉编码器每张图像运行一次，但 LLM 为每个输出 token 自回归运行。两个组件有不同的瓶颈（视觉 = GPU 内存带宽用于 conv + 注意力；LLM = KV 缓存）。DvD 将它们拆分到不同 GPU 上并流式交接。

对于 8B + 4 亿编码器模型，DvD 相对于共置约翻倍每节点吞吐量。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准主张：在 780 亿参数下匹配 Gemini 2.5 Pro 的 MMMU-Pro。在 380 亿下匹配 GPT-4o。在 80 亿下领先开源 8B 排行榜。全部基于单阶段预训练 + 指令微调配方。

对齐债务假说是可测量的：InternVL3-8B 每单位视觉基准收益失去的文本基准分数（ MMLU、GSM8K）比 Qwen2.5-VL-7B 少。模型更全面，因为训练是一体的，不是两体的。

### InternVL3.5 与 InternVL-U

InternVL3.5（2025 年 8 月）扩展配方。相同原生预训练方法，更多数据，更多参数。MMMU 改进是增量的。

InternVL-U（2026 年）添加统一生成——通过 MMDiT head 在相同骨干网上输出图像。"U"代表"理解 + 生成"，追赶 Transfusion 风格的统一模型（第 12.13 节）。相同的原生预训练骨干网支持理解和生成头。

### 原生预训练的权衡

原生预训练不是免费的：

- 计算。从头训练新 VLM 的成本与训练文本 LLM 相同——数百万 GPU 小时。事后适配重用现有 LLM 权重，节省大部分成本。
- 数据。大规模交错图文语料很少见。OBELICS 是 1.41 亿文档；MMC4 是 5.71 亿。纯文本发货 15T token。多模态预训练数据稀缺是硬约束。
- 基础 LLM 重用。原生预训练放弃了以后插入新 LLM 的选项。事后允许你通过只重训适配器将 Llama-3.1 换成 Llama-4。

InternVL3 的赌注：对齐债务比重用损失更糟糕。基准支持这一主张。生产成本壁垒阻止未来实验室廉价复制。事后 VLM 将继续存在，因为对大多数项目来说它们仍然更便宜。

## 使用方法

`code/main.py` 是训练语料混合器和 ViR 路由器模拟器。它：

- 接收目标语料混合（%文本、%交错、%标题、%视频）并计算每模态预期步数。
- 在一批查询（分布：50% 低细节、30% 中等、20% 高细节）上模拟 ViR 路由并报告平均 token 数。
- 给出给定编码器 vs LLM FLOPs 的 DvD 吞吐量估算。
- 打印事后 vs 原生预训练在参数、计算、数据和预期对齐债务症状上的并列比较。

## 练习

1. 估算 InternVL3-8B（原生预训练）和 LLaVA-OneVision-7B（事后）的计算 delta。GPU 小时大约的比例？什么解释差距？

2. InternVL3 报告 40% 文本 / 35% 交错 / 20% 标题 / 5% 视频。如果你的目标任务是视频繁重的，提出新比例并论证为什么基础模型仍然需要大量文本和标题数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。说出事后训练显示最大回归的确切基准。回归代价是多少？

4. ViR 将 60% 流量路由到低分辨率编码。它会错误路由什么样的查询（需要高分辨率但发送到低分辨率）？提出三种路由器失败模式。

5. DvD 将视觉和 LLM 拆分到不同 GPU。在什么流量模式下 DvD 反而损害吞吐量而不是帮助？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Native multimodal pretraining | "从头一起" | 文本 + 图像 + 视频 token 从第 1 步就参与损失，不是后来接上的 |
| Alignment debt | "事后惩罚" | 从将视觉螺栓固定在冻结 LLM 上带来的可测量的文本技能和答案一致性回归 |
| V2PE | "可变视觉位置编码" | 每模态可学习位置编码分配；InternVL3 的 M-RoPE 后继者 |
| ViR | "分辨率路由器" | 小型分类器，在编码前为每个查询选择所需最小分辨率，节省推理 token |
| DvD | "解耦部署" | 视觉编码器在一块 GPU，LLM 在另一块，带流式交接；大型 VLM 吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 年后续，在原生预训练骨干网上添加图像生成 head |
| Interleaved corpus | "OBELICS / MMC4" | 以自然阅读顺序包含文本和图像的文档；原生预训练的原材料 |
