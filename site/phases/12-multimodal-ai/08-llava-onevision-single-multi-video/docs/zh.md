# LLaVA-OneVision：单图像、多图像、视频统一模型

> 在 LLaVA-OneVision（Li 等，2024 年 8 月）之前，开源 VLM 世界有各自独立的传承：LLaVA-1.5 用于单图像，像 Mantis 和 VILA 这样的多图像模型，像 Video-LLaVA 和 Video-LLaMA 这样的视频模型。每个在其基准上获胜，在其他上失败。LLaVA-OneVision 论证了一个单一课程可以训练一个模型主导所有三种场景，且新兴的任务迁移效果（单图像技能导出到视频，多图像推理导出到单图像）胜过所有专家的总和。配方简单得骗人：一个视觉 token 预算在所有场景中保持恒定，加上一个明确的课程，从单图像到 OneVision（多图像）再到视频。本节解读预算、课程和新涌现的行为。

**类型：** Build
**语言：** Python（标准库，token 预算求解器 + 课程规划器）
**前置知识：** Phase 12 · 05（LLaVA），Phase 12 · 06（任意分辨率）
**时间：** 约 180 分钟

## 学习目标

- 设计一个在单图像、多图像和视频输入中保持恒定的视觉 token 预算。
- 订购训练课程，将技能从单图像迁移到视频而不发生灾难性遗忘。
- 解释当课程正确完成时，为什么单一模型以相同参数数量击败专家组合。
- 说出 LLaVA-OneVision 报告的三种新兴能力：多摄像头推理、Set-of-Mark 提示、iPhone 截图代理。

## 问题背景

图像、多图像和视频每个以不同方式考验模型。

单图像想要高分辨率 token（AnyRes，约 2880 视觉 token）来捕获 OCR 和细节。每样本预算：一张图像，2880 token。

多图像想要几张中等分辨率图像（每张约 576 token）以便跨图像推理落在上下文内。每样本预算：4-8 张图像，每张 576，2300-4600 token。

视频想要很多低分辨率帧（约每帧 196 token，池化后）来捕获时间动态。每样本预算：8-32 帧，每帧 196，1600-6200 token。

如果训练单独的模型，你选择一个预算。如果训练一个模型，你需要预算在场景间合理扩展而不爆炸上下文。

OneVision 之前，默认答案是"训练一个场景，忽略其他。"Video-LLaVA 用额外训练阶段将视频改装到图像模型。LLaVA-NeXT 用平铺添加多图像支持。都没有干净地处理三种场景。

## 核心概念

### OneVision token 预算

LLaVA-OneVision 选择统一每样本视觉 token 预算约 3000-4000 token，在不同场景中分配不同：

- 单图像：AnyRes-9（3x3 图块 + 缩略图），每个图块在 384 分辨率下 729 patches，激进双线性池化 2x2 → 每个图块 182 token。总计：9 * 182 + 182 = 1820 token。或 AnyRes-4 每图块 729 = 2916 + 729。
- 多图像：每张图像中等分辨率（384，无平铺），729 token 无池化。6 张图像预算 → 4374 token。
- 视频：32 帧在 384 分辨率下，激进 3x3 双线性池化 → 每帧 81 token。总计：32 * 81 = 2592 token。

分配保持总 token 大致恒定。LLM 从不看爆炸上下文的 batch。编码器产生不同几何形状的场景，但 LLM 消耗相同预算。

### 三阶段课程

LLaVA-OneVision 分三阶段训练：

1. 单图像 SFT（SI 阶段）。所有数据是单图像+文本。在高分辨率 AnyRes 输入上训练。这教感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加 OneVision 特定单图像数据。
2. OneVision SFT（OV 阶段）。混合单图像 + 多图像 + 视频（均匀采样帧）。在统一 token 预算上训练。这教模型处理异构 batch 形状。不重置权重——从 SI 阶段继续。
3. 任务迁移（TT 阶段）。用目标任务混合继续，通常根据产品更多地在多图像或视频上。外加可选的部署微调。

关键：课程顺序很重要。先训练视频或多图像再训练单图像，比单图像优先产生更差的图像性能，即使数据相同。论文明确做了这个消融。

### 为什么课程有效

单图像训练建立感知基础。Patch token 携带细粒度视觉特征；LLM 学习将它们与文本整合。多图像和视频引入结构挑战（哪张图像是什么，谁先发生），这在没有强感知基础的情况下很难学习。

如果你从零开始一起训练所有场景，模型会欠拟合感知（batch 中每样本的单图像数据有限）且过拟合结构（大量多图像/视频数据）。结果：一个遵循跨图像推理模式但视觉浅薄的模型。

课程排序给你从 SI 阶段感知能力，然后从 OV 阶段组合/时间推理，而不会失去任何一个。

### 跨场景新兴技能

LLaVA-OneVision 论文报告了三种新兴能力：

1. 多摄像头推理。在多图像 + 视频分开训练；在推理时被问到多摄像头驾驶场景推理。模型正确整合了视图，尽管训练中从未见过这种确切格式。
2. Set-of-Mark 提示。用户用数字标记注释图像中的物体；模型推理"标记 3 相对于标记 7 在做什么"。训练中没有标记也没有注释；从空间接地 + 多图像参考的组合中学习。
3. iPhone 截图代理。用户提供 iPhone 屏幕截图并要求计划下一步点击。训练数据包括 UI 截图、用户工作流视频和多图像前后对。泛化到代理用例。

这些不是训练任务；它们从课程的组合结构中涌现。

### 视觉 token 池化

Token 预算需要池化。OneVision 在 2D patch 网格上使用双线性插值：24x24 = 576 patches 变为 12x12 = 144（2 倍因子）或 8x8 = 64（3 倍因子）。池化在 patch 网格空间完成，不在 token 空间，以保留局部性。

每个场景的池化因子本身是一个超参数。少池化 = 更多 token = 更丰富的表示。多池化 = 更少 token = 更多帧/图像能放入。

### LLaVA-OneVision-1.5

2025 年续作（LLaVA-OneVision-1.5，arXiv 2509.23661）"完全开源"——训练数据、模型权重和代码。在一些基准上匹配专有差距，民主化配方。相同课程，更多数据，更好的基础 LLM。无架构改动。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（第 12.09 节）做出不同选择。它使用 M-RoPE 和动态 FPS 而不是固定池化。它的预算随输入缩放——1 分钟视频比 5 秒视频用更多 token。LLaVA-OneVision 固定预算并缩放池化。两者都有效；它们用可配置性换可预测性。

## 使用方法

`code/main.py` 是 OneVision 风格 VLM 的课程和预算规划器。给定每样本 token 预算和目标场景混合（假设 40% 单图像、30% 多图像、30% 视频），它：

- 分配每个场景的分辨率、池化因子和帧数。
- 检查每个场景是否在共享预算内。
- 报告预期 token 数量、LLM FLOPs，以及哪些场景 token 不够。
- 打印分阶段训练计划。

用它来规划 OneVision 微调或对 VLM 部署的每请求成本进行合理性检查。

## 练习

1. 你的产品支持 80% 单图像、10% 多图像（2-4 张图像）、10% 视频（8-16 帧）。设计 token 预算。不做重型多图像时省下的额外预算放哪里？

2. 阅读 LLaVA-OneVision 第 4.3 节（新兴能力）。提出第四种课程可能解锁但论文未报告的新兴技能。

3. 交换课程顺序——先训练多图像，然后单图像，然后视频。预测哪些基准下降及原因。

4. 论文报告视频基准在每样本仅 8 帧上训练。这在推理时泛化到 30 秒视频吗？最先坏的是什么——token 预算还是时间推理？

5. 24x24 patches 的双线性池化到 12x12 是每维 4 倍缩减。用标准库 Python 实现池化，验证每个 2x2 块的均值与双线性输出匹配。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| OneVision scenario | "单图像、多图像或视频" | 统一 VLM 处理的三种输入形状之一；预算在各场景中保持恒定 |
| Token budget | "每样本多少 token" | LLM 每训练/推理样本看到的总视觉 token，通常 3000-4000 |
| Curriculum | "训练顺序" | 阶段排序（单图像 → 多图像 → 视频），为新兴迁移选择 |
| Bilinear pooling | "Token 缩减" | 对 patch 网格（2D）应用双线性插值以减少 token 数量同时保留局部性 |
| Emergent skill | "未训练但仍有效" | 在推理时出现的能力，无匹配训练数据，因课程组合而涌现 |
| AnyRes-k | "k 图块设置" | k 个固定分辨率子图块加一张缩略图，典型 k ∈ {4, 9} |
| Task transfer | "跨场景泛化" | 在单图像上学习的技能通过共享骨干网应用到视频（反之亦然） |
