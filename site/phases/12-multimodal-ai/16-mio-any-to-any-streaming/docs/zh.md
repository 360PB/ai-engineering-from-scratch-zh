# MIO 与任意到任意流式多模态模型

> GPT-4o 发布了一个大多数开源模型无法复制的产品：一个实时听见声音、看到视频并说话的 agent。2024 年底开源生态的答案是 MIO（Wang 等，2024 年 9 月）。MIO 对文本、图像、语音和音乐进行分词，在一个因果 transformer 上训练交错序列，并生成任意模态到任意模态。AnyGPT（Zhan 等，2024 年 2 月）是概念验证；MIO 是规模化版本；Unified-IO 2（Allen AI，2023 年 12 月）是带视觉 + 动作接地的表亲。本节解读任意到任意模式——四个分词器，一个 transformer，流式友好解码。

**类型：** Learn
**语言：** Python（标准库，四模态 token 分配器 + 流式解码循环）
**前置知识：** Phase 12 · 11（Chameleon），Phase 6（Speech and Audio）
**时间：** 约 120 分钟

## 学习目标

- 设计一个容纳文本、图像、语音和音乐 token 而不冲突的共享词汇表。
- 在压缩 + 重建权衡上比较 SEED-Tokenizer（图像）和 SpeechTokenizer 残差-VQ（语音）。
- 解释建立任意到任意生成的四阶段课程。
- 说出三个开源任意到任意配方及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题背景

统一多模态模型容易声称但难以规模化构建。2024 年之前大多数"任意到任意"系统是管道化的：视觉模型 → 文本表示 → 语音模型 → 音频。每跳丢失信息、增加延迟、复杂化训练。GPT-4o 的演示视频展示了单模型替代方案，响应时间低于秒级；开源系统落后数月。

工程挑战：

- 每个模态必须有分词器，压缩损失足够小以进行重建，并以 transformer 能消费的速度产生 token。
- 单一词汇表必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。至少四万以上条目。
- 训练数据必须覆盖每对输入-输出（文本→图像、图像→语音、语音→图像等），或者模型必须组合。
- 推理必须以足够快的速度流式输出 token 以实现对话延迟（<500ms 首音频字节时间）。

## 核心概念

### 四个模态的四个分词器

MIO 的分词器堆栈：

- 文本：标准 BPE，词汇表约 32000。
- 图像：SEED-Tokenizer（2023）——带离散 codebook 的量化 VAE，4096 条目，每图像 32x32 token。
- 语音：SpeechTokenizer 残差-VQ（2023）——将 16kHz 波形编码为 8 个层次 codebook；第一层是粗糙内容，后续层添加韵律和说话人身份。
- 音乐：类似的残差-VQ（Meta 的 MusicGen / Encodec 族），4-8 个 codebook。

每个模态产生整数 token。Token 在共享词汇表中获得不相交的 ID 范围：

```
text:   0..31999
image:  32000..36095  (4096 图像 token)
speech: 36096..40191  (4096 语音基础 token，加残差层)
music:  40192..48383  (8192 音乐 token)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, 等)
```

总计：约 48k 词汇表。输入嵌入和输出投影跨越全部。

### 流式解码

语音生成使用残差-VQ。Transformer 预测基础（第 0 层）语音 token；并行解码的残差量化器预测后续层。每第 0 层 token 在 16kHz 下约 50ms 音频。

流式模式：
1. 用户对麦克风说话；实时音频分词器每 50ms 发出语音 token。
2. MIO 按 token 到达消费（prompt prefill + 增量前向）。
3. 输出 token 按生成流式输出；并行语音解码器将 token 转换为音频样本，延迟约 50-150ms。
4. 首音频字节时间：MIO 论文约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音 LLM 设计。Moshi 特别是在单块 GPU 上实现 160ms 往返。

### 四阶段课程

MIO 的训练课程：

1. 第一阶段——对齐。大规模模态对语料：文本-图像、文本-语音、文本-音乐。每对使用自己的 token 词汇段。训练共享词汇表。
2. 第二阶段——交错。多模态交错文档（带图像+视频的博客、带转录的播客等）。训练跨模态上下文。
3. 第三阶段——语音增强。额外音频数据提升语音质量而不丢失文本能力。
4. 第四阶段——SFT。跨模态指令微调：VQA、标题生成、旁白、语音到语音对话。

跳过阶段会降级特定能力：跳过第二阶段模型失去跨模态上下文；跳过第三阶段语音质量差。

### 视觉思维链

MIO 引入视觉思维链：模型将中间图像 token 作为推理步骤发出。对于"猫在爬树吗？"模型：

1. 发出 `<image>` token 渲染场景（从输入图像或草图）。
2. 发出文本分析草图。
3. 发出最终答案。

渲染的中间图像用作草稿本。在空间推理任务上基准改进。该想法与文本推理的思维链对应。

### 任意到任意的竞争者

- AnyGPT（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），类似设计。
- Unified-IO 2（arXiv:2312.17172）：添加视觉动作输出、深度、法线。更多任务多样性，规模更小。
- NExT-GPT（arXiv:2309.05519）：LLM + 模态特定扩散解码器。不是单模型方法。
- CoDi（arXiv:2305.11846）：可组合扩散；通过共享潜变量的任意到任意。

MIO 是最接近纯 token 任意到任意的。AnyGPT 是其概念祖先。

### 延迟预算

对于对话产品，每个组件的延迟都重要：

- 麦克风到音频 token：约 50ms。
- Prefill（音频 token + 历史）：约 100ms 在 8B 模型上。
- 第一个输出 token：约 50ms。
- 并行残差-VQ + 语音解码器：约 100-150ms。

总首音频字节时间：约 300ms 最低。GPT-4o 声称约 250ms。Moshi 声称 160ms。MIO/AnyGPT 在公开基准上约 400-600ms。

### 为什么任意到任意仍然困难

即使在 2026 年，开源任意到任意模型在两个轴上仍落后于封闭模型：

- 语音质量。残差-VQ 分词器是有损的；对话语音听起来比 ElevenLabs 级语音更机器人。
- 跨模态推理。问模型"唱你看到的"仍然比纯视觉任务失败更多。

这些是开放研究问题。Qwen3-Omni（第 12.20 节）是 2025 年最先进的开源尝试。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟目标是什么？列出花费时间的组件。

2. SpeechTokenizer 残差-VQ 使用 8 个 codebook。为什么并行解码残差层是必要的（vs 顺序），节省多少延迟？

3. 你的词汇表有 32k 文本 + 4k 图像 + 4k 语音。加 8k 音乐和约 10 个分隔符。在隐藏 dim 4096 下的 embedding 矩阵参数量是多少？

4. 视觉思维链发出中间图像。什么问题从中受益？什么问题因额外 token 而受损？

5. 阅读 Moshi（arXiv:2410.00037）。描述其"内心独白"技巧并与 MIO 的视觉思维链比较。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| Any-to-any | "多模态输入/输出" | 单一模型接受和发出文本、图像、语音和音乐，任意方向 |
| Residual-VQ | "语音分词器堆栈" | 多 codebook 分词，每层添加信息；基础层是内容，后续层是韵律 |
| SEED-Tokenizer | "图像编码" | 带 4096 条目 codebook 的离散图像分词器，MIO 使用 |
| Chain-of-visual-thought | "视觉草稿本" | 模型在最终答案前生成中间图像作为推理步骤 |
| Time-to-first-audio-byte | "TTFAB" | 从用户语音到第一个音频输出的延迟；<500ms 有对话感 |
| Four-stage curriculum | "训练配方" | 对齐 → 交错 → 语音增强 → SFT，按此顺序 |
