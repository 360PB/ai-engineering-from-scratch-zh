# 原生稀疏注意力（DeepSeek NSA）

> 在 64k Token，注意力吞噬 70-80% 解码延迟。每个开源模型实验室都有计划修复它。DeepSeek 的 NSA（ACL 2025 最佳论文）是坚持下来的那个：三个并行注意力分支——压缩粗粒度 Token、选择性保留细粒度 Token、以及用于局部上下文的滑动窗口——通过学习门组合。它与硬件对齐（内核友好）、原生可训练（在预训练中工作，而非在推理时螺栓固定），在 64k 解码上运行比 FlashAttention 快同时匹配或击败完整注意力质量。这节课端到端构建三个分支并展示为什么稀疏性是端到端可微分的。

**类型：** 构建
**语言：** Python（stdlib）
**先修内容：** Phase 7 · 12（KV 缓存、FlashAttention），Phase 7 · 15（注意力变体），Phase 10 · 16（差分注意力）
**学习时间：** 约 60 分钟

## 学习目标

- 陈述 NSA 的三个注意力分支以及每个捕捉什么。
- 解释为什么 NSA 是"原生可训练的"而先前稀疏注意力方法仅在推理时。
- 计算 NSA 相对完整注意力在 64k 上下文的注意力计算节省，作为压缩块大小和选择 top-k 的函数。
- 在短合成序列上用 stdlib Python 实现三分支组合并验证门权重行为。

## 问题所在

完整注意力在序列长度 N 下花费 `O(N^2)` 时间和每层 `O(N)` KV 缓存。在 64k Token，计算和内存带宽数字是灾难性的。NSA 论文的测量估计：在 64k 注意力占总解码延迟 70-80%。一切下游——TTFT、Token/秒、每百万 Token 成本——由注意力成本主导。

稀疏注意力是明显答案。先前尝试落入两类。固定模式稀疏（滑动窗口、跨步、块局部）抛弃信息，在长程召回任务上失败。推理时稀疏（KV 缓存剪枝、H2O、StreamingLLM）应用于在密集注意力上预训练的模型，只能恢复加速潜力的一小部分，因为模型从未被要求通过稀疏模式路由信息。

原生稀疏注意力（Yuan et al., DeepSeek + PKU + UW, ACL 2025 最佳论文，arXiv:2502.11089）两者都做：模型在预训练期间学习的稀疏模式，实现为内核对齐算法，在推理时实际提供计算节省。两年后，NSA 或直接后代是每个前沿长上下文模型的默认注意力。

## 核心概念

### 三个并行分支

对于每个查询，NSA 运行三次注意力，针对 KV 缓存的三个不同视图：

1. **压缩分支。** Token 按大小 `l`（通常 32 或 64）分组为块。每个块通过小型学习 MLP 压缩为单个摘要 Token。查询在压缩 Token 上运行注意力，获得整个序列的粗粒度视图。

2. **选择分支。** 使用压缩分支的注意力分数，识别与当前查询最相关的 top-k 块。从那些块加载细粒度（未压缩）Token，查询在其上运行注意力。可以将压缩分支注意力视为选择的路由信号。

3. **滑动窗口分支。** 查询关注最近 `W` 个 Token（通常 512）用于局部上下文。此分支捕捉其他两个可能错过的结构密集短程模式（句法、局部共指）。

三个分支输出通过每位置学习门组合：

```
out = g_cmp * out_cmp + g_sel * out_sel + g_win * out_win
```

`g_cmp, g_sel, g_win` 是来自查询上小型 MLP 的门权重。它们不必和为 1——可以独立加权分支。

### 为什么这是"原生可训练的"

选择步骤（top-k 块）是离散的。离散操作打破梯度流。先前稀疏注意力工作要么跳过选择上的反向传播（限制训练），要么使用连续松弛，在推理时不给出真实稀疏性。

NSA 回避了这个问题：压缩分支注意力是整个序列上可微分的粗粒度注意力。Top-k 操作重用压缩分支的 top 注意力分数来选择哪些细粒度块加载。梯度通过压缩分支分数流动（影响压缩输出和选择逻辑两者），选定块对最终输出的贡献也是可微分的。不可微分的 `top_k` 操作在前向计算图上是无操作——它只控制哪些块从内存加载。

这就是 NSA 可用于端到端预训练的原因。模型学会通过三分支联合路由信息，产生在推理时实际提供承诺加速的稀疏模式。

### 硬件对齐内核

NSA 的内核为现代 GPU 内存层次设计。内核按 GQA 组（外循环）加载查询，按组获取相应稀疏 KV 块（内循环），在 SRAM 上运行注意力。因为每个查询组看到相同的选定块（选择是每查询组，非每查询头），KV 加载在组间摊销。算术强度保持高位。

论文报告 Triton 内核在 64k 解码上比 FlashAttention 快 9 倍，加速比随序列长度增长。前向和反向内核都提供。

### 计算预算

设 N 为序列长度，l 为压缩块大小，k 为 top-k 选择数，w 为滑动窗口，b 为选定块大小（通常等于 l）。

- 压缩分支：每查询 `O(N/l)` 键，总计 `O(N * N / l)`。
- 选择分支：每查询 `O(k * b)` 键，总计 `O(N * k * b)`。
- 滑动分支：每查询 `O(w)` 键，总计 `O(N * w)`。

总计：`O(N * (N/l + k*b + w))`。

使用 `N = 64k, l = 64, k = 16, b = 64, w = 512`：每查询成本为 `1000 + 1024 + 512 = 2536` 键。完整注意力是 `64000` 键。25 倍计算减少。

使用 `N = 128k, l = 64, k = 16, b = 64, w = 512`：每查询成本为 `2000 + 1024 + 512 = 3536` 键。完整注意力是 `128000` 键。36 倍减少。收益随序列长度增长，这就是全部意义。

### 比较

| 方法 | 可微分 | 真实推理加速 | 长程召回 |
|--------|---------------|----------------------|-------------------|
| 仅滑动窗口 | 是 | 是 | 失败 |
| 跨步 / 块稀疏 | 是 | 是 | 部分 |
| KV 剪枝（H2O, StreamingLLM）| N/A（推理时） | 是 | 部分 |
| MoBA（Monkshot）| 部分 | 是 | 好 |
| NSA | 是（原生）| 是（64k 9 倍）| 匹配完整注意力 |

MoBA（Monkshot，arXiv:2502.13189）同期发表，采用类似三优于一方法，将 MoE 原理应用于注意力块。NSA 和 MoBA 是 2026 年长上下文预训练要知道的两个架构。

## 构建

`code/main.py` 在短合成序列上实现三个分支并展示：

- 压缩 MLP（为教学清晰使用简单均值池基线；真实 NSA 使用学习 MLP）。
- 由压缩分支分数驱动的 top-k 块选择。
- 最近 `w` Token 上的滑动窗口注意力。
- 门控组合。
- 比较完整注意力的计算计数打印。

### 步骤 1：将 Token 压缩为块

```python
def compress(K, l):
    n = len(K)
    n_blocks = (n + l - 1) // l
    out = []
    for b in range(n_blocks):
        start, end = b * l, min((b + 1) * l, n)
        block = K[start:end]
        summary = [sum(row[d] for row in block) / len(block) for d in range(len(K[0]))]
        out.append(summary)
    return out
```

### 步骤 2：压缩分支注意力

对压缩键运行查询的 softmax 注意力。压缩分支分数兼作 top-k 选择信号。

### 步骤 3：top-k 块选择

选取 k 个最高分压缩块的索引。从那些块加载原始未压缩 Token 并在其上运行注意力。

### 步骤 4：滑动窗口注意力

取最后 `w` Token 并在其上运行标准注意力。

### 步骤 5：门 + 组合

查询上小型 MLP 产生三个门权重。三分支输出的加权和是最终输出。

### 步骤 6：计算计数

打印每个分支每查询的键数及总计。与 N（完整注意力）比较。在 1024 Token 合成上 `l = 32, k = 4, w = 128`，NSA 每查询看到 `32 + 128 + 128 = 288` 键对完整注意力 1024——3.5 倍更少。

## 使用

NSA 正在 DeepSeek 自己的长上下文预训练流水线中发版。截至 2026 年 4 月公共推理栈中的集成状态：

- **DeepSeek 内部**：原生，已发布权重使用 NSA 或其继任者 DSA（Deepseek Sparse Attention）。
- **vLLM**：为 DeepSeek-V3.x 权重开发中的实验性 NSA 支持。
- **SGLang**：NSA 基准发布；生产路径跟随 vLLM。
- **llama.cpp / CPU**：不支持；内核分解开销在 CPU 吞吐量上不值得。

何时伸手 NSA：

- 瞄准 64k-plus 上下文的预训练或继续训练运行，有严肃计算预算。
- DeepSeek 自己的长上下文检查点推理。权重是 NSA 原生的。

何时不：

- 服务已稠密注意力预训练模型。你无法在不继续训练的情况下改装 NSA。
- 16k 以下上下文。三分支开销主导节省。
- 批处理-1 交互式聊天。延迟敏感 decode 有益，但仅在长上下文。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|----------------|------------------------|
| 压缩分支 | "粗粒度视图 | 在块平均键上注意力，每查询 O(N/l) 键提供全局上下文 |
| 选择分支 | "Top-k 块 | 在最高压缩分支分数的 k 个块上的细粒度注意力 |
| 滑动窗口 | "局部上下文 | 在最后 W 个 Token 上的注意力，用于短程模式 |
