# 多令牌预测（MTP）

> 从 GPT-2 到 Llama 3，每位自回归 LLM 在每个位置训练一个损失：预测下一个令牌。DeepSeek-V3 在每个位置增加了第二个损失：预测下下个令牌。额外 14B 参数（在 671B 模型上）通过梯度流被蒸馏回主模型，训练好的 MTP 头在推理时被改造成推测解码的草稿生成器，接受率超过 80%。1.8 倍的生成吞吐就这样免费获得了。本课从 DeepSeek 技术报告中构建顺序 MTP 模块，计算损失和共享头参数布局，并解释为什么 MTP 保留了因果链，而 Gloeckle 等人最初的并行 MTP 打破了它。

**类型：** Build
**语言：** Python（标准库）
**前置要求：** Phase 10 · 04（预训练 mini GPT）、Phase 10 · 15（推测解码）
**时长：** 约 60 分钟

## 学习目标

- 阐述 MTP 训练目标，导出预测深度上的联合损失。
- 解释 Gloeckle 等人的并行 MTP 头（2024）与 DeepSeek-V3 的顺序 MTP 模块之间的区别，以及为什么顺序设计保留了因果链。
- 计算向预训练运行添加 MTP 模块的参数和内存开销。
- 从零实现一个 MTP 模块：共享嵌入、逐深度 Transformer 块、投影层和共享输出头。

## 问题背景

下一个令牌预测是标准 LLM 训练目标。每个隐藏状态被监督去预测恰好一件事：紧跟其后的令牌。这是一个令人惊讶的弱信号。序列中大部分信息跨越多个令牌——结构、连贯性、事实性、算术流程。模型需要通过在数万亿个令牌上积累许多单令牌信号来学习这些。

MTP 问：如果每个隐藏状态同时被监督去预测多个未来令牌呢？Gloeckle 等人（Meta，2024）证明这有帮助。他们的实现将几个独立输出头放在骨干网上方，每个头预测不同的偏移量。并行、简单，但各头看到相同的隐藏状态，没有层级精化——而且预测不以因果方式链式传递，因此不能用于推测解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为顺序模块，在每个预测深度保留因果链。模型从 `h_i^(0)` 预测 `t+1`，然后从一个结合了 `h_i^(0)` 和 `E(t+1)` 嵌入的新隐藏状态 `h_i^(1)` 预测 `t+2`，以此类推。每个深度都有自己的小型 Transformer 块。共享嵌入和共享输出头将参数开销保持在可控范围内。在 DeepSeek-V3 的规模上，MTP 模块上额外的 14B 参数加上 671B 主模型权重。这 2% 的开销换来了更密集的训练信号和推理时现成的推测解码草稿。

本课从零构建单个 MTP 模块和 D 深度损失。数学简洁。实现约 150 行。

## 核心概念

### 顺序 MTP 方法

DeepSeek-V3 在主模型之上添加 D 个 MTP 模块。每个模块 `k`（`k = 1..D`）预测深度 `k` 处的令牌——即给定通过位置 `i` 的前缀，预测 `t_{i+k}`。

模块 `k` 由以下组成：

- 一个带独立注意力和 MLP 的 Transformer 块 `T_k`。
- 一个投影矩阵 `M_k`，将上一深度的隐藏状态与下一深度真实令牌的嵌入相结合。
- 共享嵌入 `E`（与主模型相同）。
- 共享输出头 `Out`（与主模型相同）。

训练时，对于通过位置 `i` 的前缀，逐深度隐藏状态为：

```
h_i^(0) = 主模型骨干在位置 i
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

逐深度预测：

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

逐深度损失是针对真实令牌 `t_{i+k}` 的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个小的加权因子——DeepSeek-V3 在训练前 10% 使用 0.3，之后使用 0.1。总训练损失为 `L_main + L_MTP`。

### 为什么是顺序而非并行

Gloeckle 最初的并行 MTP 有 D 个输出头，每个头直接作用于 `h_i^(0)`。每个头从相同的骨干隐藏状态预测 `t_{i+k}`。这可以训练，但预测相互不依赖。你不能利用 `head_1` 的输出来帮助 `head_2`——各头并行触发。

DeepSeek-V3 的顺序设计从 `h_i^(k-1)` 加上实际下一令牌嵌入 `E(t_{i+k})` 来构建 `h_i^(k)`。这保留了因果链：要预测 `t_{i+k+1}`，深度 `k+1` 处的模块看到了 `t_{i+k}` 的内容。这在结构上与自回归解码器消费自身输出的方式完全相同——使 MTP 模块直接可用作推测解码草稿生成器。

推理时：将 `h_i^(k-1)` 和草稿 `t_{i+k}` 喂入模块 `k+1`，得到 `t_{i+k+1}` 的预测。重复。这就是 EAGLE 风格的草稿，使用训练好的 MTP 模块作为草稿网络。DeepSeek-V3 报告第一个 MTP 模块接受率超过 80%，约 1.8 倍加速。

### 参数计算

对于隐藏层为 `h`、词表为 `V` 的模型：

- 主模型：数十亿参数，加一个大小为 `V * h` 的输出头。
- 共享输出头：复用主模型的头。无额外参数。
- 共享嵌入：复用主模型的嵌入表。无额外参数。
- 每个 MTP 模块：
  - 投影 `M_k`：`2h * h = 2h^2`。
  - Transformer 块 `T_k`：注意力（ MHA 约 `4h^2`）加 MLP（SwiGLU 比率 8/3 时通常约 `8h^2`）。每块约 `12h^2`。

每模块额外总参数：约 `14h^2`。DeepSeek-V3 的 `h = 7168`，D=1 模块：约 `14 * 7168^2 ≈ 720M` 参数。DeepSeek-V3 报告 14B——差距主要来自 MTP 模块中的专家层同样是 MoE 结构，使每模块参数计数膨胀。

### 推测解码的收益

预训练期间，MTP 模块使训练速度降低约 10%（更多前向计算，额外损失）。收益是双重的：

1. **更密集的训练信号。** 每个隐藏状态看到 D+1 个监督目标。DeepSeek-V3 消融实验在 MMLU、GSM8K、MATH、HumanEval 上测量的效果：一致地提升几个百分点。

2. **免费的推测解码草稿。** MTP 模块已被训练来预测接下来的几个令牌。作为草稿网络重新利用，接受率超过 80%。在此水平上，N=3 或 N=5 的推测解码提供 1.8 倍吞吐。训练时 10% 的成本在第一次运行推理时就收回了。

### 与 EAGLE 的关系

EAGLE 在预训练后**单独训练**一个小草稿模型。MTP 将草稿 baked 进预训练。两种方法通过不同流程汇聚到相似的接受率：

| 维度 | EAGLE-3 | MTP（DeepSeek-V3） |
|------|---------|-------------------|
| 训练时机 | 预训练后 | 预训练期间 |
| 与现有权重向后兼容 | 是 | 否（需要重新训练） |
| 草稿参数 | 1-2 层 Transformer | 1 个 Transformer 块 + 投影 |
| 接受率 | 0.88-0.92 | 深度 1 处 0.80+ |
| 超越加速的收益 | 仅推测解码 | 更密集训练信号 + 加速 |

## 构建它

`code/main.py` 从头构建一个 MTP 模块：共享嵌入、投影、Transformer 块、共享输出头。然后在短合成序列上计算逐深度交叉熵损失，并打印各组件的参数计数。32 个令牌的玩具词表使数字易于阅读。

### 第 1 步：共享嵌入表

单个 `vocab_size x hidden` 表被主模型和每个 MTP 模块在每个深度共同使用。不是第二份拷贝——就是同一个张量。

### 第 2 步：逐深度组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 沿特征维度拼接，然后投影到 hidden
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # 向量加法替代
    projected = matvec(M_k, concat)
    return projected
```

真正的 DeepSeek-V3 将两个 RMSNorm 后的向量拼接为 `[2h]`，然后用 `h x 2h` 矩阵投影。玩具版为保持标准库简洁使用向量加法。

### 第 3 步：深度 k 处的 Transformer 块

自注意力加 MLP。玩具版用单层线性注意力块和 SwiGLU MLP 保持结构可见，无需 numpy。

### 第 4 步：共享输出头

复用主模型的输出投影。输出词表上的 logits。

### 第 5 步：逐深度损失

softmax(logits) 对深度偏移 `k` 处真实令牌的交叉熵。用 `lambda / D` 缩放因子跨深度聚合。

### 第 6 步：参数核算

打印总参数计数、共享部分（嵌入、头）计数和每模块额外计数。展示 MTP 额外部分与主模型大小的比率。

## 使用它

MTP 已集成到 DeepSeek-V3（2024 年 12 月）和 DeepSeek-R1 系列中。推理时：

- DeepSeek 自有的服务栈将 MTP 模块作为推测解码器开箱即用。
- vLLM 和 SGLang 在 2026 年 4 月已有 DeepSeek-V3 MTP 集成路径。
- AMD 的 ROCm SGLang 教程展示了特定 MTP 推测解码配置，在 V3 检查点上测得 1.8 倍加速。

在新预训练运行中使用 MTP 的时机：

- 你控制完整预训练管道，想获取更密集的训练信号。
- 你知道将大规模服务模型，想要免费的推测解码。
- 你的隐藏大小至少为 4096。在 10 亿规模下，开销的伤害大于收益。

不适合的场景：

- 微调现有预训练密集模型。MTP 模块未经训练。
- 想与干净基线对比的研究模型。MTP 改变了架构。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| MTP 模块 | "额外损失块" | 一个小型 Transformer 块加投影，预测主模型前 `k` 个位置的令牌 |
| 预测深度 | "哪个偏移" | 使得模块 `k` 从通过位置 `i` 的前缀预测 `t_{i+k}` 的整数 `k` |
| 并行 MTP | "Gloeckle 风格" | D 个独立头作用于同一骨干隐藏状态，无条件链 |
| 顺序 MTP | "DeepSeek-V3 风格" | 每个模块以上一深度隐藏状态加下一令牌嵌入为条件；保留因果链 |
| 共享输出头 | "复用主头" | MTP 模块调用主模型的 LM 头，而非单独的输出投影 |
| 共享嵌入 | "复用主表" | 同一词表嵌入表处处使用；无重复参数 |
| 投影矩阵 M_k | "结合隐藏 + 下一令牌" | 将上一隐藏状态和目标令牌嵌入折叠为下一深度输入的 `h x 2h` 线性层 |
| 联合损失 L_MTP | "平均额外损失" | 逐深度交叉熵损失的算术平均，由 `lambda` 缩放 |
| 深度 1 接受率 | "MTP 草稿有多准" | D=1 MTP 模块的 top-1 预测等于主模型 top-1 预测的比率；DeepSeek-V3 上 80%+ |
| Lambda 加权 | "额外损失重要性" | 逐深度缩放因子；DeepSeek-V3 训练初期 0.3，后期 0.1 |
