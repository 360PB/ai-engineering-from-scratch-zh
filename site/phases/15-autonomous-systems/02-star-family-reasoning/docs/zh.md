# STaR、V-STaR、Quiet-STaR — 自学推理

> 最小的自改进循环存在于推理过程中。模型生成一条思维链，保留最终得出正确答案的那些，然后在其上微调。这就是 STaR。V-STaR 添加了一个验证器，使推理时选择更好。Quiet-STaR 将推理推广到每个 token。三个都有效。都不是魔法——循环会保留任何偶然到达正确答案的捷径。

**类型：** 学习
**语言：** Python（标准库，自举循环模拟器）
**前置知识：** Phase 13 · 01-03（推理与 CoT），Phase 15 · 01（长时程框架）
**时间：** 约 60 分钟

## 问题

教模型推理的直接方法是收集人类书写的推理轨迹。这很昂贵、很慢，而且受限于人类愿意编写的高质量思维链的数量。

STaR（Self-Taught Reasoner，自学推理者，Zelikman 等，2022 年）提出问题：如果模型自己编写推理并对照已知答案进行评分呢？循环是：

1. 采样一条推理轨迹加上答案。
2. 如果最终答案正确，保留该轨迹。
3. 在保留的轨迹上微调。
4. 重复。

它有效。GSM8K 和 CommonsenseQA 都在没有新的人类注释的情况下得到了改进。但循环有一个内置的偏差：任何产生正确答案的推理轨迹都被保留，无论推理本身是否合理。V-STaR（Hosseini 等，2024 年）用一个学习的验证器修补了这一点；Quiet-STaR（Zelikman 等，2024 年）将这个想法推广到每个 token 的内部推理。

## 概念

### STaR：在有效的样本上自举

从一个具有一定弱推理能力的基础模型开始。在每个训练问题上，采样一条推理轨迹加上答案。如果答案与标签匹配，保留该（问题，推理，答案）三元组。在保留集上微调模型。重复。

一个细节很关键。如果模型永远无法正确完成一个问题，循环就无法学习它。STaR 添加了**合理化（rationalization）**：对于模型失败的问题，将正确答案作为提示注入，重新提示模型产生一个能导出该答案的推理。合理化的推理被添加到训练集中。

原始论文（Zelikman 等，2022 年）的结果：一个 GPT-J 基础模型通过重复 STaR 轮次加上合理化，在 GSM8K 上从 5.8% 提高到 10.7%——大约 5 个百分点的绝对提升。在 CommonsenseQA 上，经过 STaR 训练的 GPT-J 6B 达到 72.5%，与用人工注释的 rationales 训练的 GPT-3 175B（~73%）相当——后者大了约 30 倍。

### V-STaR：用 DPO 训练验证器

STaR 丢弃了不正确的推理。Hosseini 等（2024 年）观察到这些也是有价值的数据：每个（推理，"这正确吗"）配对都可以训练一个验证器。他们在正确和错误的解决方案上使用直接偏好优化（DPO）来构建一个排序器。在推理时，采样 N 条推理，选择验证器排名最高的。

报告的提升：在 GSM8K 和 MATH 上比之前的自改进基线高 4 到 17 个百分点，大部分收益来自使用验证器进行推理时选择，而不是用于额外的生成器微调。

### Quiet-STaR：每个 token 的内部推理

Zelikman 等（2024 年）提出问题：如果模型在每个 token 位置学习生成一个简短的内部推理，而不仅仅是在问题和答案之间呢？Quiet-STaR 训练一个模型在每个预测 token 之前发出一个隐藏的"思考"，然后通过一个学习的权重将思考感知的预测与基线预测混合。

结果：Mistral 7B 在 GSM8K 上实现了绝对零样本提升，从 5.9% 到 10.9%，在 CommonsenseQA 上从 36.3% 到 47.2%，无需任务特定的微调。模型学会了"何时思考"——难的 token 有更长的内部推理；简单的 token 几乎没有。

### 为什么三者共享一个安全问题

三种方法都使用最终答案作为梯度信号。一个通过有缺陷的推理——利用捷径、猜测或使用非泛化的模式——达到正确答案的推理，会得到正向强化。在分布内问题上，捷径有效。在分布外问题上，它会静默失效。

V-STaR 的验证器通过学习对推理进行排序来缓解，但验证器在相同的标签集上训练。它可能学会偏好格式工整的错误推理，而不是诚实的未知。更安全的设计是将 STaR 风格的数据与（a）过程监督奖励模型（奖励中间步骤而不仅仅是答案）和（b）留出 OOD 评估结合起来，这会打破简单的捷径。

### 对比

| 方法 | 训练信号 | 推理成本 | 数据浪费 | 已知失败模式 |
|---|---|---|---|---|
| STaR | 如果正确则保留（推理，答案） | 1x | 丢弃所有不正确的推理 | 捷径推理 |
| STaR + 合理化 | 上述 + 正确答案提示的重试 | 1x | 较少 | 合理化的推理可能不可信 |
| V-STaR | STaR + DPO 验证器（两类） | Nx（best-of-N） | 最小 | 验证器可能强化自信的错误 |
| Quiet-STaR | 每个 token 推理 + 混合权重 | 1.5-3x | 最小 | 仍然是答案条件梯度 |

### 这在 2026 年技术栈中的位置

STaR 已经不新鲜了。但这一模式在 2025-2026 年无处不在。在可验证的数学问题上进行强化学习（DeepSeek-R1、Kimi-k1.5、o1）是 STaR 的答案条件梯度信号，在前沿训练规模上扩展。过程奖励模型（Lightman 等，2023 年；OpenAI 的"Let's verify step by step"）是过程监督的替代方案。AlphaEvolve（Lesson 3）是用于代码的 STaR，用程序评估器代替标签。Darwin Godel Machine（Lesson 4）是用于智能体脚手架本身的 STaR。

理解 STaR 使所有这些都变得清晰。它是最小可行的自改进循环。

## 使用

`code/main.py` 在一个玩具算术任务上运行模拟的 STaR 循环。你可以观察：

- 准确率如何在自举轮次中攀升。
- 捷径如何混入：模拟器包含一个"懒惰"推理类，40% 的时间得到正确答案但泛化很差。观察 STaR 是否保留它们。
- 一个验证器（V-STaR 风格）如何在推理时提供帮助，但无法完全剪除训练期间引入的捷径。

## 交付

`outputs/skill-star-loop-reviewer.md` 帮助你在训练之前审查一个拟议的自学推理管道。

## 练习

1. 运行模拟器。将捷径频率设为零，然后设为 0.4。即使两者在训练分布上都达到 >90%，最终准确率有多大差异？
2. 在模拟器中添加一个留出 OOD 测试。从不同分布中抽取问题，并在分布内和 OOD 集上评估自举后的模型。量化差距。
3. 阅读 Quiet-STaR 论文（arXiv:2403.09629）第 3 节。用三句话分别解释"end-of-thought" token 和混合权重头。
4. 将 STaR 的"正确则保留"过滤器与一个过程监督替代方案进行比较，该方案独立奖励每个推理步骤。找出标签成本差异和可能的质量差异。
5. 设计一个评估，可以在部署的模型中捕获捷径推理。它不需要完美——只需要打破 STaR 循环会强化的最简单的捷径。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|---|---|---|
| STaR | "自学推理者" | 在生成正确答案的模型生成推理上微调；重复 |
| 合理化（Rationalization） | "提示重试" | 在基础模型失败的问题上注入正确答案并重新提示生成推理 |
| V-STaR | "验证器 STaR" | 在正确和错误的推理上用 DPO 训练验证器，用于推理时选择 |
| Quiet-STaR | "每个 token 的推理" | 在每个 token 位置生成隐藏思考；与基线预测混合 |
| 答案条件梯度（Answer-conditioned gradient） | "基于结果信号" | 训练循环奖励最终答案，而不是推理步骤 |
| 过程奖励模型（Process reward model） | "步骤级验证器" | 在每步正确性上训练的奖励模型，而不是结果——与 STaR 对比 |
| 捷径推理（Shortcut rationale） | "正确答案，错误推理" | 通过非泛化模式到达标签的推理；STaR 会保留这些 |
