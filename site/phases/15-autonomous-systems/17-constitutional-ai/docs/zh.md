# 宪政AI与规则覆盖

> Anthropic 于2026年1月22日发布的 Claude 宪章长达79页，采用 CC0 协议。它从基于规则的路线转向基于推理的对齐，建立了四层级优先级体系：(1) 安全与支持人类监督，(2) 伦理，(3) Anthropic 准则，(4) 乐于助人。行为分为硬编码禁止（生物武器提升、CSAM），操作者和用户均无法覆盖；以及软编码默认值，操作者可在定义范围内调整。2022年原始版本（Bai 等）通过自我批评和 RLAIF 在宪章上进行有害性训练。坦诚的说明：基于推理的对齐依赖于模型将原则泛化到未预见情境的能力。Anthropic 自己的2023年参与实验表明公众来源与企业原则之间约有50%的分歧；2026版本未纳入这些发现。

**类型：** 学习
**语言：** Python（标准库，四层级优先级解析器）
**前置条件：** Phase 15 · 06（自动化对齐研究），Phase 15 · 10（权限模式）
**时长：** 约60分钟

## 问题

已部署的智能体遭遇的输入是其设计者从未见过的。没有规则列表能足够长以覆盖它们。没有规则列表能足够短以在计算压力下快速应用。实际问题：如何让智能体对齐那些在长尾案例和快速推理中都能存活的原则？

基于规则的对齐（RBA）：列出所有禁止项。检查快速、审计容易、无法保持最新、对未预见的近似情况往往过度拒绝。基于推理的对齐（2026版 Claude 宪章）：编码原则，让模型推理。可扩展到未见案例，审计更难，失败模式是原则误用而非漏掉规则。

2026版宪章采取明确的中间立场。硬编码禁止——其错误性不依赖上下文的事项（生物武器提升、CSAM）——采用 RBA：无论操作者或用户指令如何，一律禁止。其他一切在四层级体系内基于推理：安全与支持人类监督优先；伦理其次；Anthropic 声明的准则第三；乐于助人最后。操作者可在软编码区域内调整默认值，但无法触及硬编码禁止项。

## 概念

### 四层级优先级体系

1. **安全与支持人类监督。** 最高优先级。模型优先不削弱人类和 Anthropic 监督与纠正AI的能力。这不是"保持谨慎"；而是具体地"不要以使人类监督更困难的方式行事。"
2. **伦理。** 诚实、不伤害他人、不欺骗、不操纵。当与 Anthropic 准则冲突时优先于后者。
3. **Anthropic 准则。** Anthropic 决定重要的操作规范：产品范围、交互模式、何时使用哪些工具。
4. **乐于助人。** 最低优先级。在更高优先级范围内尽可能有用。

层级冲突时，上级优先。这与 Unix 优先级或网络 QoS 形状相同——这种框架旨在产生可预测的解析，而非在任一单一维度上产生最佳情况。

### 硬编码禁止 vs 软编码默认值

**硬编码：**
- 生物武器 / CBRN 提升
- CSAM
- 对关键基础设施的攻击
- 被直接询问时欺骗用户关于模型身份

操作者无法覆盖这些。用户无法覆盖这些。它们尽可能在模型权重层面（RLHF / 宪政AI训练）强制执行，在做不到时在推理层强制执行。

**软编码默认值（操作者，可调）：**
- 回复长度默认值
- 主题范围（模型可拒绝操作者部署范围之外的主题）
- 风格（正式 vs 随意）
- 工具使用模式

操作者调整发生在声明的边界内。操作者不能通过重命名来移除硬编码禁止项。

### 2022年 CAI 训练

原始宪政AI（Bai 等，2022）对有害性进行了训练：

1. 针对一组提示生成回复。
2. 让模型对照宪章（明确原则）批评每个回复。
3. 根据批评修改回复。
4. 在修订后的配对上进行 RLAIF（从AI反馈中进行强化学习）。

结果：模型以有原则的解释拒绝有害请求，而非一刀切的拒绝。2026版宪章使用这一训练的衍生版本，加上对明确层级体系的额外后训练。

### 基于推理的对齐能捕获什么与漏掉什么

**能捕获：**
- 未预见的允许原语组合，但原则适用明确。
- 与禁止项接近的类比的新请求。
- 依赖"你没有说X被禁止"的社会工程攻击。

**会漏掉：**
- 利用原则歧义的攻击（"用户要求了这个所以乐于助人说可以"）。
- 两个原则以未预见方式冲突且层级顺序模糊的场景。
- 跨训练周期的原则解读缓慢漂移（重新解读）。

### 2023年参与实验

Anthropic 进行了2023年实验，将企业编写的宪章与通过公众输入生成的宪章（约1000名美国受访者）进行比较。两个版本在约50%的原则上达成一致。在分歧之处，公众来源版本在某些问题上更严格（政治内容处理），在另一些问题上更宽松（AI身份自我披露）。2026版宪章未纳入公众来源的发现。这是该方法中一个已记录的张力。

### 为什么硬编码禁止是必要的

仅靠基于推理的对齐无法封闭尾部。攻击者若能让模型接受一个前提（例如"我们是一间持牌生物武器研究实验室"），往往能绕过依赖案例推理的原则。硬编码禁止不会因前提框架而弯曲。它们是第14课"硬宪法限制"在对齐层的体现。

### 宪章在技术栈中的位置

宪章不是第14课的 kill switch。它处于模型层：模型的权重被训练为何种偏好。Kill switch 和金丝雀令牌处于运行时层：运行时允许什么。两者都需要。一个因模型权重宽松而触发所有错误动作的运行时是运行时问题。一个因运行时过于限制而拒绝所有正确动作的模型是运行时问题。各层覆盖不同类别。

## 用现成库

`code/main.py` 实现了一个最小化的四层级优先级解析器。解析器接收一个建议的动作和一组原则评估（安全、伦理、准则、乐于助人），并返回动作、拒绝或修改后的动作。驱动运行一个小案例集：明确允许、明确禁止、硬编码禁止、跨层级模糊案例。

## 产出

`outputs/skill-constitution-review.md` 审计部署的宪章层：什么是硬编码的、什么是软编码的、操作者可以在哪里调整，以及四层级优先级体系是否实际上是解析顺序。

## 练习

1. 运行 `code/main.py`。确认硬编码禁止项在乐于助人评分很高时仍然触发。将解析器修改为将乐于助人权重置于伦理之上；观察失败模式。

2. 阅读 Claude 宪章（公开，79页，CC0）。识别一条你认为描述不足的原则。写两段解释具体歧义并提出更严格的表述。

3. 为客服智能体设计一套软编码默认值。操作者调整什么？操作者不能碰什么？为每个边界提供理由。

4. 阅读 Bai 等2022年 CAI 论文。描述一个宪政AI的批评-修改循环会产生比一刀切规则更差结果的案例。识别其类别。

5. Anthropic 的2023年参与实验发现公众与企业原则之间约有50%的分歧。选择一个对生产部署重要的类别（如政治中立）。提出一种设计，让操作者能表达自己的价值观，同时硬编码禁止项保持不动。

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|---|---|---|
| Constitutional AI（宪政AI） | "Anthropic的对齐方法" | 自我批评 + 相对于成文宪章的 RLAIF |
| Reason-based alignment（基于推理的对齐） | "原则，而非规则" | 模型通过推理处理未见案例的原则 |
| Hardcoded prohibition（硬编码禁止） | "永不做X" | 操作者或用户均无法覆盖的基于规则的禁止 |
| Soft-coded default（软编码默认值） | "操作者，可调" | 声明边界内的行为，由操作者控制 |
| Four-tier hierarchy（四层级体系） | "优先级顺序" | safety > ethics > guidelines > helpfulness |
| RLAIF | "AI反馈强化学习" | 奖励来自模型生成批评的强化学习 |
| Participatory constitution（参与式宪章） | "公众来源的原则" | 2023年 Anthropic 实验；与企业版本约50%分歧 |
| Principle drift（原则漂移） | "解读滑移" | 模型如何解读固定原则文本的缓慢变化 |
