# AI Scientist v2 — 研讨会级自主研究

> Sakana 的 AI Scientist v2（Yamada 等，arXiv:2504.08066）运行完整的研究循环：假设、代码、实验、图表、写稿、提交。它是第一个有生成论文通过 ICLR 2025 研讨会同行评审的系统。独立评估（Beel 等）发现 42% 的实验因编码错误而失败，文献综述经常将既定概念错误标记为新颖。Sakana 自己的文档警告代码库执行 LLM 编写的代码并推荐 Docker 隔离。这两半画面正是关键所在。

**类型：** 学习
**语言：** Python（标准库，研究循环状态机玩具）
**前置知识：** Phase 15 · 03（AlphaEvolve），Phase 15 · 04（DGM）
**时间：** 约 60 分钟

## 问题

研究是一个开放的任务。与 AlphaEvolve 的算法搜索或 DGM 的基准约束自修改不同，研究结果没有机器可检查的正确性标准。论文由评审而不是单元测试评判。这使得循环更难闭合——如果闭合则更有价值，因为研究是复合进步所在之处。

AI Scientist v1（Sakana，2024 年）通过从人类编写的模板出发来闭合循环。LLM 在固定脚手架内填写实验。AI Scientist v2（Yamada 等，2025 年）通过使用带视觉语言模型批评循环的智能树搜索来移除模板要求。系统生成想法、实施实验、产生图表、写论文，并在评审反馈上进行迭代。

同行评审结论：一份 v2 生成的论文被接受参加 ICLR 2025 研讨会（已披露）。独立评估结论：系统远非可靠。两者都是真的。

## 概念

### 架构

1. **想法生成。** LLM 在给定主题和先前文献的条件下提出研究想法。v1 使用模板；v2 使用在假设空间上的智能搜索。
2. **新颖性检查。** 文献检索步骤检查想法是否已发表。这是 Beel 等评估发现错误标记的步骤——既定方法经常被归类为新颖。
3. **实验计划。** 智能体起草实验方案并编写代码。
4. **执行。** 代码在沙箱中运行。失败反馈到重试循环。在 Beel 等的测量中，42% 的实验在此阶段因编码错误而失败。
5. **图表生成。** 视觉语言模型读取生成的图表并重写以提高清晰度。这是 v2 的关键技术创新。
6. **写稿。** LLM 起草论文，与内部评审进行迭代。
7. **可选：提交。** 论文提交到会议。

### 研讨会接受结果意味着什么

一份 v2 生成的论文通过了 ICLR 2025 研讨会的同行评审。作者向程序委员会披露了论文的来源。接受是一个数据点；它不是声称系统"做研究"的许可证。

重要背景：研讨会论文的门槛低于主会论文。同行评审有噪声；给定日期只有一小部分投稿被接受。一次成功是概念证明，不是可靠性声明。Nature 2026 年论文记录了端到端循环，本身由人类研究员合著；它不是"系统写了一篇 Nature 论文"。

### 独立评估发现了什么

Beel 等（arXiv:2502.14297）进行了外部评估。主要发现：

- **实验失败。** 42% 的实验因编码错误而失败（错误的导入、形状不匹配、未定义变量）。重试循环捕获了一些，但不是全部。
- **新颖性错误标记。** 文献检索步骤经常将既定概念标记为新颖。这是研究版的幻觉。
- **呈现质量差距。** 视觉语言图表批评产生了出版级的视觉呈现，掩盖了底层实验弱点。

最后一个发现是本 phase 最重要的。一个产生令人信服的输出但没有做令人信服的研究的系统，比一个明显失败的系统更危险，而不是更安全。评估必须触及底层声明，而不是停在图表上。

### 沙箱逃逸问题

Sakana 自己的仓库 README 警告：

> 由于本软件的性质（执行 LLM 生成的代码），我们无法保证安全。存在危险包、失控的网络访问和意外进程的spawn等风险。使用风险自负，并考虑 Docker 隔离。

这是未验证领域中自主性的操作形态。LLM 编写代码；代码运行；代码可以做进程被允许做的任何事情。如果没有硬限制文件系统、网络和进程操作的沙箱，任何自我导向的研究智能体都可以外泄数据、燃烧算力或重写自己。

AlphaEvolve 的沙箱故事更容易，因为其评估器是严格的。AI Scientist v2 的循环运行开放式的代码和开放式的目标。这就是为什么它需要更强的隔离（最低 Docker；seccomp / gVisor 更佳）和每份提交在离开系统之前的手动审查。

### v2 在前沿技术栈中的位置

| 系统 | 目标 | 输出类型 | 评估器 | 已知失败 |
|---|---|---|---|---|
| AlphaEvolve | 算法 | 代码 | 单元 + 基准 | 受评估器严格性约束 |
| DGM | 智能体脚手架 | 代码 | SWE-bench | 奖励 hacking |
| AI Scientist v2 | 研究论文 | 文本 + 代码 + 图表 | 同行评审（弱） | 实验失败、错误标记、精美掩盖弱点 |

v2 三者中自动评估器最弱，输出面最宽，公开发布Artifact的路径最短。操作控制（沙箱、审查、披露）承担了大部分安全工作。

## 使用

`code/main.py` 将 v2 循环模拟为状态机：想法 → 新颖性检查 → 实验 → 图表 → 写稿 → 审查 → 接受或迭代。每个状态都有基于 Beel 等发现的配置失败概率。运行模拟器 N 轮并计数：

- 有多少想法达到提交。
- 有多少提交会在精美论文下隐藏关键实验缺陷。
- 重试预算如何在质量和产出之间权衡。

## 交付

`outputs/skill-ai-scientist-sandbox-review.md` 是研究循环智能体产生的任何东西在离开沙箱前的两道门槛审查清单。

## 练习

1. 使用默认参数运行 `code/main.py`。多少比例的循环运行产生"干净"的论文？多少比例产生图表批评精美掩盖了实验失败缺陷的论文？
2. 默认值已经使用了 Beel 等的 42% / 25%。用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 重新运行，然后用 `--experiment-failure 0.60 --novelty-mislabel 0.40`。在两次运行之间，精美但有缺陷的论文占比如何变化？
3. 阅读 Sakana 的 AI Scientist v2 仓库 README 中关于沙箱要求的内容。说出两个你会对多天自主运行应用的额外限制（超出 Docker）。
4. 阅读 Beel 等第 4 节关于呈现质量差距的内容。设计一个额外的评估器来捕获精美外表但实验有缺陷的论文。
5. 为研究智能体输出设计一个人类审查协议，规模比"博士阅读每篇论文"更好。找出瓶颈并围绕它设计。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|---|---|---|
| AI Scientist v1 | "Sakana 的模板化研究智能体" | 在固定脚手架中填写实验 |
| AI Scientist v2 | "无模板研究智能体" | 带 VLM 图表批评的智能树搜索 |
| 智能树搜索（Agentic tree search） | "分支研究智能体" | 并行扩展多个实验计划；通过内部批评者剪枝 |
| 视觉语言批评（Vision-language critique） | "图表上的 VLM 润色" | 多模态模型读取图表并重写以提高清晰度 |
| 文献检索（Literature retrieval） | "新颖性检查" | 搜索先前工作以确认想法新颖性——记录显示会错误标记 |
| 精美掩盖（Polish masking） | "精美论文，破损研究" | 呈现质量超过实验质量；隐藏弱点 |
| 沙箱逃逸（Sandbox escape） | "LLM 代码逃出" | 智能体执行的代码做了循环设计者未预期的行为 |
