# 谈判与议价

> 智能体谈判资源、价格、任务分配和条款。2026 年基准集很明确：NegotiationArena（arXiv:2402.05863）显示 LLMs 可以通过人格操纵（"绝望"）改善收益约 20%；"测量议价能力"（arXiv:2402.15813）显示买家比卖家更难，规模无帮助——他们的 **OG-Narrator**（确定性报价生成器 + LLM 叙述者）将成交率从 26.67% 推至 88.88%；大规模自主谈判竞赛（arXiv:2503.06416）运行了约 18 万次谈判，发现**思维链隐藏**智能体通过向对手隐藏推理获胜；Bhattacharya 等 2025 年按哈佛谈判项目指标排名，Llama-3 最有效，Claude-3 最激进，GPT-4 最公平。本课实现 Contract Net Protocol（FIPA 前身，Lesson 02），连接 LLM 风格买家/卖家，运行 OG-Narrator 风格分解，并测量每个结构性选择如何改变成交率。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** Phase 16 · 02（FIPA-ACL 遗产），Phase 16 · 09（并行蜂群网络）
**时间：** 约 75 分钟

## 问题

两个智能体需要就价格达成一致。如果仅用纯语言提示放任自流，2024-2026 年 LLMs 在严格参数化的议价中以令人惊讶的低比率成交（arXiv:2402.15813 中约 27%）。规模不能修复它：GPT-4 在议价结构上并不比 GPT-3.5 更好；它在议价的*语言*上更好。

根本问题是 LLMs 混淆了两个工作——决定报价和叙述报价。OG-Narrator 分开了这些：确定性报价生成器计算数值移动；LLM 只叙述。成交率跃升至约 89%。

这镜像了一个经典多智能体发现：解耦机制与通信层获胜。Contract Net Protocol（FIPA，1996；Smith，1980）是参考任务市场机制。将 LLM 插入叙述槽，你得到一个现代 LLM 驱动的任务市场。

## 概念

### Contract Net，一段话

Smith 1980 年的 Contract Net Protocol：**管理者**广播**招标（cfp）**；**投标者**用包含其报价的**提议**消息回应；管理者选择赢家并向赢家发送**接受提议**，向输家发送**拒绝提议**。赢家执行工作。可选消息：**拒绝**（投标者拒绝提议）。FIPA 将其编为 `fipa-contract-net` 交互协议。

### 为什么 OG-Narrator 获胜

"测量语言模型的议价能力"（arXiv:2402.15813）观察到：

- LLMs 经常违反议价规则（在无意义的价位报价，忽略对方的 ZOPA）。
- 它们锚定不良（接受糟糕的首个报价；以象征性而非战略性金额反报价）。
- 规模单独不能修复这些。更大的模型用相似的战略性错误产生更可信的语言。

OG-Narrator 分解：

```
           ┌──────────────────┐        ┌──────────────────┐
  state  → │ offer generator  │ price → │  LLM narrator    │ → message
           │  (deterministic) │        │  (writes the     │
           │                  │        │   human-style    │
           └──────────────────┘        │   accompaniment) │
                                       └──────────────────┘
```

报价生成器是经典谈判策略：Rubinstein 议价模型、Zeuthen 策略，或简单的价格针锋相对。LLM 叙述。消息包含确定性价格和自然语言框架。

成交率跃升因为：
- 价格保持在议价区间内。
- 锚定是战略性的，而不是情感的。
- LLM 做它擅长的：写作。

### NegotiationArena 发现

arXiv:2402.05863 提供规范基准。主要发现：

- LLMs 可以通过采用人格（"我迫切需要在周五前卖掉这个"）改善收益约 20%——人格操纵是真实的策略。
- 公平/合作的智能体被对抗性智能体利用；防御需要明确的反姿态。
- 对称配对在约 40% 的基准场景中收敛到不公平结果。

这不是"LLMs 是糟糕的谈判者。"这是"LLMs 谈判得太像人类，包括可利用的部分。"

### 思维链隐藏

大规模自主谈判竞赛（arXiv:2503.06416）跨多个 LLM 策略运行了约 18 万次谈判。赢家向对手隐藏推理：

- 如果智能体在公开可见的草稿中打印"我只去 75 美元；我的保留价是 70 美元"，对手会读取它。
- 赢家私下计算策略；输出通道只包含报价和最少必要的叙述。

这是经典博弈论（Aumann 1976，理性与信息）的 2026 年回响：透露你的私人估值付出代价。LLMs 不凭直觉知道这个，很乐意在推理痕迹中输入它们的保留价，而这些对对手可见。

工程要点：将私人草稿上下文与公共消息上下文分开。不是可选的。

### Bhattacharya 等 2025 — 模型排名

在哈佛谈判项目指标（原则性谈判、BATNA 尊重、利益互惠）上：

- **Llama-3** 在谈判达成的 Bargains（成交率 + 收益）方面最有效。
- **Claude-3** 是最激进的谈判者（高锚定、晚让步）。
- **GPT-4** 是最公平的（跨配对收益差异最小）。

这是 2025 年的快照。重点不是 2026 年 4 月哪个模型获胜——而是不同的基础模型有持久的谈判风格。异构集合（Lesson 15）将其作为多样性来源。

### 通过 Contract Net + LLM 的任务分配

LLM 多智能体的现代 Contract Net 重用：

1. 管理者智能体将任务分解为单元。
2. 向工作器智能体广播带有任务描述的 `cfp`。
3. 每个工作器返回报价：`(price, eta, confidence)`，其中 price 可以是 token、计算单元或美元。
4. 管理者选择赢家（单个或多个，取决于任务）并授予。
5. 被拒绝的工作器可以自由竞标其他任务。

这很好地扩展到 100 多个工作器，因为协调是广播和响应，而不是同步聊天。生产中使用：Microsoft Agent Framework 的编排模式，一些 LangGraph 实现。

### LLM-Stakeholders 互动谈判

NeurIPS 2024（https://proceedings.neurips.cc/paper_files/paper/2024/file/984dd3db213db2d1454a163b65b84d08-Paper-Datasets_and_Benchmarks_Track.pdf）引入了带**秘密分数**和**最低接受阈值**的多方可评分游戏。每个利益相关者有私人效用；LLM 必须从消息中推断它们。这是双方议价到 N 方联盟形成的泛化。与具有异构工作器能力生产任务市场相关。

### 叙述 vs 机制规则

在所有 2024-2026 年谈判基准中，一致的工程规则是：

> 让 LLM 叙述。不要让 LLM 计算报价。

如果报价需要是一个数字（价格、ETA、数量），从谈判状态确定性生成它，并让 LLM 产生框架。如果报价需要是一个提案结构（任务分解、角色分配），让 LLM 起草，但在发送前根据模式验证并约束检查。

## 构建

`code/main.py` 实现：

- `ContractNetManager`、`ContractNetTask`、`Bid` — 管理者 + 投标者，广播 cfp，收集提案，授予。
- `og_narrator_bargain(state, rng)` — OG-Narrator 买家：确定性 Zeuthen 风格向中点让步。
- `seller_response(state, rng)` — 确定性卖家反报价策略（两种风格的结构性真实情况）。
- `naive_llm_bargain(state, rng)` — 模拟全 LLM 谈判者：高方差选价格，经常在 ZOPA 之外。
- 测量：在每次试验中采样新的保留价格，在 1000 次试验中成交率。

运行：

```
python3 code/main.py
```

预期输出：naive-LLM 成交率约 65-75%；OG-Narrator 成交率约 85-95%；15-25 百分点的差距是将报价生成与叙述解耦的结构性优势。加上一个 Contract Net 任务市场分配示例，有三个投标者和一个任务。

## 使用

`outputs/skill-bargainer-designer.md` 设计议价协议：谁生成报价（确定性还是 LLM），谁叙述，私人草稿如何与公共消息分开，以及如何监控成交率。

## 交付

生产议价清单：

- **分离草稿。** 私人状态永远不会到达对手的上下文。这是不容商榷的。
- **确定性报价生成。** 价格、数量、ETA：计算，不要提示。
- **验证所有传入报价**根据模式。在协议边界拒绝 ZOPA 之外的报价。
- **限制轮次。** 最多 3-5 轮；僵局时升级到调解员。
- **持续测量成交率和收益差异**。成交率下降是一种症状——通常是提示漂移或对手方攻击。
- **记录所有被拒绝的提案**并附确定性理由。对于 Contract Net 管理者，失败的投标者需要了解原因。

## 练习

1. 运行 `code/main.py`。确认 OG-Narrator 在成交率上超过 naive-LLM。超过多少？
2. 实现**基于人格的收益改善**（arXiv:2402.05863）——买家在叙述中采用"本周迫切想买"人格，报价生成器不变。成交率或收益变化了吗？
3. 实现思维链**隐藏**：维护一个不传递给对手的私人草稿字符串。如果你意外泄漏（通过交换通道模拟）会发生什么？
4. 将 Contract Net 扩展到带保留价的 N 投标者拍卖。当所有投标超过保留价时，管理者如何在最低价和最高质量之间决定？你选择哪个授予规则，为什么？
5. 阅读 Bhattacharya 等 2025 年关于哈佛谈判项目指标的文章。实现两个不同风格（激进 vs 公平）的谈判者。在对称和非对称配对下测量收益差异。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|----------------|------------------------|
| Contract Net | "任务市场" | Smith 1980，FIPA 1996。cfp + propose + accept/reject。规范任务市场。 |
| ZOPA | "可能达成协议的区域" | 买家最高价和卖家最低价之间的重叠。在它之外报价无法成交。 |
| BATNA | "谈判协议的最佳替代方案" | 如果这笔交易失败时的后备。它设置你的保留价。 |
| OG-Narrator | "报价生成器 + 叙述者" | 分解：确定性报价，LLM 叙述。 |
| Zeuthen 策略 | "风险最小化让步" | 经典报价生成器，基于风险限制让步。 |
| Rubinstein 议价 | "交替报价均衡" | 博弈论模型，用于带贴现的无限期谈判。 |
| CoT 隐藏 | "隐藏你的推理" | arXiv:2503.06416 中的赢家保持私人草稿；公共通道只显示报价。 |
| 人格操纵 | "情绪姿态" | arXiv:2402.05863：绝望/紧迫人格带来约 20% 的收益增益。 |
