# 心智理论与协调

> 拥有心智理论（Theory of Mind，ToM）让智能体对他人的信念、意图和知识状态进行推理。FoMo-S 是 2025 年精细化的 ToM 评估基准，覆盖 7 项原子能力（感知、目标、信念、意图、知识、情感、动机）和 4 个复杂度等级（单主体/联合/完全递归/嵌套递归），揭示当前模型在联合推理和完全递归 ToM 上存在明显缺陷。多智能体协调需要 ToM：理解谁持有什么信息、谁在试图做什么、谁的意图会与自己的计划冲突。在协调游戏中，基于 ToM 的智能体可以预测其他玩家的行动并提前规划；没有 ToM 的智能体只能看到对手的行为而无法理解其意图。

**类型：** 学习
**前置要求：** Phase 16 · 17（生成式智能体）
**时间：** 约 60 分钟

## 学习目标

- 解释心智理论（ToM）的定义及其为何对多智能体协调至关重要。
- 在 FoMo-S 基准上列出 7 项原子 ToM 能力和 4 个复杂度等级。
- 识别当前模型在联合推理和完全递归 ToM 上的具体缺陷。
- 设计一个多智能体协调场景，其中 ToM 感知智能体比非 ToM 智能体有可量化的优势。

## 问题

两个智能体在一个共享世界中行动。一个智能体看到另一个向冰箱走去。它知道冰箱通常是放食物的地方，但不知道这个智能体刚刚在跑步中消耗了大量卡路里。如果它有心智理论，它可以推断出另一个智能体可能是饿了，因此可以在厨房等待并主动提供帮助。没有 ToM 的智能体只能看到"走向冰箱"这一行为，无法推断意图。

在协调场景中，这种差异决定了你是否能主动配合还是只能被动反应。

## 概念

### 心智理论定义

心智理论（Theory of Mind，ToM）是推断他人心理状态的能力——信念、欲望、意图、知识、情绪。人类从约 4-5 岁发展出这种能力，这也是区分人类发展中的里程碑。在智能体系统中，ToM 意味着对其他智能体的心理状态进行推理。

**对多智能体工程的相关性：**

- **协调**：理解他人的意图可以预测行为并主动配合。
- **委托**：知道谁知道什么，避免重复询问或传递已知信息。
- **冲突解决**：理解对方为什么不同意你（基于他们的信念状态）而不是仅仅不同意。

### FoMo-S 评估基准

FoMo-S（2025 年）是 ToM 评估的精细化版本，定义了 7 项原子能力：

1. **感知**（Perception）：理解他人看到了什么。
2. **目标**（Goal）：理解他人的目标是什么。
3. **信念**（Belief）：理解他人相信什么（包括错误的信念）。
4. **意图**（Intention）：理解他人打算做什么。
5. **知识**（Knowledge）：理解他人知道什么、不知道什么。
6. **情感**（Emotion）：理解他人的情绪状态。
7. **动机**（Motivation）：理解他人为什么做某事。

以及 4 个复杂度等级：

| 等级 | 名称 | 描述 | 示例 |
|------|------|------|------|
| L1 | 单主体 | 一个智能体对另一个的 ToM 推理 | "A 认为 B 饿了" |
| L2 | 联合 | 两个或多个智能体共同的 ToM 状态 | "A 认为 B 和 C 都知道 X" |
| L3 | 完全递归 | A 认为 B 认为 A 认为…… | "A 认为 B 认为 A 不知道 Y" |
| L4 | 嵌套递归 | ToM 嵌套在 ToM 中 | "A 认为 B 想要 C 认为 D 做了 X" |

### 当前模型在 FoMo-S 上的表现

2025 年精细化评估揭示：

- **L1（单主体）**：大多数前沿模型表现良好，接近人类水平。
- **L2（联合）**：性能显著下降——在需要理解群体心理状态时失败。
- **L3（完全递归）**：明显缺陷——当需要递归推理时急剧下降。
- **L4（嵌套递归）**：严重不足——这是最难的任务，当前模型表现远低于人类。

**含义**：对于多智能体工程，使用 L2-L3 复杂度以上的 ToM 推理时，必须意识到模型能力的上限。系统设计应避免深层递归 ToM 要求。

### ToM 在多智能体协调中的价值

**协调游戏中的 ToM 感知智能体：**

在没有 ToM 的情况下，智能体只能观察他人行动并做出反应。加入 ToM 后，智能体可以：

1. **预测行动**：基于他人的目标/意图预测他们的下一步行动。
2. **主动配合**：在对方行动之前就主动配合。
3. **信息共享**：知道谁不知道什么，避免重复询问。
4. **冲突避免**：识别可能导致冲突的意图并在计划阶段解决。

**示例：会议室协调**

- 无 ToM：A 智能体看到 B 智能体离开会议室，推断 B 可能已完成。
- 有 ToM：A 智能体推断 B 可能在等 A 的同意（因为 B 知道 A 是决策者），并主动发出信号。

### 实现 ToM 感知智能体的实用模式

**模式 1：心理状态注释**

每个智能体维护一个关于其他智能体的心理状态注释集合：

```
other_agents["B"].beliefs["meeting_topic"] = "budget"
other_agents["B"].goals["immediate"] = "get_approval"
other_agents["B"].knowledge["who_knows_what"] = {...}
```

**模式 2：ToM 推理调用**

当需要推理他人时，调用 LLM 进行 ToM 推理：

```
ToM_reason(agent="B", query="B会怎么理解我的请求？")
```

**模式 3：意图预测**

在采取行动前，预测他人的反应：

```
predict_reaction(from="A", action="propose_new_plan", to="B")
```

### 故障模式

- **过度推理**：ToM 推理的 token 成本可能很高。将其限制在真正需要的地方。
- **ToM 幻觉**：模型可能虚构他人的心理状态。在高风险决策前验证。
- **深度 ToM 崩溃**：当递归层数超过模型能力时，结果无意义。设置最大递归深度。
