# 心智社会与多智能体辩论

> Minsky 1986 年的前提——智能是专家的社会——每十年被重新发现。2023 年 Du 等人将其变成具体算法：多个 LLM 实例提出答案，阅读彼此的答案，批评，并更新。经过 N 轮他们在共识上收敛，在六个推理和事实性任务上胜过了零样本 CoT 和反思。两个发现很重要：**多个智能体**和**多轮**独立贡献。社会中胜过了单智能体独白；多轮交换胜过了一次性投票。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** Phase 16 · 04（原语模型）
**时间：** 约60分钟

## 问题

自洽——对同一模型采样 N 次并取多数答案——是你能加上的最便宜的推理改进。它工作，但很快饱和。你可以加倍样本，不再看到有意义的跳跃。

辩论打破了饱和。不是 N 个来自同一模型的独立样本，N 个智能体阅读彼此的推理并修改。因为样本不再是 i.i.d.，它们之间的相关性下降，汇聚点通常正确，而 i.i.d. 投票是确信错误的。

## 概念

### Du 等人 2023 年算法

来自 arXiv:2305.14325（ICML 2024）：

1. 每个 N 智能体产生问题的初始答案。
2. 对于第 r = 2..R 轮：每个智能体被展示其他智能体第 r-1 轮的答案，并被问"考虑到这些，给出你更新的答案。"
3. R 轮后，对最终答案多数投票。

论文在 MMLU、GSM8K、传记、MATH 和事实性基准上测试。辩论持续胜过 CoT 和自我反思。

### 两个独立旋钮

同一篇论文的消融：

- **仅智能体数量**（1 轮，N 的多数投票）胜过大多数任务上的单智能体，但 plateau。
- **仅轮数**（1 个智能体看到自己之前的推理）几乎没有帮助——反思的已知弱点。
- **两者一起**产生大跳跃。多智能体之间的多轮交换驱动增益。

### 为什么有效

两个机制：

1. **暴露分歧。** 当智能体看到另一智能体有不同结论的推理链时，它必须证明或更新。无论哪种，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **相关误差减少。** 在自洽中，所有样本来自同一模型，所以误差相关——你平均成一个确信错误的答案。不同模型或不同种子去相关。不同*辩论观点*进一步去相关。

### 异构辩论

A-HMAD 和相关后续使用*不同基础模型*用于不同智能体。Llama + Claude + GPT 辩论减少单文化崩溃（Lesson 26），因为一个模型族的相关误差不被其他模型共享。

缺点：参与辩论的弱模型可以将共识拖向其错误答案（见"我们应该走 MAD 吗？"，arXiv:2311.17371）。

### NLSOM——129 智能体扩展

Zhuge 等人（"自然语言社会中思维风暴"，arXiv:2305.17066）将此想法扩展到 129 个成员社会。结果：专业化与自我组织随规模涌现，系统在视觉问答等任务上胜过单智能体。

### 失败模式

- **谄媚级联。** 所有智能体听从最确信的声音。辩论崩溃为最大声音。提示对抗性角色（"一个智能体必须争论反对立场"）有帮助。
- **话题漂移。** 跨多轮辩论偏离原始问题。缓解：每轮重新注入问题。
- **计算爆炸。** N 智能体 × R 轮 = N·R LLM 调用，每次都有增长的上下文。5 智能体、5 轮辩论是 25 次调用，且上下文增长。每个问题成本可超过单 CoT 调用的 10 倍。

## 构建

`code/main.py` 运行 3 智能体 × 3 轮辩论关于数学问题，每个智能体以不同的（可能错误的）答案开始。智能体是脚本化的——每个"更新"通过根据脚本置信加权平均邻居答案。收敛在轮次日志中可见。

演示显示两个关键效应：

- 单轮交换将智能体移近正确答案。
- 超过第 2 轮的额外轮显示递减收益（匹配 Du 等人的 plateau）。

运行：

```
python3 code/main.py
```

## 使用

`outputs/skill-debate-configurator.md` 为新任务配置辩论：智能体数量、轮数、异构性（同模型 vs 混合）、角色分配（对称 vs 一个对抗）。还估算运行前的 token 成本。

## 交付

如果你交付辩论：

- **轮数上限为 3。** Du 等显示 3 轮捕获大部分增益。更多是成本，不是质量。
- **智能体上限为 5。** 超过 5，上下文膨胀和成本占主导。
- **默认异构。** 池中至少两个不同基础模型。
- **对抗槽。** 一个无论什么都提示反驳的智能体。打破谄媚。
- **记录每轮。** 隐藏中间轮的辩论系统无法调试或审计。

## 练习

1. 运行 `code/main.py`，然后设置轮数为 5，观察递减收益。在哪轮额外收敛停止？
2. 添加具有对抗角色的第四个智能体：总是不同意当前多数。打破还是改善收敛？
3. 打印（每轮协议分数（多数答案上的智能体分数）。何时达到 1.0，那等同于"正确"吗？
4. 阅读 Du 等人第 4 节消融。用此代码复制"仅智能体"vs"仅轮数"vs"两者"结果。
5. 阅读"我们应该走 MAD 吗？"(arXiv:2311.17371) 并列出除轮询之外的两种辩论变体——例如，法官主导、辩论链。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|----------------|------------------------|
| 心智社会 | "Minsky 的想法" | 交互专家的智能；1986 年框架现在通过 LLM 辩论操作化。 |
| 多智能体辩论 | "智能体争论" | N 智能体提议、批评、修改 R 轮，多数投票。 |
| 共识 | "他们同意" | 不是认知真理——只是多数答案的分数。可以确信错误。 |
| 轮 | "交换步骤" | 一轮 = 每个智能体阅读他人并更新一次。 |
| 异构辩论 | "混合模型族" | 使用不同基础模型去相关误差。 |
| 谄媚级联 | "每个人同意最大声音" | 辩论失败，智能体听从最确信智能体，不管正确性。 |
| NLSOM | "129 智能体社会" | 自然语言心智社会；Zhuge 等人的扩展版本。 |
| 相关误差 | "同一模型，相同 bug" | 自洽饱和的原因；辩论跨不同观点去相关。 |
