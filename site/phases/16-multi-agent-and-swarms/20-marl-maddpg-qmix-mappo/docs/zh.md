# 多智能体强化学习 — MADDPG、QMIX、MAPPO

> 多智能体强化学习（MARL）在协作、竞争和混合博弈中训练多个智能体同时行动。三个代表性算法代表三种范式：**MADDPG**（集中式训练 + 分散式执行，CTDE 范式）用于协作任务；**QMIX**（值分解，monotonic 约束保证全局最优可从局部最优推导）用于协作团队的 Dec-POMDP；**MAPPO**（PPO 在多智能体场景，2022-2024 年成为主流基线）在 MARL 基准上接近甚至超越 MADDPG 和 QMIX，且实现更简单。三个算法的核心选择在于：CTDE vs 完全分散、值分解 vs 策略梯度、是否需要处理竞争性智能体。

**类型：** 学习 + 构建
**语言：** Python（PyTorch，toy MADDPG/QMIX/MAPPO）
**前置要求：** Phase 09（强化学习）、Phase 16 · 09（并行群体网络）
**时间：** 约 90 分钟

## 学习目标

- 解释三种 MARL 范式：CTDE（ MADDPG）、值分解（QMIX）、策略梯度（MAPPO）及其适用场景。
- 实现 MADDPG 的集中式评论家 + 分散式行动者架构。
- 解释 QMIX 的单调性约束及其为何允许全局最优从局部最优推导。
- 在 MARL 基准（Multi-Agent Particle Environments, SMAC）上比较三种算法的样本效率和收敛稳定性。

## 问题

单智能体 RL 中，智能体学习一个策略，最大化折扣奖励总和。多智能体 RL 中，多个智能体同时学习，环境中包含其他学习智能体。这带来两个新问题：

1. **非平稳性**：当我改变策略时，环境对我来说是不同的（因为其他智能体也在学习）。
2. **信用分配**：在协作任务中，团队的产出是所有智能体共同贡献的——如何知道每个智能体贡献了多少？

三种算法用不同方式解决这两个问题。

## 概念

### MADDPG（Multi-Agent DDPG）

**架构：** 集中式训练 + 分散式执行（CTDE）。

- **评论家**：知道所有智能体的状态和动作（全局信息），学习 Q 值。
- **行动者**：只看到本地观测，选择动作。

**为什么可行：**
评论家在训练时可以使用全局信息（处理非平稳性），但执行时行动者只需要本地信息（分散式执行）。

**数学：**
对于智能体 i：
```
Q_i(s, a_1, ..., a_n)  # 评论家：全局状态，所有智能体的动作
π_i(o_i)               # 行动者：本地观测
```

**梯度更新：**
```
∇_θ_i J = E[∇_a_i Q_i · ∇_θ_i π_i(o_i)]  # 策略梯度
```

**局限性：**
- 评论家随智能体数量线性增长。
- 假设所有智能体都可以被集中训练（不适用于竞争场景）。

### QMIX（值分解）

**核心思想：** 全局 Q 值 Q_tot 不能直接分解为各智能体 Q_i 的简单加权和，但通过施加单调性约束可以保证全局最优可通过局部贪婪动作实现。

**单调性约束：**
```
∂Q_tot / ∂Q_i ≥ 0  ∀i  # 全局 Q 对每个局部 Q 的偏导非负
```

这意味着当每个局部 Q 值最大时，全局 Q 值也最大。

**架构：**
- 局部 Q 网络：每个智能体一个，输入本地观测和动作。
- 混合网络（Mixing Network）：将局部 Q 合并为全局 Q，施加单调性约束。

**为什么强于简单加权和：**
在协作中，简单加权和可能允许全局最优对应非局部贪婪动作（每个智能体不贪婪但团队最优）。单调性约束堵住这个漏洞。

**适用场景：**
- 协作团队任务（完全合作）
- 部分可观测（Dec-POMDP）
- 大规模智能体（可扩展，因为局部 Q 可以独立训练）

### MAPPO（Multi-Agent PPO）

**核心思想：** PPO 在多智能体场景中的应用。2022-2024 年的 MARL 主流基线。

**关键设计：**
- **中心化价值函数（CVF）**：和 MADDPG 类似，用全局信息训练价值函数。
- **Clip 目标**：限制策略更新幅度，防止破坏性大更新。
- **重要性采样**：在 CVF 估计中使用重要性采样修正偏差。

**性能：**
在 MARL 基准（SMAC、MAgent）上，MAPPO 接近或超越 MADDPG 和 QMIX，且实现更简单（借助成熟的 PPO 基础）。

**为什么 MAPPO 成为 2024-2026 年主流：**
- PPO 的样本效率高于 DDPG 类方法（更大的有效批次大小）。
- Clip 机制提供稳定的策略更新。
- 实现简单（可以用 RLlib 等成熟框架）。

### 三种算法比较

| 属性 | MADDPG | QMIX | MAPPO |
|------|--------|------|-------|
| 范式 | CTDE + 策略梯度 | CTDE + 值分解 | CTDE + 策略梯度（PPO） |
| 局部 Q | 无 | 有单调性约束 | 无（用价值函数） |
| 适用场景 | 协作为主 | 大规模协作 | 协作/竞争均可 |
| 实现复杂度 | 中 | 高 | 低（借助 PPO） |
| 样本效率 | 低 | 中 | 高 |
| 稳定性 | 低（策略梯度方差大） | 中 | 高（Clip 机制） |
| 可扩展性 | 低 | 高 | 高 |
| 竞争场景 | 支持 | 仅协作 | 支持 |

### 协作/竞争/混合场景的选择

- **完全协作**（如多机器人搬运）：QMIX 或 MAPPO。
- **竞争**（如 Sumo 风格的博弈）：MADDPG 或 MAPPO。
- **混合**（如星际争霸：己方协作，对抗敌方）：MAPPO（更灵活）+ 自我博弈（self-play）增强。
- **部分可观测 + 大规模**（如 SMAC）：QMIX（可扩展）或 MAPPO。

### 故障模式

- **非平稳性崩溃**：MADDPG 中评论家训练速度跟不上智能体策略变化。解决：经验回放需要包含历史策略样本。
- **QMIX 单调性过强**：单调性约束限制了表达力，可能错过非单调最优解。
- **MAPPO CVF 偏差**：中心化价值函数因部分可观测产生偏差。解决：使用多步返回（TD(λ)）减少偏差。

## 构建它

`code/main.py` 实现三个算法的简化版本：

- **MADDPG**：用于简单的 2D 协作导航任务（两个智能体共同到达目标）。
- **QMIX**：用于协作拾取任务（多智能体共同收集散布的物品）。
- **MAPPO**：用于竞争性任务（两个智能体追逐/躲避角色互换）。

每个实现在 500 episodes 内演示收敛趋势。

## 关键术语

| 术语 | 人们怎么说 | 实际上指什么 |
|------|-----------|--------------|
| MARL | "多智能体 RL" | 多智能体强化学习 |
| CTDE | "集中训练分散执行" | 训练时用全局信息，执行时用本地信息 |
| MADDPG | "集中评论家 DDPG" | CTDE + 策略梯度；局部动作者，全局评论家 |
| QMIX | "单调 Q 分解" | 值分解 + 单调性约束；允许从局部 Q 推导全局最优 |
| MAPPO | "多智能体 PPO" | PPO 在多智能体场景；2024-2026 年主流基线 |
| CVF | "中心化价值函数" | MAPPO 中用全局信息训练的价值函数 |
| 单调性约束 | "单调 Q" | QMIX 中保证 ∂Q_tot/∂Q_i ≥ 0 的约束 |
| 非平稳性 | "环境变化" | 其他智能体学习时环境特性的变化 |
| 信用分配 | "谁该得功劳" | 协作中如何分解团队奖励给各智能体 |
