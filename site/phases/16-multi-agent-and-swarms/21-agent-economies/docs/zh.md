# 智能体经济 — 资源分配、市场与激励

> 当多个自主智能体在一个共享环境中行动时，它们实际上是在参与一个经济体。资源（token、计算、内存、信息）需要分配，激励需要设计，外部性需要处理。智能体经济学的核心框架来自机制设计（mechanism design）：你想要什么结果（目标）→ 你设置什么规则（机制）→ 智能体如何响应（激励相容）。2026 年的实际问题：多智能体系统如何处理 token 预算分配（谁的请求优先处理）、信息市场（智能体之间买卖信息）、计算资源调度（谁获得 GPU 时间）、以及恶意智能体的防御。

**类型：** 学习
**语言：** Python（标准库，toy 经济模拟器）
**前置要求：** Phase 16 · 20（MARL）
**时间：** 约 60 分钟

## 学习目标

- 解释机制设计的三个核心概念：目标、机制、激励相容。
- 设计一个 token 预算分配机制（优先级拍卖、比例分配、固定配额）。
- 解释信息市场：智能体之间何时买卖信息，什么决定价格。
- 识别智能体经济中的恶意行为（资源耗尽攻击、女巫攻击、泛洪攻击）并设计防御。

## 问题

你的系统中现在有 5 个自主智能体，各自追求自己的目标。它们共享有限的 GPU 资源、token 预算和信息库。问题是：

1. **谁获得 GPU 时间？** 智能体 A 愿意为优先处理支付多少？
2. **信息价值多少？** 智能体 B 有一个关于市场的信息——卖给谁？多少钱？
3. **恶意智能体如何防御？** 智能体 C 试图通过发送大量请求来耗尽所有人的资源。

这些问题不是技术问题，是经济学问题。智能体经济学提供处理它们的框架。

## 概念

### 机制设计基础

机制设计是"逆向博弈论"：从你想要的全局结果出发，设计游戏规则使理性参与者自然达到该结果。

**三个核心概念：**

- **目标（Objective）**：你想要达到的社会福利函数。
  - 帕累托最优：没有人能在不让别人变差的情况下变得更好。
  - 功利主义：最大化所有智能体奖励之和。
  - 最大化最小公平（Maximin）：最大化最差智能体的福利。

- **机制（Mechanism）**：决定资源分配的游戏规则。
  - 拍卖：出价最高者获得资源。
  - 配额：每个智能体获得固定份额。
  - 优先级队列：按紧急程度排序。

- **激励相容（Incentive Compatibility）**：说实话是最优策略。
  - 占优策略激励相容（DSIC）：无论他人如何选择，说实话总是最优。
  - 贝叶斯激励相容（BIC）：在他人说实话的期望下，说实话是最优。

**关键定理：Myerson 引理（启示原理）**
如果一个机制是 DSIC 的，那么一定存在一个对应的直接机制产生相同结果。这大大简化了机制设计。

### Token 预算分配机制

**场景：** 5 个智能体竞争有限的 token 预算。

**机制 1：优先级拍卖**
每个智能体提交竞价 b_i，用于购买优先级 p_i。
```
p_i = b_i / sum(b)  # 按比例分配优先级
```

优点：信息价值高的智能体愿意出更高价，信息被引导到最有价值的地方。
缺点：富智能体可能主导；恶意竞标可能导致资源浪费。

**机制 2：按比例分配**
每个智能体获得固定比例：
```
budget_i = total_budget * (weight_i / sum(weights))
```

优点：稳定、可预测。
缺点：无法反映紧急性；智能体可能囤积权重。

**机制 3：紧急度加权**
按紧急度函数 e_i 分配：
```
priority_i = f(deadline_i, importance_i, wait_time_i)
```

优点：反映实际需求。
缺点：需要正确的紧急度函数；可能被操纵。

### 信息市场

**场景：** 智能体 B 有一个关于某产品的市场分析报告。卖给谁？多少钱？

**信息定价的原则：**

- **边际价值**：买家的边际收益 = 信息对其决策的价值改善。
- **稀有性**：信息越稀有，价格越高。
- **可验证性**：可验证的信息比不可验证的信息更有价值。

**简单的信息拍卖：**

```python
def info_auction(info_seller, info_内容, buyers):
    bids = [buyer.愿意支付(info_内容) for buyer in buyers]
    winner = argmax(bids)
    price = second_price(bids)  # 维克瑞拍卖：出价次高者价格
    transfer(info_seller, winner, info_内容, price)
```

**实际问题：**
- 信息复制成本为零。一旦卖出，买家可以免费复制给其他人。
- 解决方案：订阅模式（持续付费获取最新信息）或 DRM 模式（可信执行环境保护）。

### 计算资源调度

**场景：** 智能体需要 GPU 时间来运行推理。

**机制：需求聚合 + 比例分配**

```
每个 tick:
    汇总所有智能体的需求请求
    按紧急度和历史公平分配 GPU 时间片
    超时请求自动降级（减少 token 预算或降低模型大小）
```

**优先级动态调整：**
- 持续低优先级请求：逐渐降低其优先级（防止滥用）。
- 高价值任务：允许突发（burst）到分配之上，然后在后续 tick 摊销。

### 恶意行为与防御

**1. 资源耗尽攻击（Resource Exhaustion）**
一个恶意智能体发送大量低价值请求，耗尽 token 预算。

防御：
- 速率限制：每个智能体每秒请求数有上限。
- 最小价值门槛：每个请求必须有估计价值 > 阈值。
- 信誉系统：历史高价值请求的智能体获得更多份额。

**2. 女巫攻击（Sybil Attack）**
一个对手创建多个虚假智能体身份，获得不成比例的份额。

防御：
- 身份验证：每个智能体需要一个可信身份（基于 stake 或认证）。
- 基于 stake 的分配：token 分配权重与锁定 stake 成正比。
- 去中心化身份（DID）：基于加密身份而非账户列表。

**3. 泛洪攻击（Flooding）**
同一智能体同时向多个其他智能体发送垃圾信息。

防御：
- 信息质量门槛：每个信息必须经过评分门槛。
- 发送成本：发送信息需要消耗 token（类似电子邮件 postage）。

### 激励结构设计

**激励与目标的对应：**

| 目标 | 激励结构 | 机制 |
|------|---------|------|
| 最大化总产出 | 产出的比例分成 | 按比例分配资源 |
| 公平性 | 最低保障 + 权重调整 | 加权公平队列 |
| 质量优先 | 质量加权分配 | 质量评分 × 数量 = 总优先级 |
| 快速迭代 | 紧急度折扣 | 紧急度越高，等待成本越高 |

**目标漂移（Goal Misalignment）：**
当个人激励与集体目标冲突时会发生。例：每个智能体最大化自己的 token 份额，但集体需要 token 预算运行一个高价值任务。

解决：引入税收/补贴机制。对过度消耗资源的智能体征税，对高价值集体任务参与者补贴。

## 关键术语

| 术语 | 人们怎么说 | 实际上指什么 |
|------|-----------|--------------|
| 机制设计 | "逆向博弈论" | 从目标出发设计游戏规则 |
| 激励相容 | "说实话是最优策略" | IC 约束：说谎不比说实话好 |
| DSIC | "强激励相容" | 占优策略激励相容；无论他人如何选择，说实话总最优 |
| Myerson 引理 | "启示原理" | DSIC 机制 ⇔ 直接机制存在 |
| 资源耗尽攻击 | "DoS 攻击" | 通过大量低价值请求耗尽共享资源 |
| 女巫攻击 | "多身份攻击" | 创建多个虚假身份获取不成比例资源 |
| 信誉系统 | "信任积分" | 基于历史行为调整未来资源分配权重 |
| 目标漂移 | "激励错位" | 个人激励与集体目标不一致 |
