# 生成式智能体与涌现模拟

> Park 等人 2023 年（UIST '23，arXiv:2304.03442）用三部分架构填充了 **Smallville**，一个 25 个智能体的沙盒：**记忆流**（自然语言日志）、**反思**（智能体对自己记忆流生成的高层综合）、**规划**（日级行为，再分解为子计划）。标志性成果是情人节派对涌现：一个智能体被赋予"想举办情人节派对"，无需进一步脚本，通过 24 个初始对此毫不知情的智能体，产生了邀请传播、协调日期、派对最终举办。本课程是该架构的实现指南，涵盖消融实验、故障模式，以及如何为 2026 年生产系统准备。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置要求：** Phase 16 · 04（原始模型）、Phase 16 · 13（共享内存）
**时间：** 约 75 分钟

## 问题

大多数多智能体系统是紧脚本团队：规划者规划、编码者编码、审查者审查。这对定义明确的任务有效。但无法捕捉当智能体拥有记忆、优先级和开放世界时涌现出的非脚本行为。学术研究、社会模拟，以及日益增长的游戏 AI 都需要第二种类型。

Smallville 架构是这类系统的基准。Park 2023 年之前，最佳智能体模拟是浅层脚本追随者；之后，该模式成为开放世界生成式智能体的默认架构。如果你在 2026 年构建智能体模拟，你要么使用 Smallville 的三个组件，要么明确说明为何不这样做。

## 概念

### 三个组件

**记忆流。** 一个只追加的日志，记录观察、动作、反思和规划。每个条目有时间戳、类型、描述（自然语言）和派生元数据：**新鲜度**、**重要性**（智能体自评 1-10）、以及**相关性**（与当前查询的余弦相似度）。

```
[2026-02-14 09:12:03] 观察：Isabella Rodriguez 问我是否喜欢爵士乐
[2026-02-14 09:14:22] 反思：我喜欢关于音乐的长时间对话
[2026-02-14 10:05:00] 规划：今晚参加 Isabella 的情人节派对
```

记忆检索结合三个分数：`score = w_recency * e^(-decay * age) + w_importance * importance + w_relevance * cos_sim`。Top-k 条目进入当前 prompt。

**反思。** 定期（每 N 条记忆或重要事件发生时），智能体从近期记忆中生成高层综合。反思条目重新进入流中，与任何其他记忆一样可检索。这就是智能体建立"理解"的方式——该架构相当于长期信念。

**规划。** 自上而下分解。首先是日级计划的大纲（"去上班，与 Klaus 共进晚餐"），然后是小时级计划，再到动作级计划。计划可修订：当观察与计划矛盾时，智能体仅重新规划受影响的部分。

### 三个组件缺一不可（消融实验）

Park 等人对每个组件进行了消融实验。去掉任一都会损害可信度：

- 没有**观察**，智能体错失上下文，依据过时信念行事。
- 没有**反思**，智能体无法形成高层信念；交互停留在浅层。
- 没有**规划**，行为变成反应性噪音；目标消散。

人类评分者的可信度分数在三者齐全时最高；去掉任一都产生可测量的回归。

### 情人节派对的涌现

一个智能体 Isabella Rodriguez 被赋予目标"想在 2 月 14 日 5 点于 Hobbs Cafe 举办情人节派对"。其他 24 个智能体未获得此种子。在模拟的几天里：

1. Isabella 的规划包含邀请他人。
2. 每份邀请成为邻居记忆流中的一条观察。
3. 该邻居的反思生成信念："Isabella 要举办派对。"
4. 邻居的规划加入"2 月 14 日参加派对"。
5. 邻居告知其他邻居。邀请在无中央协调的情况下传播。
6. 2 月 14 日 5 点，多个智能体汇聚于 Hobbs Cafe。

这是技术意义上的涌现：系统级行为（一个派对）从本地交互（双边邀请 + 个人规划）中产生，无中央编排器。

### 记录的故障模式

Park 等人明确记录了：

- **空间规范错误。** 智能体走进已关闭的商店。智能体试图共用单人洗手间。智能体在不 intended 进食的房间进食。模型无法仅从环境单独推断社会-物理规范。
- **记忆溢出。** 深度模拟运行导致记忆检索成本增长。实际补救措施：定期记忆压缩（总结-裁剪）和低重要性条目的衰减。
- **反思幻觉。** 反思可能虚构记忆中不存在的关联。缓解措施：在反思 prompt 中包含源记忆 ID，并在检索时验证。

这些是可复现的故障模式：2026 年任何智能体模拟都继承它们。

### 三组件实现规则

1. **记忆是只追加的。** 永不修改记忆条目。纠正在日志中以新条目追加。
2. **重要性分数很廉价。** 在写入时调用 LLM 评分 1-10。缓存分数。
3. **检索是排序的，不是过滤的。** 按组合分数取 Top-k；不用硬过滤（会丢失上下文）。
4. **反思定期运行。** 当未处理记忆的重要性之和超过阈值（例如 150）时触发。
5. **规划可修订。** 当新观察与计划矛盾时，仅重新生成受影响的部分，而非整个计划。

### Smallville 之外的生成式智能体

2024-2026 年后续文献扩展了该架构：

- **用于政策/市场研究的多智能体社会模拟。** 类似 Smallville 的人群模拟用户对功能的反应。比 A/B 测试快；准确性存在争议。
- **游戏 NPC AI。** 带 Smallville 智能体的 RPG 产生涌现性故事情节，而非脚本任务。
- **生成式智能体评估基准。** 指标不是任务准确性，而是长时间运行的可信度和行为连贯性。

该架构是可供参考的基准。扩展时替换组件（向量存储替代记忆、检索增强反思、神经符号规划）但保持三部分结构。

### 为何这对多智能体工程很重要

Smallville 证明了当组件正确时，多智能体涌现是廉价的。该架构已在开源模型上复现（小模型失去可信度是平稳的，而非急剧的）。任何需要**涌现社交行为**的生产系统使用此形状。任何需要**紧密任务执行**的系统使用主管/角色/原语模式。

## 构建它

`code/main.py` 用标准库 Python 实现三个组件（含脚本化智能体策略，无真实 LLM）。演示以微型方式复现情人节派对涌现：

- `MemoryStream` — 带新鲜度/重要性/相关性检索的只追加日志。
- `reflect(stream)` — 对近期高重要性记忆的脚本化反思。
- `plan(agent_state)` — 基于当前信念的日级和小时级计划。
- 场景：5 个智能体。智能体 1 以"5 点举办派对"开始。在模拟 tick 中，邀请传播，智能体汇聚。

运行：

```
python3 code/main.py
```

预期输出：逐 tick 跟踪。到最后一个 tick，至少 3/5 个智能体在计划中显示派对，且汇聚于派对地点。单个种子无需任何编排器产生了协调到达。

## 使用它

`outputs/skill-simulation-designer.md` 设计一个生成式智能体模拟：智能体数量、记忆 schema、反思节奏、规划范围和评估指标。

## 交付它

生产模拟规则：

- **记忆是数据库。** 规模化后用真实存储（向量 DB、Postgres）。内存标准库仅用于原型。
- **记录检索跟踪。** 每个动作记录驱动它的 Top-k 记忆。这是你的调试能力。
- **每个智能体 token 有预算。** 每个智能体每 tick 的 retrieve + reflect + plan 是 O(k) 次 LLM 调用。N 个智能体 × T 个 tick × 每 tick 调用数可以吞噬你的预算。
- **定期压缩记忆。** 总结-裁剪低重要性条目。保留策略是设计决策，不是细节。
- **显式检测空间/社会规范违反。** 架构不会学习它们。

## 练习

1. 运行 `code/main.py`。确认 3+ 个智能体在派对汇聚。将智能体增加到 10 个——涌现是否仍然发生？
2. 移除反思步骤。行为看起来如何？映射到 Park 2023 的消融发现。
3. 引入竞争性种子目标（"Klaus 想在 5 点举办研究讲座"）。智能体会分裂还是一个目标主导？什么决定它？
4. 添加空间约束：Hobbs Cafe 最多容纳 4 个智能体。模拟是否优雅地处理溢出，还是遇到"单人洗手间"故障模式？
5. 阅读 Park 等人（arXiv:2304.03442）第 6 节（涌现行为实验）。识别你的微型版本无法复现的一个行为。你需要增强架构的哪个组件？

## 关键术语

| 术语 | 人们怎么说 | 实际上指什么 |
|------|-----------|--------------|
| Memory stream | "智能体的日记" | 只追加的观察、动作、反思、规划日志 |
| Recency | "记忆有多新" | 按年龄的指数衰减分数 |
| Importance | "智能体有多在乎" | 写入时自评 1-10。已缓存 |
| Relevance | "与当前查询有多相关" | 余弦相似度（基于嵌入） |
| Reflection | "高层信念" | 从近期记忆生成的综合，重新作为新记忆摄入 |
| Plan | "日/小时/动作分解" | 自上而下的计划树。当观察矛盾时可修订 |
| Smallville | "Park 2023 的沙盒" | 25-智能体模拟，产生了情人节派对涌现 |
| Believability | "质量指标" | 人类评分者对行为是否像可信的智能体的评分 |
