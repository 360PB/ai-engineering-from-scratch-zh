# 智能体的共识与拜占庭容错

> 经典分布式系统 BFT 遇上随机 LLM。2025-2026 年出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）按置信度探测加权每个投票；**DecentLLMs**（arXiv:2507.14928）通过并行工作器提案和几何中位数聚合实现无领导者；**WBFT**（arXiv:2505.05103）结合加权投票与层次结构聚类将核心节点和边缘节点分开。"AI 智能体能达成一致吗？"（arXiv:2603.01213）的诚实经验结果是，即使标量一致今天也是脆弱的——一个欺骗性智能体可以损害 Mixture-of-Agents。BFT 是必要的但不是充分的。本课构建一个最小 BFT 协议，注入三个智能体特定攻击（拜占庭谎言、谄媚从众、相关错误同质化），并测量每个共识变体如何应对。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** Phase 16 · 07（心灵社会与辩论），Phase 16 · 13（共享内存）
**时间：** 约 75 分钟

## 问题

你有 N 个 LLM 智能体，每个产生一个答案。它们不同意。多数投票选择了错误的一个，因为两个智能体相关（相同的基础模型、相同的训练数据、相同的失败模式）。第三个智能体恰好以一种新颖的方式出错——所以多数是虚假多数。

现在加上一个欺骗性智能体：它故意撒谎。或者一个谄媚性智能体：它在最后一个发言者之后阅读其他人的答案并与之对齐。在经典 BFT 中，假设是拜占庭节点是 `f < n/3` 的一部分，并且行为任意。2026 年的现实是 LLM 节点即使诚实也是随机的，跨模型相关，并受彼此输出的影响。你不能将它们视为独立的伯努利投票者。

经典 BFT（PBFT，1999）没有错——它是不完整的。它处理任意位翻转。它不处理"三个诚实智能体因共享训练数据而共享一个幻觉"。本课从 PBFT 基础构建，并叠加三个 2025-2026 年的适应。

## 概念

### 经典 BFT 给你的东西

实用拜占庭容错（Castro & Liskov，OSDI 1999）容忍 `f < n/3` 个拜占庭节点。协议有三个阶段（预准备、准备、提交）和两个原语（签名消息、仲裁证书）。在 `n >= 3f + 1` 个诚实或恶意节点中就单一值达成一致。

保证很强但假设：

1. **独立故障。** 拜占庭节点不协调。
2. **诚实节点真正诚实。** 诚实输出的正确性不是问题；协议只对齐不一致。
3. **问题有真实答案。** 在错误事实上的共识仍然是共识。

LLM 智能体违反所有三个。两个运行相同基础模型的智能体共享故障。一个"诚实"的 LLM 仍然产生幻觉。在模糊问题上，"真相"是智能体决定的内容——没有外部 oracle。

### 三个 LLM 特定攻击

**拜占庭谎言。** 一个智能体输出一个故意错误的答案。如果 `f < n/3`，经典 BFT 处理这个。

**谄媚从众。** 一个智能体在投票前读取其他人的答案并与最后一个发言者对齐。不是恶意的，但与最大声的声音相关。经典 BFT 不能阻止这个，因为智能体通过每个签名检查。

**相关错误同质化。** 三个智能体共享一个基础模型。它们产生相同的错误答案。多数错了。经典 BFT 没有帮助，因为所有三个都"诚实"地同意。

### 2025-2026 年的回应

**CP-WBFT**（arXiv:2511.10400）— 置信度探测加权 BFT。每个投票者在其答案上附加置信度探测（自我报告的概率，或单独的校准模型预测）。投票权重随置信度缩放。在完全图上报告 +85.71% BFT 改进。缓解：对谄媚从众（conformity 智能体在自愿立场上的置信度往往较低）。

**DecentLLMs**（arXiv:2507.14928）— 无领导者。工作器智能体并行提案，评估器智能体对提案评分，最终答案是评分位置的几何中位数。当 `f < n/2` 时具有鲁棒性。缓解：拜占庭谎言和相关错误（几何中位数对异常值具有鲁棒性，并向密集聚类拉近，而不是模型偏差的平均值）。

**WBFT**（arXiv:2505.05103）— 带层次结构聚类的加权 BFT。投票权重由响应质量加上从历史学习的信任分数分配。将智能体聚类为核心和边缘；核心智能体必须首先达成共识，边缘智能体跟随。缓解：可扩展性（核心共识小而快）和部分针对同质化（可以选择核心以实现多样性）。

### 经验："AI 智能体能达成一致吗？"（arXiv:2603.01213）

论文测量跨多个前沿模型的标量一致（LLM 智能体就单一数值达成一致）。发现令人不安：

- 即使没有对手，LLM 智能体在许多基准上以高于 30% 的比率在标量问题上不同意。
- 一个采用欺骗人格的单个智能体可以将 Mixture-of-Agents 共识拉离诚实基线 40+ 个百分点。
- 不同意率与模型多样性相关——异构集成比同构集成不同意更多（好：不相关错误）但也漂移更慢（坏：更长的协议时间）。

要点：BFT 给你对齐输出的机制，但不告诉你对齐的输出是否正确。结合验证（Phase 16 · 08 角色专业化）、多样性（Phase 16 · 15 辩论变体）和评估器智能体（Phase 16 · 24 基准）。

### 核心协议，精简

LLM 智能体的最小 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信度探测 c_i 在 [0, 1]
3. 聚合器从所有 n 个智能体收集 (a_i, c_i)
4. 聚合器按语义聚类（等效答案）
5. 聚合器为每个聚类 C 计算权重：
     w(C) = sum_{i in C} c_i
6. 赢家 = 最大权重聚类，如果 max > threshold * sum(c_i)
   否则：重试或升级
7. 少数聚类记录出处用于事后审计
```

语义聚类步骤是 LLM 特定的转折。两个答案"该研究报道了 4.2%"和"4.2% 的提升"是同一聚类。朴素的字符串相等检查会遗漏这一点。生产中，使用便宜的嵌入模型或显式规范化。

### 阈值调优

`threshold` 参数决定何时接受何时重试。太低：你接受弱多数。太 高：你永远不会接受任何东西。经验范围：对于 `n=5-7` 为 0.5-0.67，对于更小的 `n` 更高。低于阈值时升级到人类或不同的智能体集合。

### 共识无法帮助的地方

- **模糊问题。** 如果问题没有真实答案，共识是一种观点。称之为那样。
- **复合问题。** "写代码并解释它"——两个答案。独立对每个投票。
- **对抗性多轮。** 如果智能体可以观察先前行并模仿（Du 2023 辩论），它们无论真相如何都开始彼此同意。限制轮次（通常 2-3）。

## 构建

`code/main.py` 实现：

- `AgentVoter` — 带（答案、置信度）的脚本化策略。
- `MajorityVote` — 经典 plurality。
- `CPWBFT` — 带语义聚类的置信度加权投票。
- `DecentLLMs` — 在评分提案上的几何中位数聚合。
- `Scenario` — 在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度撒谎。
2. `sycophancy`：一个智能体复制它看到的第一个答案，置信度匹配。
3. `monoculture`：三个智能体共享一个错误答案（相关错误），置信度适中。

运行：

```
python3 code/main.py
```

预期输出：（攻击、聚合器）→ 最终答案的表格，正确答案高亮显示。Plurality 在同质化案例中失败。CPWBFT 的置信度加权缓解谄媚。当同质化少于一半人口时，DecentLLMs 的几何中位数向诚实聚类拉近。

## 使用

`outputs/skill-consensus-designer.md` 为多智能体集合设计共识协议：聚类方法、加权、阈值，以及低于阈值轮次的升级策略。

## 交付

在交付任何共识机制之前：

- **用至少上述三种模式进行攻击测试。** 你的协议应该可预测地失败，而不是静默失败。
- **记录每个少数聚类并附带出处。** 少数聚类是你的相关错误早期预警系统。
- **强制限制轮次。** 不是"持续辩论直到达成一致"——那会奖励谄媚。
- **将一致性与正确性分开。** 共识输出进入验证器；验证器独立于集合。
- **监控协议率。** 急剧上升意味着从众偏差；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认 plurality 在同质化攻击中失败，但当同质化置信度低于 0.7 时 CPWBFT 部分缓解它。
2. 添加第四种攻击模式：**静默弃权**——一个智能体拒绝回答（"我不知道"）。每个聚合器应该如何对待弃权？实现你的选择。
3. 将语义聚类从字符串规范化交换为嵌入相似度（使用任何开源嵌入模型）。对谄媚攻击有什么影响？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探测校准步骤（单独的校准模型检查每个智能体自我报告的置信度）。测量在同质化场景上的准确率增益。
5. 阅读"AI 智能体能达成一致吗？"（arXiv:2603.01213）。重现一个简化的标量一致实验：三个智能体，一个标量问题，欺骗性人格提示。CPWBFT 或 DecentLLMs 捕获它吗？

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|----------------|------------------------|
| BFT | "拜占庭容错" | Castro-Liskov 1999 协议，用于在 `f < n/3` 任意故障下达成共识。 |
| 拜占庭 | "任何不良行为" | 可以撒谎、丢弃消息、安静失败的节点——除了安全崩溃之外的任何行为。 |
| 置信度探测 | "你有多确定？" | 附加到投票的自我报告或校准器预测的概率。 |
| 语义聚类 | "相同答案，不同措辞" | 在计票前对等效答案进行分组。 |
| 几何中位数 | "鲁棒中心" | 最小化到样本点距离之和的点。对异常值鲁棒，与平均值不同。 |
| 同质化 | "相同模型，相同失败" | 当智能体共享训练数据或基础模型时的相关错误。 |
| 谄媚从众 | "与最大声的声音同意" | 智能体的投票偏向于最先/最大声发言的人。 |
| 核心/边缘 | "层次 BFT" | WBFT 分割：首先小核心共识，边缘节点跟随。限制延迟。 |
