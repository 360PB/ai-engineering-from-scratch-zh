# 评估与协作基准测试

> 2025-2026 年的五项基准测试覆盖了多智能体评估领域。**MultiAgentBench / MARBLE**（ACL 2025，arXiv:2503.01935）以里程碑 KPI 评估星型/链式/树状/图状拓扑结构；**图状拓扑最适合研究场景**，认知规划可将里程碑达成率提升约 3%。**COMMA** 评估多模态非对称信息协作；以 GPT-4o 为代表的前沿模型甚至难以击败随机基线。**MedAgentBoard**（arXiv:2505.12371）涵盖四类医学任务，常发现多智能体并不优于单 LLM。**AgentArch**（arXiv:2509.10769）基准测试企业级智能体架构，组合了工具调用、记忆与编排层。**SWE-bench Pro**（[arXiv:2509.16941](https://arxiv.org/abs/2509.16941)）共 1865 道题目，覆盖 41 个仓库，横跨业务应用、B2B 服务和开发者工具；前沿模型在 Pro 上得分约 23%，而在 Verified 上为 70%+——这是一次现实检验。Claude Opus 4.7（2026 年 4 月）在 Pro 上据报达到 **64.3%**（使用了显式智能体团队协作；尚无 Anthropic 官方一手来源——视为初步数据）；Verdent（智能体脚手架）在 Verified 上达到 **76.1% pass@1**（[Verdent 技术报告](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。**AAAI 2026 Bridge Program WMAC**（https://multiagents.org/2026/）是 2026 年社区的核心关注点。本课以 MARBLE 的指标为基础，执行拓扑结构与指标的全面对比，并确立"仅通过 SWE-bench Verified 不能证明泛化能力"这一规则。

**前置知识：** 第 16 阶段 · 15（投票与辩论拓扑结构）、第 16 阶段 · 23（失败模式）

## 问题

当一篇论文声称"我们的多智能体系统更好"时，问题在于：比什么好、在什么上好、如何测量？2023-2024 年的多智能体评估时代是一片混乱——每个人都自选指标、自选基线、自选任务集。2025-2026 年的基准测试引入了结构。

没有公认的基准，你无法有意义地比较两个多智能体系统。更糟的是，没有留出式基准，前沿模型可能被污染。SWE-bench Verified 在 2025 年中已部分被混入训练语料；前沿分数被抬高；Pro 被设计为未受污染的现实检验。

本课列举 2026 年的五项经典基准测试，说明各自测量什么，并教你以批判眼光阅读基准测试声明。

## 概念

### MultiAgentBench（MARBLE）——ACL 2025

arXiv:2503.01935。在研究、编码和规划任务上评估四种协作拓扑结构（星型、链式、树状、图状）。基于里程碑的 KPI 追踪部分进展，而非仅评估最终成功。

实测结果：

- **图状**拓扑最适合研究场景；支持任意节点间的互相批评。
- **链式**最适合逐步精化的编码任务。
- **星型**最适合快速事实整合。
- 超过约 **4 个智能体**后出现"协作税"。
- **认知规划**在各拓扑结构中将里程碑达成率提升约 3%。

使用场景：需要将协作拓扑结构进行同类比较时。MARBLE 代码库（https://github.com/ulab-uiuc/MARBLE）提供了评估器。

### COMMA——多模态非对称信息

涵盖各智能体拥有不同观测模态、须在信息不完整共享的情况下协作的任务。报告的结果令人不安：在 COMMA 的智能体间协作任务上，包括 GPT-4o 在内的前沿模型甚至难以击败**随机基线**。这一信号表明：多智能体多模态任务尚未得到充分训练和充分评估——LLM 能合理处理单模态合作；但多模态协作会崩溃。

使用场景：你的系统涉及多模态或非对称信息协作时。COMMA 的空结果是一个警示：声称之前先测量。

### MedAgentBoard——领域压力测试

arXiv:2505.12371。四类医学任务：诊断、治疗方案生成、报告撰写、患者沟通。比较多智能体与单 LLM 以及传统规则系统。

结论：多智能体**并不能**在大多数类别中优于单 LLM。多智能体的优势是有限的——当子任务可清晰分离时（诊断 + 治疗），任务分解有帮助；当协调开销超过专业化收益时（报告撰写），多智能体反而有害。

使用场景：你的领域有明确可用的单 LLM 基线时。如果 MedAgentBoard 的结论具有普遍性，许多提出的多智能体系统存在过度工程化问题。

### AgentArch——企业架构

arXiv:2509.10769。企业场景下，工具使用、记忆与编排层层叠加。基准测试隔离每个层的贡献：添加工具有多大帮助？添加记忆呢？添加多智能体编排呢？

使用场景：设计企业级智能体技术栈，并需要论证每一层价值时。AgentArch 有助于避免购买你无法衡量其价值的特性。

### SWE-bench Pro——现实检验

arXiv:2509.16941。1865 道题目，覆盖 41 个仓库，横跨业务应用、B2B 服务和开发者工具。设计上**不受污染**——采用较晚的训练截止日期。前沿模型在 Pro 上得分约 23%，在 Verified 上为 70%+。差距就是污染信号。

2026 年 4 月成绩：
- Claude Opus 4.7 在 Pro 上：**64.3%**（使用显式智能体团队协作报告；尚无 Anthropic 官方一手来源——视为初步数据）。
- Verdent（智能体脚手架）在 Verified 上：**76.1% pass@1**（[技术报告](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。
- 前沿模型在 Pro 上无智能体脚手架的原始分数：约 23-35%（[SWE-bench Pro 论文](https://arxiv.org/abs/2509.16941)）。

核心启示："我们通过了 SWE-bench Verified"不再是能力的证明。Pro 是当前的准入测试。智能体团队脚手架在 Pro 上产生可衡量的收益（约 30-40 个百分点的差距），这是 2026 年多智能体协作最有说服力的实证论据之一。

### AAAI 2026 WMAC

AAAI 2026 Bridge Program——多智能体协作研讨会（https://multiagents.org/2026/）。2026 年多智能体 AI 研究的社区核心关注点。录稿论文和研讨会论文集是评估新方法的权威平台；对于生产决策，应优先采纳 WMAC 录稿论文的结论，而非 arXiv 预印本。

### 以批判眼光阅读基准测试声明——2026 年检查清单

当有人声称多智能体结果时：

1. **哪个基准测试、哪个分割？** S WE-bench Verified 与 Pro 差异巨大。在错误的分割上报告的数字毫无价值。
2. **污染检查。** 该基准测试是在模型训练截止日期之后发布的吗？如果不是，谨慎对待。
3. **基线对比。** 对比的是单 LLM 基线、随机基线，还是以往的多智能体工作？不是"对比同一系统未调参的版本"。
4. **统计显著性。** N 次试验、p 值、置信区间。前沿模型高方差；单次运行会误导。
5. **任务多样性。** 一项任务还是多项任务？泛化能力对生产环境很重要。
6. **成本披露。** 每任务 token 数、墙上时钟时间。90% 的方案需要 20 倍成本，这是商业决策，不是能力声明。

### 所有基准测试都测量不好的领域

- **长时程协作。** 墙上时钟时间长达数天。所有现有基准测试都只运行短期任务。
- **对抗性韧性。** 当一个智能体被恶意控制或攻陷时会发生什么？
- **部署后的分布漂移。** 基准测试是静态的；生产环境分布会变化。
- **成本归一化性能。** 大多数基准测试报告原始准确率，而非准确率/美元。

为你真正关心的维度构建内部基准测试往往才是正确的做法。

## 动手实现

`code/main.py` 是一个非交互式演练：

- 在一个玩具任务上模拟 3 个多智能体系统。
- 为每个系统计算 MARBLE 风格的里程碑指标。
- 通过从"训练"集中扣留任务来运行污染检查。
- 显式对比随机基线。
- 输出基准测试声明记分卡。

运行：

```bash
python3 code/main.py
```

预期输出：系统记分卡，包含原始准确率、里程碑达成率、每任务成本、与随机基线的差距，以及污染检查备注。

## 产出

生产环境评估规范：

- **构建内部基准测试**，反映你实际的生产分布。公开基准测试有参考价值，但不能替代。
- **在每次对比中加入随机基线**。如果你在协作任务上不能大幅击败随机基线，说明任务本身可能定义不当。
- **准确率与成本一同报告。** Token 成本和墙上时钟时间。运维团队两者都需要。
- **每季度重建基准测试。** 生产分布会变化；过时的基准测试会误导。
- **避免过度拟合公开基准测试。** 如果你的团队专门针对 SWE-bench Pro 数字做优化，在生产环境上会退化。

## 练习

1. 运行 `code/main.py`。识别三个模拟系统中每里程碑成本最低的系统。它是否与原始准确率最高的系统一致？
2. 阅读 MultiAgentBench（arXiv:2503.01935）。针对你自己的任务领域，决定 MARBLE 会推荐哪种拓扑结构。从论文结果中给出理由。
3. 阅读 SWE-bench Pro 论文。是什么让它具有抗污染性？同样的技术能否应用于你关心的其他基准测试？
4. 阅读 COMMA 关于多模态协作的结论。设计一个简单的多模态协作任务，可以加入你的内部基准测试。什么样的信号才算有价值？
5. 用基准测试声明检查清单评估一篇近期多智能体论文的头条结论。你会给该声明什么评分？

## 关键术语

| 术语 | 常见说法 | 实际含义 |
|------|---------|---------|
| MARBLE | "MultiAgentBench" | ACL 2025；以里程碑 KPI 评估星型/链式/树状/图状拓扑结构。 |
| COMMA | "多模态基准测试" | 多模态非对称信息协作；前沿模型难以击败随机基线。 |
| MedAgentBoard | "领域压力测试" | 四类医学任务；常发现多智能体并不优于单 LLM。 |
| AgentArch | "企业基准测试" | 工具 + 记忆 + 编排层层叠加。 |
| SWE-bench Pro | "抗污染" | 1865 题，41 个仓库；Pro 上约 23% vs Verified 上 70%+（污染信号）。 |
| Milestone achievement（里程碑达成率） | "部分得分" | 奖励进展而不只评估最终成功的基准测试。 |
| Contamination（污染） | "基准测试泄露进训练" | 发布后，基准测试混入训练语料；分数被抬高。 |
| WMAC | "AAAI 2026 Bridge Program" | 多智能体协作研讨会；社区核心关注点。 |
| Coordination tax（协作税） | "协作开销" | 超过 4 个智能体后协调成本超过并行收益。 |
