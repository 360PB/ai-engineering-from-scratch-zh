# 投票、自我一致性与辩论拓扑

> 最便宜的聚合：采样 N 个独立智能体，多数投票。Wang 等 2022 年自我一致性用采样 N 次的同一模型做到了。多智能体将其扩展到**异构**智能体以逃脱同质化——不同模型、不同提示、不同温度、不同上下文。除了多数投票，辩论拓扑很重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星形/链形/树形/图形协调，发现**图形最适合研究**，超过约 4 个智能体后有"协调税"。AgentVerse（ICLR 2024）记录了两个涌现模式——志愿者行为和从众行为——从众既是特性（发现共识）也是风险（群体思维，Lesson 24）。本课映射拓扑空间，构建每个变体，并测量协调税。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置知识：** Phase 16 · 07（心灵社会与辩论），Phase 16 · 14（共识与 BFT）
**时间：** 约 75 分钟

## 问题

辩论可以提高准确率（Du 等，arXiv:2305.14325）。它也可以降低它。辩论是否有帮助取决于四个结构性选择：

1. 谁与谁交谈（拓扑）。
2. 多少轮（Du 2023：轮次和智能体都独立地重要）。
3. 智能体是否异构（不同基础模型打破同质化）。
4. 是否存在对抗性声音（钢铁侠 vs 稻草人）。

将"运行 5 个智能体并投票"放到任务上的团队经常比单个智能体差。失败不是随机的。它们追踪拓扑和异构性。本课是拓扑地图。

## 概念

### 自我一致性，单模型基线

Wang 等 2022 年（"自我一致性改善思维链推理"）在 temperature > 0 下采样同一模型 N 次，并对推理路径答案进行多数投票。结果在 GSM8K 上：N=40 样本比单次贪心解码有实质性增益。自我一致性是单智能体多数投票的前身。

限制：自我一致性使用一个基础模型。错误由构造而相关。如果模型有系统性偏差，所有 N 个样本都共享它。

### 多智能体投票，异构扩展

将 N 个样本替换为 N 个**不同**智能体。不同基础模型（Claude、GPT、Llama），不同提示，不同工具访问。好处：错误不相关。代价：不同智能体花费不同；协调它们增加开销。

2026 年异构辩论的规范名称是 **A-HMAD** — 对抗性异构多智能体辩论。不是普遍采用，但论文用这个术语表示"不同模型辩论，减少同质化崩溃的相关错误。"

### 四种拓扑

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

星形：一个中心，所有其他只与中心对话。等同于无回程通道的主管-工作器。
链形：线性，每个智能体看到前一个的输出。类似流水线。
树形：层次，用于层次智能体系统（Lesson 06）。
图形：任意到任意。包括全连接团和任意 DAG。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在包括研究、编码和规划的任务套件上对星形、链形、树形、图形进行了基准测试。关键测量结果：

- **图形**拓扑在研究任务上胜出。信息任意流向任意；智能体可以相互批评。
- **星形**在快速答案事实任务上胜出。中心过滤并整合。
- **链形**在逐步流水线（阶段精炼）上胜出。
- 超过约 4 个智能体时出现**协调税**。墙钟时间和 token 成本增长快于质量。

4 个智能体的上限是经验的，不是根本的。它反映 2026 年 LLM 上下文容量：每个智能体的上下文充满同行的输出，一旦每个人都能看到每个人，添加第 N+1 个智能体的边际价值下降。

### 多智能体辩论策略（"我们应该变得 MAD 吗？"）

arXiv:2311.17371 是 2023 年 MAD 策略调查。关键发现被其他人复制：与自我一致性**结构相似**（独立采样 + 聚合）的 MAD 变体在使用相同预算时经常表现不如自我一致性。当智能体真正异构且辩论具有对抗性结构（一个智能体反对）时，MAD 帮助最大。

### AgentVerse 涌现模式

AgentVerse（ICLR 2024，https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf）记录了即使没有显式设计也会从多智能体辩论中涌现的两种行为：

- **志愿者。** 智能体主动提供帮助（"我可以承担下一步"）。有用：它将工作分配给子任务最有能力的智能体。
- **从众。** 智能体调整立场以匹配批评者，即使批评者是错的。这是辩论版的谄媚（Lesson 14）。

从众是"辩论直到达成一致"奖励欺负者的原因。有限轮次加单独裁判可以缓解。

### 异构性：实际移动准确率的旋钮

2024-2026 年实践文献中的一个模式：将 N 个智能体中的一个换成不同的基础模型，比将 N 增加 1 得到更大的准确率提升。直觉是同质化——每个新的独立错误源比一个额外的相关样本更有价值。

在极限，异构性胜过数量。在大多数有清晰真实答案的任务上，三个不同模型胜过五个同一模型的副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中引用）形式化了"陪审团"——一小群专业智能体，通过在每个阶段投票来完善答案。与普通多数投票不同，陪审团有角色：一个智能体盘问，一个提供上下文，一个评分可信度。陪审团方法是普通投票（便宜，易受同质化）和完整 MAD（昂贵，易受从众）之间的中间地带。

### 何时投票加辩论占主导

- 问题有真实答案（事实、数学、代码行为）。投票收敛是有意义的。
- 智能体可以访问不同源或工具（异构性可用）。
- 轮次有界（通常 2-3）并且有单独的法官或验证器。
- 预算允许 3-5 个智能体。在图形拓扑上超过 5-7，协调税占主导地位。

### 何时投票加辩论有害

- 问题是意见形的。智能体收敛到看起来最自信的答案，而不是最正确的。
- 所有智能体共享一个基础模型。同质化使共识毫无意义。
- 轮次无界。从众每次都赢。
- 任务简单。使用 N=5 的自我一致性的单个智能体更便宜且同样准确。

## 构建

`code/main.py` 实现：

- `run_star(agents, hub, question)` — 中心轮询每个工作器，聚合。
- `run_chain(agents, question)` — 顺序精炼。
- `run_tree(root, children, question)` — 深度-2 聚合的层次结构。
- `run_graph(agents, question, rounds)` — 全对全辩论，有限轮次。
- 脚本化异构性旋钮：每个智能体有一个 `error_bias` 表示其系统性错误。
- 测量工具：在 N=3、5、7 下运行每个拓扑并报告（准确率、total_tokens、wallclock_simulated）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N →（准确率、token、延迟）的表格。图形在研究风格任务上 N=3-5 时胜出；星形在快速事实任务上胜出；图形在 N=7 时显示协调税（延迟膨胀快于准确率）。

## 使用

`outputs/skill-topology-picker.md` 是一个技能，读取任务描述并推荐拓扑（星形/链形/树形/图形）、N（智能体数量）、异构性配置（要使用的基础模型）和轮次上限。

## 交付

对于任何集合：

- 从 **N=5 的自我一致性**开始，使用一个强基础模型。这是便宜的基线。
- 如果准确率重要，升级到 **N=3 的异构投票**。测量差值。
- 只有在任务有结构（研究、多步）且有限轮次可行时，才升级到 **辩论拓扑**。
- 始终记录少数聚类。当少数持续正确时，你有多样性信号。
- 与准确率一起对墙钟和 token 进行基准测试。"在 10 倍成本下更好的准确率"是一个业务决策。

## 关键术语

| 术语 | 人们常说 | 实际含义 |
|------|----------------|------------------------|
| 自我一致性 | "采样 N 次，投票" | Wang 2022。单一模型，N 个 temperature>0 样本，对推理路径多数投票。 |
| 异构性 | "不同模型" | 不同基础模型或提示家族的集合。打破同质化。 |
| MAD | "多智能体辩论" | 智能体在多轮交换批评的通用术语。见 Du 2023。 |
| A-HMAD | "对抗性异构 MAD" | MAD 变体，强调不同模型 + 对抗性结构。 |
| 拓扑 | "谁与谁交谈" | 星形、链形、树形、图形。决定信息流。 |
| 协调税 | "收益递减" | 在图形上超过约 4 个智能体，成本增长快于质量。 |
| 志愿者行为 | "主动帮助" | AgentVerse 涌现模式：智能体主动承担一步。 |
| 从众行为 | "压力下的协议" | AgentVerse 涌现模式：智能体与批评者对齐。 |
| 陪审团 | "小型专业小组" | Sibyl 风格有角色（盘问者、上下文、评分者）的集合。 |
