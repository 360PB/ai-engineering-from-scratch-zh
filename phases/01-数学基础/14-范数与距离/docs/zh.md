# 范数与距离

> 你的距离函数定义了"相似"的含义。选错了，下游一切都毁。

**类型：** 构建
**语言：** Python
**前置要求：** Phase 1，课程 01（线性代数直觉）、02（向量、矩阵与运算）
**时间：** 约 90 分钟

## 学习目标

- 从零实现 L1、L2、余弦、Mahalanobis、Jaccard 和编辑距离函数
- 为给定 ML 任务选择合适的距离度量，并解释为什么替代方案会失败
- 将 L1 和 L2 范数与 LASSO 和 Ridge 正则化及其几何约束区域联系起来
- 演示相同数据集在不同度量下产生不同的最近邻

## 问题背景

你有两个向量。也许它们是词嵌入。也许它们是用户画像。也许它们是像素数组。你需要知道：它们有多接近？

答案完全取决于你选择哪个距离函数。两个数据点可以是一种度量下的最近邻而在另一种度量下相距甚远。你的 KNN 分类器、推荐引擎、向量数据库、聚类算法、损失函数——它们都依赖这个选择。选错了，你的模型优化的就是错误的东西。

没有通用的最佳距离。L2 适用于空间数据。余弦相似度主导 NLP。Jaccard 处理集合。编辑距离处理字符串。Mahalanobis 考虑相关性。Wasserstein 移动概率质量。每个都编码了不同的"相似"假设。

本课程从零构建每个主要距离函数，展示何时是正确工具，并演示相同数据在不同度量下如何产生完全不同的最近邻。

## 核心概念

### 范数：测量向量大小

范数测量向量的"大小"。每个两个向量之间的距离函数都可以写成它们差的范数：d(a, b) = ‖a - b‖。所以理解范数就是理解距离。

### L1 范数（曼哈顿距离）

L1 范数对所有分量的绝对值求和。

```
‖x‖_1 = |x_1| + |x_2| + ... + |x_n|
```

它被称为曼哈顿距离，因为它测量你在只能沿轴移动的城市网格中行走的距离。没有对角线。

```
点 A = (1, 1)
点 B = (4, 5)

L1 距离 = |4-1| + |5-1| = 3 + 4 = 7

在网格上，你向东走 3 个街区，向北走 4 个街区。
```

何时使用 L1：
- 高维稀疏数据（文本特征、独热编码）
- 当你想要对离群值稳健时（单个大差异不会主导）
- 特征选择问题（L1 正则化促进稀疏）

与 L1 正则化（Lasso）的联系：在损失函数中添加 ‖w‖_1 惩罚绝对权重值之和。这将小权重推到恰好为零，执行自动特征选择。L1 惩罚在权重空间中创建菱形约束区域，而菱形的角在轴上，即某些权重为零的地方。

与损失函数的联系：平均绝对误差（MAE）是预测和目标之间 L1 距离的平均值。它线性惩罚所有误差，与 MSE 相比对离群值稳健。

### L2 范数（欧几里得距离）

L2 范数是直线距离。平方根下分量平方和。

```
‖x‖_2 = sqrt(x_1^2 + x_2^2 + ... + x_n^2)
```

这是你在几何课中学过的距离。毕达哥拉斯定理的 n 维形式。

```
点 A = (1, 1)
点 B = (4, 5)

L2 距离 = sqrt((4-1)^2 + (5-1)^2) = sqrt(9 + 16) = sqrt(25) = 5.0

直线，对角线穿过网格。
```

何时使用 L2：
- 低到中等维连续数据
- 当特征规模可比时
- 物理距离（空间数据、传感器读数）
- 像素级图像相似度

与 L2 正则化（Ridge）的联系：在损失函数中添加 ‖w‖_2^2 惩罚大权重。与 L1 不同，它不会将权重推到零。它按比例将所有权重缩向零。L2 惩罚创建圆形约束区域，所以轴上没有角。权重变小但很少恰好为零。

与损失函数的联系：均方误差（MSE）是 L2 距离平方的平均值。平方惩罚大误差比小误差重得多。

```
MAE（L1 损失）：  |y - y_hat|         线性惩罚。对离群值稳健。
MSE（L2 损失）：  (y - y_hat)^2       二次惩罚。对离群值敏感。
```

### Lp 范数：通用族

L1 和 L2 是 Lp 范数的特例：

```
‖x‖_p = (|x_1|^p + |x_2|^p + ... + |x_n|^p)^(1/p)
```

不同的 p 值产生不同形状的"单位球"（距原点距离 1 的所有点的集合）：

```
p=1:    菱形      （角在轴上）
p=2:    圆/球      （通常的圆球）
p=3:    超椭圆       （圆角方形）
p=inf:  正方形/超立方体   （沿轴的平面边）
```

### L-无穷范数（切比雪夫距离）

当 p 趋近无穷时，Lp 范数收敛到最大绝对分量。

```
‖x‖_inf = max(|x_1|, |x_2|, ..., |x_n|)
```

两点之间的距离由它们在最不同维度上的单维决定。所有其他维度被忽略。

```
点 A = (1, 1)
点 B = (4, 5)

L-inf 距离 = max(|4-1|, |5-1|) = max(3, 4) = 4
```

何时使用 L-无穷：
- 当任何单一维度的最坏情况偏差重要时
- 棋盘（象棋中的国王以 L-无穷移动：任何方向一步代价为 1）
- 制造容差（每个维度必须在规格内）

### 余弦相似度与余弦距离

余弦相似度测量两个向量之间的角度，忽略它们的大小。

```
cos_sim(a, b) = (a . b) / (‖a‖_2 * ‖b‖_2)
```

它的范围从 -1（相反方向）到 +1（相同方向）。垂直向量的余弦相似度为 0。

余弦距离将其转换为距离：cosine_distance = 1 - cosine_similarity。它的范围从 0（相同方向）到 2（相反方向）。

```
a = (1, 0)    b = (1, 1)

cos_sim = (1*1 + 0*1) / (1 * sqrt(2)) = 1/sqrt(2) = 0.707
cos_dist = 1 - 0.707 = 0.293
```

为什么余弦主导 NLP 和嵌入：在文本中，文档长度不应影响相似度。一篇关于猫的文档是另一篇关于猫文档的两倍长，应该仍然是"相似的"。余弦相似度忽略大小（长度），只关心方向。两个具有相同词分布但不同长度的文档指向相同方向，获得余弦相似度 1.0。

何时使用余弦相似度：
- 文本相似度（TF-IDF 向量、词嵌入、句子嵌入）
- 大小是噪声而方向是信号的领域
- 推荐系统（用户偏好向量）
- 嵌入搜索（向量数据库几乎总是使用余弦或点积）

### 点积相似度 vs 余弦相似度

两个向量的点积是：

```
a . b = a_1*b_1 + a_2*b_2 + ... + a_n*b_n
      = ‖a‖ * ‖b‖ * cos(angle)
```

余弦相似度是按两个大小归一化的点积。当两个向量已经归一化到单位长度（大小 = 1）时，点积和余弦相似度相同。

```
如果 ‖a‖ = 1 且 ‖b‖ = 1:
    a . b = cos(a 和 b 之间的角度)
```

当它们不同时：点积包含大小信息。具有更大大小的向量获得更高的点积分数。在某些检索系统中这很重要，你希望"热门"项目排名更高。大小作为隐含的质量或重要性信号。

```
a = (3, 0)    b = (1, 0)    c = (0, 1)

dot(a, b) = 3     dot(a, c) = 0
cos(a, b) = 1.0   cos(a, c) = 0.0

两者在方向上达成一致，但点积也反映大小。
```

实践中：
- 当你想要纯方向相似度时使用余弦相似度
- 当大小携带有意义信息时使用点积
- 许多向量数据库（Pinecone、Weaviate、Qdrant）让你在它们之间选择
- 如果你的嵌入已经 L2 归一化，选择无关

### Mahalanobis 距离

欧几里得距离对所有维度一视同仁。但如果你的特征相关或规模不同，L2 给出误导性结果。

Mahalanobis 距离考虑数据的协方差结构。

```
d_M(x, y) = sqrt((x - y)^T * S^(-1) * (x - y))
```

其中 S 是数据的协方差矩阵。

直观上：Mahalanobis 距离首先去相关和归一化数据（白化），然后在变换后的空间中计算 L2 距离。如果 S 是单位矩阵（不相关、单位方差特征），Mahalanobis 距离退化为欧几里得距离。

```
示例：身高和体重是相关的。
一个 6'2" 和 180 磅的人并不罕见。
一个 5'0" 和 180 磅的人很罕见。

欧几里得距离可能说他们离均值同样远。
Mahalanobis 距离正确地识别第二个为异常值，
因为它考虑了身高-体重的相关性。
```

何时使用 Mahalanobis 距离：
- 离群值检测（与均值具有大 Mahalanobis 距离的点是离群值）
- 当特征规模和相关性不同时的分类
- 当你有足够数据来估计可靠的协方差矩阵时
- 制造业质量控制（多元过程监控）

### Jaccard 相似度（用于集合）

Jaccard 相似度测量两个集合之间的重叠。

```
J(A, B) = |A ∩ B| / |A ∪ B|
```

它的范围从 0（无重叠）到 1（相同集合）。Jaccard 距离 = 1 - Jaccard 相似度。

```
A = {cat, dog, fish}
B = {cat, bird, fish, snake}

交集 = {cat, fish}         大小 = 2
并集 = {cat, dog, fish, bird, snake}  大小 = 5

Jaccard 相似度 = 2/5 = 0.4
Jaccard 距离 = 0.6
```

何时使用 Jaccard：
- 比较标签、类别或特征集合
- 基于词存在（而非频率）的文档相似度
- 近似重复检测（MinHash 近似 Jaccard）
- 比较二元特征向量（存在/不存在数据）
- 评估分割模型（Intersection over Union = Jaccard）

### 编辑距离（Levenshtein 距离）

编辑距离计算将一个字符串转换为另一个所需的最少单字符操作数。操作有：插入、删除或替换。

```
"kitten" -> "sitting"

kitten -> sitten  （k -> s 替换）
sitten -> sittin  （e -> i 替换）
sittin -> sitting （插入 g）

编辑距离 = 3
```

使用动态规划计算。填充一个矩阵，其中条目 (i, j) 是字符串 A 的前 i 个字符和字符串 B 的前 j 个字符之间的编辑距离。

```
        ""  s  i  t  t  i  n  g
    ""   0  1  2  3  4  5  6  7
    k    1  1  2  3  4  5  6  7
    i    2  2  1  2  3  4  5  6
    t    3  3  2  1  2  3  4  5
    t    4  4  3  2  1  2  3  4
    e    5  5  4  3  2  2  3  4
    n    6  6  5  4  3  3  2  3
```

何时使用编辑距离：
- 拼写检查和纠正
- DNA 序列比对（带加权操作）
- 模糊字符串匹配
- 清理文本数据的去重

### KL 散度（不是距离，但按距离方式使用）

KL 散度测量一个概率分布与另一个的差异。在课程 09 中有介绍，但属于这里的讨论，因为人们把它当作"距离"使用，尽管它不是。

```
D_KL(P || Q) = sum(p(x) * log(p(x) / q(x)))
```

关键性质：KL 散度**不是对称的**。

```
D_KL(P || Q) != D_KL(Q || P)
```

这意味着它不满足距离度量的基本要求。它也不满足三角不等式。它是散度，不是距离。

前向 KL（D_KL(P || Q)）是"均值寻找"：Q 试图覆盖 P 的所有模式。
反向 KL（D_KL(Q || P)）是"模式寻找"：Q 专注于 P 的单个模式。

当你看到 KL 散度时：
- VAE（ELBO 中的 KL 项推动隐分布靠近先验）
- 知识蒸馏（学生试图匹配教师分布）
- RLHF（KL 惩罚使微调模型靠近基础模型）
- 策略梯度方法（约束策略更新）

### Wasserstein 距离（推土机距离）

Wasserstein 距离测量将一个概率分布转换为另一个所需的最小"工作"量。可以这样想：如果一个分布是一堆土，另一个是一个坑，你需要移动多少土，走多远？

```
W(P, Q) = 所有运输计划 gamma 上关于 E[d(x, y)] 的下确界
```

对于 1D 分布，它简化为累积分布函数绝对差值的积分：

```
W_1(P, Q) = 积分 |CDF_P(x) - CDF_Q(x)| dx
```

为什么 Wasserstein 重要：
- 它是一个真正的度量（对称，满足三角不等式）
- 即使分布不重叠也能提供梯度（KL 散度趋于无穷）
- 这一性质使 Wasserstein GAN（WGAN）成为可能，解决了原始 GAN 的训练不稳定性问题

```
无重叠的分布：

P: [1, 0, 0, 0, 0]    Q: [0, 0, 0, 0, 1]

KL 散度：无穷（零的对数）
Wasserstein：4（移动所有质量 4 个箱）

Wasserstein 给出有意义的梯度。KL 不行。
```

何时使用 Wasserstein：
- GAN 训练（WGAN、WGAN-GP）
- 比较可能不重叠的分布
- 最优传输问题
- 图像检索（比较颜色直方图）

### 为什么不同任务需要不同距离

| 任务 | 最佳距离 | 原因 |
|------|---------|------|
| 文本相似度 | 余弦 | 大小是噪声，方向是含义 |
| 图像像素比较 | L2 | 空间关系重要，特征规模可比 |
| 稀疏高维特征 | L1 | 稳健，不会放大稀有大差异 |
| 集合重叠（标签、类别） | Jaccard | 数据自然为集合值，而非向量 |
| 字符串匹配 | 编辑距离 | 操作映射到人类编辑直觉 |
| 离群值检测 | Mahalanobis | 考虑特征相关性和规模 |
| 比较分布 | KL 散度 | 测量使用 Q 代替 P 丢失的信息 |
| GAN 训练 | Wasserstein | 即使分布不重叠也能提供梯度 |
| 嵌入（向量数据库） | 余弦或点积 | 嵌入被训练为在方向中编码含义 |
| 推荐 | 点积 | 大小可以编码受欢迎程度或置信度 |
| DNA 序列 | 加权编辑距离 | 替换成本因核苷酸对而异 |
| 制造业 QC | L-无穷 | 任何维度的最坏情况偏差重要 |

### 与损失函数的联系

损失函数是应用于预测与目标的距离函数。

```
损失函数           使用的距离       行为
MSE                 L2 平方             严重惩罚大误差
MAE                 L1                  平等惩罚所有误差
Huber 损失          大误差用 L1，       两者兼得：对离群值稳健，
                    小误差用 L2          在零附近平滑梯度
交叉熵             KL 散度          测量分布不匹配
Hinge 损失          max(0, margin - d)  仅在低于 margin 时惩罚
Triplet 损失        L2（通常）          将正例拉近，将负例推远
对比损失            L2                  相似对接近，不相似对超过 margin
```

### 与正则化的联系

正则化在损失函数中添加权重上的范数惩罚。

```
L1 正则化（Lasso）：   loss + lambda * ||w||_1
  -> 稀疏权重。某些权重恰好为零。
  -> 自动特征选择。
  -> 解有角（零处不可微）。

L2 正则化（Ridge）：   loss + lambda * ||w||_2^2
  -> 小权重。所有权重缩向零。
  -> 无特征选择（没有权重恰好为零）。
  -> 处处平滑解。

Elastic Net：          loss + lambda_1 * ||w||_1 + lambda_2 * ||w||_2^2
  -> 结合 L1 的稀疏性和 L2 的稳定性。
  -> 相关特征组被一起保留或丢弃。
```

为什么 L1 产生稀疏而 L2 不产生：在 2D 权重空间中描绘约束区域。L1 是菱形，L2 是圆形。损失函数的轮廓（椭圆）最可能在菱形的角处接触，其中一个权重为零。它们在圆上的平滑点接触，其中两个权重都非零。

### 最近邻搜索

每个距离函数都隐含一个最近邻搜索问题：给定一个查询点，在数据集中找到最近的点。

精确最近邻搜索在 n 个点、d 维的数据集中每查询为 O(n * d)。对于大数据集，这太慢。

近似最近邻（ANN）算法以少量精度换取巨大加速：

```
算法         方法                      使用者
KD-树         轴对齐空间划分             scikit-learn（低维）
球树          嵌套超球体                 scikit-learn（中维）
LSH           随机哈希投影              近似重复检测
HNSW          分层可导航小世界          FAISS、Qdrant、Weaviate
IVF           倒排文件索引，             FAISS（十亿规模）
              基于聚类的搜索
乘积量化       压缩向量，               FAISS（内存受限）
              在压缩空间中搜索
```

HNSW（分层可导航小世界）是现代向量数据库中的主导算法。它构建一个多层图，其中每个节点连接到其近似最近邻。搜索从顶层（稀疏、长跳转）开始，下降到底层（密集、短跳转）。

## 使用方法

最常见的实际用途：在向量数据库中查找相似项。

```python
import numpy as np

def cosine_similarity_matrix(X):
    norms = np.linalg.norm(X, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1, norms)
    X_normalized = X / norms
    return X_normalized @ X_normalized.T

embeddings = np.random.randn(1000, 768)

sim_matrix = cosine_similarity_matrix(embeddings)

query_idx = 0
similarities = sim_matrix[query_idx]
top_k = np.argsort(similarities)[::-1][1:6]
print(f"Top 5 most similar to item 0: {top_k}")
print(f"Similarities: {similarities[top_k]}")
```

当你调用 `model.encode(text)` 然后搜索向量数据库时，这就是底层发生的事。嵌入模型将文本映射到向量。向量数据库在查询向量和每个存储向量之间计算余弦相似度（或点积），使用 ANN 算法避免检查所有向量。

## 核心术语

| 术语 | 通常说法 | 实际含义 |
|------|---------|---------|
| 范数 | "向量的大小" | 将向量映射到非负标量的函数，满足三角不等式、绝对齐次性和仅对零向量为零 |
| L1 范数 | "曼哈顿距离" | 绝对分量值之和。在优化中产生稀疏权重。对离群值稳健 |
| L2 范数 | "欧几里得距离" | 分量平方和的平方根。欧几里得空间中的直线距离 |
| Lp 范数 | "通用范数" | 分量绝对值 p 次幂之和的 p 次根。L1 和 L2 是特例 |
| L-无穷范数 | "最大范数"或"切比雪夫距离" | 最大绝对分量值。当 p 趋近无穷时 Lp 范数的极限 |
| 余弦相似度 | "向量之间的角度" | 按两个大小归一化的点积。范围从 -1 到 +1。忽略向量长度 |
| 余弦距离 | "1 减余弦相似度" | 将余弦相似度转换为距离。范围从 0 到 2 |
| 点积 | "未归一化余弦" | 分量乘积之和。等于余弦相似度乘以两个大小 |
| Mahalanobis 距离 | "考虑相关性的距离" | 在已白化（去相关和归一化）空间中的 L2 距离，使用数据协方差矩阵 |
| Jaccard 相似度 | "集合重叠" | 交集大小除以并集大小。用于集合，而非向量 |
| 编辑距离 | "Levenshtein 距离" | 将一个字符串转换为另一个所需的最少插入、删除和替换次数 |
| KL 散度 | "分布之间的距离" | 不是真正的距离（不对称）。测量使用 Q 代替 P 丢失的额外比特数 |
| Wasserstein 距离 | "推土机距离" | 将质量从一个分布运输到另一个的最小工作。真正的度量 |
| 近似最近邻 | "ANN 搜索" | 算法（HNSW、LSH、IVF）比精确搜索快得多地找到近似最近点 |
| HNSW | "向量数据库算法" | 分层可导航小世界图。用于快速近似最近邻搜索的多层图 |
| L1 正则化 | "Lasso" | 在损失中添加权重 L1 范数。驱动权重为零（稀疏） |
| L2 正则化 | "Ridge"或"权重衰减" | 在损失中添加权重 L2 范数平方。将权重缩向零而不稀疏 |
| Elastic Net | "L1 + L2" | 结合 L1 和 L2 正则化。比单独使用任一者更好地处理相关特征组 |
