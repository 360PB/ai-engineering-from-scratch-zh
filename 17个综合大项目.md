# 17 个综合大项目（Capstone）

> 每个项目都是 **20~40 小时的端到端实战**，不是"做个 Demo 截图发朋友圈"，而是"从 0 到 1 做出一个能上线、能被人用、能通过测试的产品"。

---

## 项目 01 — 终端原生编码 Agent（35 小时）

**一句话：做一个像 Claude Code 或 Cursor 那样的命令行 AI 编程助手。**

你输入一句话，比如"修复 worker.rs 里的竞态条件"，Agent 自己规划步骤、读代码、改代码、跑测试，最后直接给你开一个 Pull Request。

**你会遇到的真问题**：
- Agent 在第 47 轮搜索时返回了 8MB 的结果，把上下文窗口撑爆了，怎么截断？
- Agent 想执行 `rm -rf /`，怎么在底层拦住它？
- 50 轮对话花掉 5 美元，怎么控制成本？

**做出来的东西**：一个基于终端的编码 Agent，带计划面板、工具调用流、成本追踪，能在 E2B 沙箱里安全运行，最后自动提交 PR。

**用到哪些课**：环境搭建 · NLP · Transformer · LLM 从零构建 · LLM 工程 · 工具与协议 · Agent 工程 · 自主系统 · 基础设施

## 项目 02 — 代码库语义搜索（20 小时）

**一句话：做一个能"读懂"整个代码库的搜索引擎。**

不是按文件名或关键词搜，而是问"用户认证逻辑在哪"、"哪个函数处理退款"，系统直接定位到正确的函数和注释。

**你会遇到的真问题**：
- 怎么把代码拆成"有意义的块"？按函数？按类？按逻辑单元？
- 代码改了之后，向量数据库里的旧结果怎么更新？
- 怎么区分"这个函数叫 auth"和"这个函数真的在做认证"？

**做出来的东西**：一个跨仓库的语义搜索系统，支持自然语言查询，返回相关代码片段 + 解释。

**用到哪些课**：NLP · Transformer · LLM 工程 · 工具与协议 · 基础设施

## 项目 03 — 实时语音助手（25 小时）

**一句话：做一个 Siri 级别但不依赖苹果的语音助手——你说中文，它听懂、思考、回答，全程流式不卡顿。**

完整 pipeline：语音输入 → 实时识别（ASR）→ LLM 思考 → 语音合成（TTS）输出。中间没有"请稍等"的加载圈。

**你会遇到的真问题**：
- ASR 和 LLM 之间的延迟怎么压到 200ms 以内？
- 用户说话时打断它，怎么优雅处理？
- 不同口音、背景噪音、多人同时说话怎么办？

**做出来的东西**：一个低延迟的语音对话系统，支持打断、多轮对话、自定义音色。

**用到哪些课**：语音与音频 · Transformer · LLM 工程 · 工具与协议 · Agent 工程 · 基础设施

## 项目 04 — 多模态文档问答（25 小时）

**一句话：上传一张发票、一份合同、一张流程图，直接问"总金额多少""第三条违约责任是什么""这个箭头指向哪个模块"。**

系统不是只看文字，是先看图（OCR + 视觉理解），再结合文本理解，最后组织答案。

**你会遇到的真问题**：
- 表格被拍歪了、有水印、光线暗，怎么提取准确？
- 图里有流程图、架构图、柱状图，不同图怎么给不同策略？
- 答案里既有文字内容又有图里的信息，怎么融合？

**做出来的东西**：一个支持图片/PDF/扫描件的多模态问答系统，返回带引用来源的答案。

**用到哪些课**：计算机视觉 · NLP · Transformer · LLM 工程 · 多模态 AI · 基础设施

## 项目 05 — 自主研究 Agent（40 小时）

**一句话：给 Agent 一个模糊的研究方向，它自己查文献、做实验、写论文、过同行评审——全部自动化。**

这是整门课最硬核的项目，对标 Sakana AI 的 AI-Scientist。你给 Agent 一句话："研究 1B 参数以下 Transformer 的注意力稀疏模式"，它自己：
1. 去 Semantic Scholar 搜 50 篇相关论文
2. 提出假设，设计对比实验
3. 在沙箱里跑训练（预算 $30）
4. 把结果写成 LaTeX 论文
5. 5 个 LLM 评委按 NeurIPS 标准打分
6. 过不了的打回去重写，最多 3 轮

**你会遇到的真问题**：
- Agent 写的代码会不会偷偷 `curl` 外传数据？沙箱怎么防逃逸？
- 每次实验花 $5，怎么在预算内找到最好的方向？
- 自动生成的论文会不会胡说八道？评审怎么发现？

**做出来的东西**：一个端到端的自主研究 Agent，输入一个研究方向，输出一篇可评审的论文 + 完整实验复现包。

**用到哪些课**：ML 基础 · 深度学习 · Transformer · LLM 从零构建 · Agent 工程 · 自主系统 · 多智能体 · 伦理安全

## 项目 06 — K8s DevOps 排障 Agent（30 小时）

**一句话：K8s 集群出故障了，Agent 自动看日志、查指标、定位根因、给修复方案。**

不是简单的"Pod 崩溃了"报警，而是"这个微服务响应变慢是因为上游数据库连接池耗尽，同时下游缓存命中率下降"这种根因分析。

**你会遇到的真问题**：
- 日志分散在 20 个 Pod 里，怎么快速聚合和过滤？
- 指标时序数据怎么和事件日志关联？
- Agent 给出的修复建议（比如重启服务）真的安全吗？要不要人工确认？

**做出来的东西**：一个连接 Prometheus/Grafana + K8s API 的排障 Agent，自动诊断常见故障并输出带证据的修复方案。

**用到哪些课**：LLM 工程 · 工具与协议 · Agent 工程 · 自主系统 · 基础设施 · 伦理安全

## 项目 07 — 端到端微调 Pipeline（30 小时）

**一句话：从"有一堆原始数据"到"部署一个微调好的模型"，全部自动化。**

覆盖完整链路：数据清洗 → 分词 → 预训练/微调（SFT + RLHF/DPO）→ 评估 → 量化 → 部署 → 监控。不是调 API，是自己跑训练。

**你会遇到的真问题**：
- 数据里有隐私信息，怎么处理？
- 训练到一半 GPU 挂了，怎么恢复？
- 微调后的模型在测试集上表现好，但生产环境翻车，怎么早发现？

**做出来的东西**：一个可复用的微调 Pipeline，输入数据和配置，输出量化后的部署模型 + 评估报告。

**用到哪些课**：ML 基础 · 深度学习 · Transformer · LLM 从零构建 · LLM 工程 · 基础设施 · 伦理安全

## 项目 08 — 生产级 RAG 聊天机器人（30 小时）

**一句话：做一个企业能用的文档问答机器人，不是玩具——要考虑合规、安全、成本、审计。**

面向受监管行业（金融、医疗、法律）。用户问"根据 2023 版合规手册，这笔交易需要哪些审批"，系统必须：
- 准确检索相关条款
- 给出带引用来源的答案
- 不知道的时候说不知道（不 hallucinate）
- 所有问答留痕审计
- 敏感数据脱敏

**你会遇到的真问题**：
- 用户问"怎么绕过审批流程"，怎么拒绝回答？
- 不同用户权限不同，能看到的内容怎么隔离？
- 答案错了导致用户做错决策，责任怎么界定？

**做出来的东西**：一个通过安全审查的 RAG 系统，带审计日志、权限隔离、护栏、人工复核流程。

**用到哪些课**：NLP · Transformer · LLM 工程 · 多模态 AI · 基础设施 · 伦理安全

## 项目 09 — 代码迁移 Agent（30 小时）

**一句话：把一整个老项目的代码自动升级到新版框架，比如 Python 2 升 3、Vue 2 升 3、或者换 ORM。**

不是正则替换那么简单。Agent 要：
1. 理解旧代码的语义
2. 找出所有不兼容的地方（包括隐式依赖）
3. 生成迁移后的代码
4. 跑测试验证
5. 生成迁移报告

**你会遇到的真问题**：
- 旧代码里有一堆没人记得的 hack，怎么安全处理？
- 改了 500 个文件，怎么保证没漏掉某个边缘情况？
- 测试覆盖率不够，怎么判断迁移是否正确？

**做出来的东西**：一个仓库级代码迁移 Agent，支持自定义迁移规则，输出带测试验证的迁移结果。

**用到哪些课**：NLP · Transformer · LLM 工程 · 工具与协议 · Agent 工程 · 自主系统 · 基础设施

## 项目 10 — 多智能体软件工程团队（40 小时）

**一句话：不是"一个 AI 写代码"，而是"一个 AI 团队写代码"——架构师出方案，4 个程序员并行写，审阅员把关，测试员验收。**

模仿真实团队分工：
- **架构师**（Opus）：读需求，拆任务，定义接口
- **程序员**（Sonnet，4 个并行）：各负责一个子任务，在独立 git worktree 里写
- **审阅员**（GPT）：合并后审代码，不通过打回
- **测试员**（Gemini）：跑全部测试，出报告

**你会遇到的真问题**：
- 4 个程序员同时改同一个文件，合并冲突怎么解决？
- 架构师的需求写得太模糊，程序员各自理解不同怎么办？
- 4 个 AI 跑下来 token 成本是一个 AI 的 4 倍，真的值得吗？

**做出来的东西**：一个多智能体协作系统，输入 GitHub Issue，输出合并就绪的 PR，带每个角色的成本统计。

**用到哪些课**：LLM 工程 · 工具与协议 · Agent 工程 · 自主系统 · 多智能体 · 基础设施

## 项目 11 — LLM 可观测性与评估仪表盘（25 小时）

**一句话：做一个"LLM 系统的 Grafana"——实时监控你的 AI 应用健康度、成本、质量和安全。**

生产环境里跑 LLM 应用，你需要知道：
- 每轮对话花多少钱？哪个用户最烧钱？
- 回答质量在下降吗？ hallucinate 率升高了吗？
- 有没有人试图注入恶意提示？
- 哪个模型的性价比最高？

**做出来的东西**：一个 LLM 可观测性平台，集成 Langfuse/Phoenix，展示实时指标、成本分析、质量评估、安全告警。

**用到哪些课**：LLM 工程 · 工具与协议 · 基础设施 · 伦理安全

## 项目 12 — 视频理解 Pipeline（30 小时）

**一句话：上传一段视频，问"第三个人说了什么""红色汽车什么时候出现""这个场景表达了什么情绪"。**

系统要：
1. 把视频拆成场景片段
2. 提取画面内容（物体、人物、动作、文字）
3. 提取音频内容（对话、音效）
4. 做时序对齐（"在 2:34 处，A 说 X，同时画面出现 Y"）
5. 回答用户问题

**做出来的东西**：一个视频问答系统，支持自然语言查询，返回带时间戳的答案。

**用到哪些课**：计算机视觉 · 语音与音频 · Transformer · LLM 工程 · 多模态 AI · 基础设施

## 项目 13 — MCP Server + 注册表与治理（25 小时）

**一句话：给你的公司内部工具（数据库、Jira、Slack、监控）统一包一层"AI 接口"，让任何 AI 助手都能安全调用。**

MCP（Model Context Protocol）是 2026 年的标准。这个项目你要：
1. 写 10 个内部工具的 MCP Server（比如只读查数据库、看 Jira 工单）
2. 搭一个注册表，让全公司发现有哪些工具可用
3. 加权限控制：谁能查什么、destructive 操作要人工审批
4. 留审计日志：谁、什么时候、调了什么工具

**做出来的东西**：一个生产级 MCP 生态——Server 集群 + 注册表 + 权限治理 + 审计。

**用到哪些课**：LLM 工程 · 工具与协议 · Agent 工程 · 基础设施 · 伦理安全

## 项目 14 — 投机解码推理服务器（25 小时）

**一句话：做一个比标准 vLLM 快 2~3 倍的推理服务，核心技术是"小模型猜、大模型验"。**

投机解码（Speculative Decoding）的原理：用一个小而快的模型猜测下一个 token，大模型一次性验证多个猜测。猜对了就白赚速度，猜错了再退回来修正。

**你会遇到的真问题**：
- 小模型猜得太离谱反而拖慢速度，怎么平衡？
- 不同任务（代码补全 vs 聊天）猜测成功率差异很大，怎么自适应？
- 怎么在 K8s 上横向扩展？

**做出来的东西**：一个支持投机解码的推理服务器，带 EAGLE-3 优化，能横向扩展，比基线快 2 倍以上。

**用到哪些课**：深度学习 · Transformer · LLM 从零构建 · 基础设施

## 项目 15 — Constitutional 安全 Harness + 红队靶场（30 小时）

**一句话：给你的 AI 系统穿上"防弹衣"——能防提示注入、能防越狱、能防数据泄露，还能自己找自己的漏洞。**

不是加个关键词过滤就完事。你要：
1. 设计一套"宪法"规则（Constitutional AI），告诉 AI 什么能说什么不能做
2. 搭红队靶场，自动化攻击自己的系统（ jailbreak、数据提取、权限绕过）
3. 用 Garak、PyRIT、Llama Guard 做安全测试
4. 输出安全评估报告

**做出来的东西**：一个带 Constitutional 安全层的 AI 应用 + 自动化红队测试框架 + 安全评估报告。

**用到哪些课**：LLM 从零构建 · LLM 工程 · 工具与协议 · Agent 工程 · 伦理安全

## 项目 16 — GitHub Issue 自动转 PR Agent（30 小时）

**一句话：在 GitHub Issue 上贴个标签 `@agent fix this`，14 分钟后自动出现一个修复好的 PR。**

和项目 01（终端编码 Agent）的区别：这个是**异步云 Agent**，没有交互终端。触发方式是 GitHub 标签或评论，Agent 在云端沙箱里跑完整个流程，最后自动提交 PR。

**你会遇到的真问题**：
- 怎么自动推断这个仓库的构建环境？（Python？Node？Go？用什么包管理器？）
- PR 打开之前必须保证 CI 通过，怎么在沙箱里跑完整测试？
- Agent 有没有可能 force-push 覆盖主分支？怎么从机制上禁止？
- 每天每个仓库花多少钱，怎么设上限？

**做出来的东西**：一个 GitHub App + 云 Worker，输入带标签的 Issue，输出通过 CI 的 PR，带修复说明和成本统计。

**用到哪些课**：LLM 工程 · 工具与协议 · Agent 工程 · 自主系统 · 基础设施

## 项目 17 — 个人 AI 导师（25 小时）

**一句话：做一个专属于你的 AI 老师——它知道你的学习进度、薄弱点、偏好，自适应调整教学内容。**

不是通用 ChatGPT，是：
- 你学线性代数时卡在特征值，它会检测到并换种方式讲
- 你视觉型学习者，它多画图；你听觉型，它多举例子
- 你上周学过 SVM，这周学神经网络时会自动关联对比
- 支持语音对话、上传手写笔记拍照问、视频讲解

**做出来的东西**：一个自适应多模态 AI 导师，支持个性化学习路径、多模态交互（语音/图像/文本）、学习进度追踪。

**用到哪些课**：NLP · 语音与音频 · LLM 工程 · 多模态 AI · Agent 工程 · 基础设施 · 伦理安全
